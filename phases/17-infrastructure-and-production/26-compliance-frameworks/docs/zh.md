# 合规性 —— SOC 2、HIPAA、GDPR、PCI-DSS、欧盟人工智能法案(EU AI Act)、ISO 42001

> 对于2026年的企业交易，多框架覆盖是基本门槛。**《欧盟AI法案》**：自2024年8月1日起生效。大多数高风险要求于2026年8月2日执行。违反高风险系统义务最高罚款1500万欧元或全球年营业额的3%（第99(4)条）；禁止的AI实践最高罚款3500万欧元或7%（第99(3)条）。如果服务欧盟用户，则全球适用。**《科罗拉多州AI法案》**：2026年6月30日生效（由SB25B-004从2026年2月推迟）——高风险系统需进行影响评估，用户拥有对AI决策的上诉权。弗吉尼亚州在信贷/就业/住房/教育领域类似。**SOC 2 Type II**：B2B AI的实际要求（金融科技要求Type II而非Type I）。**GDPR**：目前最大的AI特定罚款是2024年9月荷兰DPA对Clearview AI处以3050万欧元；意大利Garante于2024年12月对OpenAI处以1500万欧元，但2026年3月上诉后推翻，该裁决仍在进一步审查中。推理时实时PII脱敏是可辩护的标准；后处理清理是不够的。**HIPAA**：医疗保健领域——没有BAA，不能将受保护健康信息发送给外部AI服务。**PCI-DSS**：AI交互层覆盖要求配置+合同协议，而非自动满足。**ISO 42001**：新兴的AI治理标准，与ISO 27001一起成为日益增长的采购要求。参考概要：OpenAI持有SOC 2 Type 2、ISO/IEC 27001:2022、ISO/IEC 27701:2019、GDPR/CCPA/HIPAA (BAA)/FERPA、ChatGPT支付组件的PCI-DSS。跨框架映射减少审计疲劳：访问控制映射到ISO 27001 A.5.15-5.18、GDPR第32条、HIPAA §164.312(a)。

## 学习目标

- 列举2026年与LLM产品相关的七个框架，并将每个框架匹配到对应的客户群体。
- 引用EU AI Act的执行时间线（2024年8月生效；高风险执行2026年8月）和两级罚款上限（高风险义务1500万欧元/3%，禁止实践3500万欧元/7%）。
- 解释为什么后处理PII清理不足以满足GDPR，并指出实时推理层脱敏是可辩护的标准。
- 描述跨框架控制映射（例如，访问控制映射到ISO 27001 A.5.15-5.18 + GDPR第32条 + HIPAA §164.312(a)）。

## 问题

一家企业客户的采购部门要求提供SOC 2 Type II、GDPR、HIPAA BAA、ISO 27001和“EU AI Act合规声明”。你的团队只有SOC 2 Type I。距离Type II还有六个月，并且尚未开始GDPR第30条记录。

多框架覆盖不是LLM问题——而是企业SaaS问题，外加LLM特定覆盖。2026年的采购团队希望看到一个矩阵，每行一个框架，每列一个控制，而不是PDF。

## 核心概念

### 七个框架

|  框架  |  范围  |  LLM特定要求  |
|-----------|-------|--------------------------|
|  SOC 2 Type II  |  B2B SaaS基线  |  过程控制在6-12个月内审计  |
|  HIPAA  |  美国医疗保健  |  需要BAA；未经签署协议，受保护健康信息不能离开基础设施  |
|  GDPR  |  欧盟用户  |  实时PII脱敏；数据主体权利；第30条记录  |
|  PCI-DSS  |  支付数据  |  配置+合同用于涉及支付的AI  |
|  EU AI Act  |  服务欧盟用户  |  风险等级分类；高风险系统：符合性评估、文档化、日志记录  |
|  Colorado AI Act  |  服务科罗拉多州居民  |  影响评估；上诉权  |
|  ISO 42001  |  AI治理  |  新兴；与ISO 27001配对  |

### EU AI Act时间线

- 2024年8月1日：生效。
- 2025年2月2日：禁止的AI实践开始执行。
- 2026年8月2日：高风险系统开始执行（符合性评估、文档化、日志记录）。
- 2027年8月：根据协调立法制定的产品中的高风险系统。

风险等级：不可接受（禁止）、高风险（符合性+日志记录）、有限风险（透明度）、最小风险（无约束）。大多数B2B LLM SaaS属于有限风险；高风险适用于就业、信贷、教育、执法、移民、基本服务。

罚款（第99条）：违反高风险系统义务最高罚款1500万欧元或全球年营业额的3%（第99(4)条）；禁止的AI实践最高罚款3500万欧元或7%（第99(3)条）；以较高者为准。

### GDPR——实时脱敏是标准

后处理清理（在LLM看到PII之后进行脱敏）不是可辩护的立场——模型已经看到了数据。实时推理层脱敏是2026年的标准：

- 在LLM调用之前进行实体识别。
- 一致的分词（Mesh方法）保留语义。
- 仅存储脱敏后的提示和经过同意的选择性原始数据。

近期执法：Clearview AI被罚款3050万欧元（荷兰DPA，2024年9月）是迄今为止最大的AI特定GDPR罚款；OpenAI被罚款1500万欧元（意大利Garante，2024年12月）是最大的LLM特定罚款，但2026年3月上诉后推翻，该裁决仍在进一步审查中。后处理声明在审计中未被接受。

### HIPAA——BAA不是可选的

未经签署的商业伙伴协议，不能将受保护健康信息发送给外部AI服务。三大超大规模云LLM平台（Bedrock、Azure OpenAI、Vertex）都提供BAA。OpenAI直接API提供BAA。Anthropic直接API提供BAA。发送受保护健康信息前请确认。

### SOC 2 Type II

Type I：控制措施已设计和文档化。
Type II：控制措施在6-12个月内有效运行。

2026年B2B采购默认要求Type II。Type I是起点；Type II是门槛。

常见审计驱动因素：访问日志（谁看到了什么）、变更管理（如何部署的）、风险评估（季度）、事件响应（是否测试过？）。阶段17·25的审计日志可直接复用。

### 跨框架映射

一项访问控制策略可满足多个框架的控制要求：

| 控制  |  框架 |
|---------|-----------|
| 访问日志记录  |  ISO 27001 A.5.15-5.18, GDPR Art. 32, HIPAA §164.312(a) |
| 变更管理  |  ISO 27001 A.8.32, PCI DSS Req. 6, HIPAA 违规通知范围 |
| 传输加密  |  ISO 27001 A.8.24, GDPR Art. 32, HIPAA §164.312(e) |
| 机密管理  |  ISO 27001 A.8.19, PCI DSS Req. 8, SOC 2 CC6.1 |

合规工具（Drata, Vanta, Secureframe）可自动完成此映射。在规模较大时值得投入成本。

### ISO 42001 — 新兴标准

2023年底发布。与ISO 27001一同成为日益增长的采购要求。AI治理框架，涵盖风险管理、数据质量、透明度、人工监督。

### OpenAI 参考简介

OpenAI 持有 SOC 2 Type 2、ISO/IEC 27001:2022、ISO/IEC 27701:2019、GDPR/CCPA/HIPAA (BAA)/FERPA、ChatGPT支付组件的PCI-DSS认证。这大致是2026年企业入场的基本要求。

### 你应该记住的数字

- 欧盟AI法案罚款：高至1500万欧元/3%（高风险义务，Art. 99(4)）；高至3500万欧元/7%（禁止行为，Art. 99(3)）。
- 欧盟AI法案高风险执行日期：2026年8月2日。
- 有记录的AI相关最大GDPR罚款：3050万欧元，Clearview AI（荷兰数据保护局，2024年9月）。
- 有记录的大语言模型特定最大GDPR罚款：1500万欧元，OpenAI（意大利Garante，2024年12月；2026年3月上诉推翻）。
- SOC 2 Type II窗口期：已运行控制措施6-12个月。
- 科罗拉多州AI法案生效日期：2026年6月30日（因SB25B-004从2026年2月推迟）。
