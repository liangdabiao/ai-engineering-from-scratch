# STaR, V-STaR, Quiet-STaR — 自教推理 (Self-Taught Reasoning)

> 最小的自我改进循环隐藏在推理过程之中。模型生成一系列推理链，保留那些得到正确答案的推理链，并基于它们进行微调。这就是STaR。V-STaR增加了一个验证器，使得推理时的选择更加精准。Quiet-STaR将推理过程下沉到每个词元级别。这三种方法都有效。它们并非魔法——这个循环会保留任何恰好得到正确答案的捷径。

**类型：** 学习
**语言：** Python（标准库，自举循环模拟器）
**前置知识：** 阶段13 · 01-03（推理与CoT），阶段15 · 01（长周期框架）
**时间：** 约60分钟

## 问题

教模型推理最直接的方法是收集人类编写的推理痕迹。这既昂贵又缓慢，并且受限于人类愿意编写多少高质量的思维链。

STaR（Self-Taught Reasoner，Zelikman 等人，2022）提出：如果模型自己编写推理过程，并根据已知答案对其进行评分，结果会怎样？循环如下：

1. 采样一个推理痕迹和答案。
2. 如果最终答案正确，保留该痕迹。
3. 基于保留的痕迹进行微调。
4. 重复。

它是有效的。GSM8K和CommonsenseQA都在没有新的人工标注的情况下得到了改善。但这个循环有一个固有的偏差：任何产生正确答案的推理过程都会被保留，无论推理本身是否合理。V-STaR（Hosseini 等人，2024）通过一个学习得到的验证器修复了这个问题；Quiet-STaR（Zelikman 等人，2024）将这一思想推广到每个词元的内部推理。

## 核心概念

### STaR：基于有效的方法自举

从一个具有一些弱推理能力的基础模型开始。在每个训练问题上，采样一个推理过程和答案。如果答案与标签匹配，保留（问题，推理过程，答案）三元组。在保留的数据集上对模型进行微调。重复。

有一个关键点。如果模型永远无法正确解答一个问题，那么循环就无法从中学习。STaR增加了**合理化**（rationalization）：对于模型失败的问题，注入正确答案作为提示，并重新提示模型生成一个导向正确答案的推理过程。合理的推理过程会被添加到训练集中。

原始论文中的结果（Zelikman 等人，2022）：通过多次带有合理化的STaR循环，GPT-J基础模型在GSM8K上的准确率从5.8%提升到10.7%——绝对提升约5个百分点。在CommonsenseQA上，经过STaR训练的GPT-J 6B达到了72.5%，与经过微调的GPT-3 175B（~73%）相当——后者是一个大约30倍大的模型，并且是在手工标注的推理过程上训练的。

### V-STaR：使用DPO训练验证器

STaR丢弃了不正确的推理过程。Hosseini 等人（2024）观察到这些也是数据：每一对（推理过程，“是否正确”）都可以训练一个验证器。他们对正确和不正确的解答都使用直接偏好优化来构建一个排序器。在推理时，采样N个推理过程并选择验证器排名最高的一个。

报告的提升：在GSM8K和MATH上，相比先前的自我改进基线方法，提升了+4到+17个百分点，其中大部分增益来自使用验证器进行推理时的选择，而非额外的生成器微调。

### Quiet-STaR：每个词元的内部推理

Zelikman 等人（2024）提出：如果模型学会在每个词元位置生成一个简短的内部推理，而不仅仅是在问题和答案之间，结果会怎样？Quiet-STaR训练模型在每个预测词元之前发射一个隐藏的“想法”，然后通过一个学习得到的权重将考虑想法的预测与基线预测混合。

结果：Mistral 7B在GSM8K上从5.9%提升到10.9%，在CommonsenseQA上从36.3%提升到47.2%，且没有进行任务特定的微调。模型学会了“何时思考”——困难的词元得到更长的内部推理；简单的词元则几乎没有。

### 为什么这三种方法都存在安全隐患

这三种方法都使用最终答案作为梯度信号。一个通过有缺陷的推理（利用捷径、猜测或使用非泛化模式）得到正确答案的推理过程会得到正强化。在分布内的问题上，捷径有效。在分布外的问题上，它会静默失效。

V-STaR的验证器通过学习排序推理过程来缓解问题，但验证器是在相同的标签集上训练的。它可能学会偏好格式良好的错误推理，而非诚实的模棱两可。更安全的设计是将STaR风格的数据与(a)过程监督的奖励模型（奖励中间步骤，而不仅仅是答案）以及(b)能够打破简单捷径的留出分布外评估结合起来。

### 比较

|  方法  |  训练信号  |  推理成本  |  数据浪费  |  已知失败模式  |
|---|---|---|---|---|
|  STaR  |  如果正确则保留（推理过程，答案）  |  1x  |  丢弃所有不正确的推理过程  |  捷径推理过程  |
|  STaR + 合理化  |  上述 + 带正确答案提示的重试  |  1x  |  较少  |  合理的推理过程可能不合理  |
|  V-STaR  |  STaR + 来自两类的DPO验证器  |  Nx（最佳-N）  |  最小  |  验证器可能强化自信的错误  |
|  Quiet-STaR  |  每个词元的推理 + 混合权重  |  1.5-3x  |  最小  |  仍然基于答案条件的梯度  |

### 这在2026年的技术栈中处于什么位置

STaR是旧的。但这一模式在2025-2026年无处不在。可验证数学问题上的强化学习（DeepSeek-R1, Kimi-k1.5, o1）是STaR的基于答案条件的梯度信号，规模更大。过程奖励模型（Lightman 等人，2023；OpenAI的“让我们逐步验证”）是过程监督的替代方案。AlphaEvolve（第3课）是代码版的STaR，使用程序评估器代替标签。Darwin Godel Machine（第4课）是智能体框架本身的STaR。

理解STaR能让所有这些概念豁然开朗。它是最小可行的自我改进循环。

## 使用它

`code/main.py`在玩具算术任务上运行模拟STaR循环。你可以观察到：

- 准确率如何随自举轮次提升。
- 捷径如何潜入：模拟器包含一个“懒惰”的推理过程类，它在40%的情况下得到正确答案，但泛化能力很差。观察STaR是否保留它们。
- 验证器（V-STaR风格）如何在推理时提供帮助，但无法完全剪除训练期间引入的捷径。

## 练习

1. 运行模拟器。将捷径频率设为0，然后设为0.4。尽管两次运行在训练分布上都达到>90%的准确率，但最终准确率相差多少？

2. 在模拟器中添加一个保留的分布外测试集。从不同分布中抽取问题，并在分布内和分布外数据集上评估自举模型。量化差距。

3. 阅读Quiet-STaR论文（arXiv:2403.09629）第3节。分别用三句话解释“思考结束”标记和混合权重头。

4. 比较STaR的“正确则保留”过滤器与另一种过程监督方法，后者独立奖励每个推理步骤。指出标注成本差异和可能的质量差异。

5. 设计一种评估方法，能够捕捉部署模型中的捷径推理。它不必完美——它必须打破STaR循环会加剧的最简单捷径。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|---|---|---|
|  STaR  |  "Self-Taught Reasoner"  |  在模型生成的、导向正确答案的推理上进行微调；重复  |
|  Rationalization  |  "Hinted retry"  |  注入正确答案，并在基础模型失败的问题上重新提示生成推理  |
|  V-STaR  |  "Verifier STaR"  |  在正确和错误的推理上DPO训练一个验证器，用于推理时选择  |
|  Quiet-STaR  |  "Per-token rationales"  |  在每个标记位置生成隐藏思考；与基线预测混合  |
|  Answer-conditioned gradient  |  "Outcome-based signal"  |  训练循环奖励最终答案，而非推理步骤  |
|  Process reward model  |  "Step-level verifier"  |  在每步正确性而非结果上训练的奖励模型——与STaR形成对比  |
|  Shortcut rationale  |  "Right answer, wrong reasoning"  |  通过非泛化模式达到标签的推理；STaR保留这些  |
