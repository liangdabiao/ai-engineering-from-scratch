# CAIS、CAISI 与社会规模风险

> 人工智能安全中心（CAIS，旧金山，由 Hendrycks 和 Zhang 于2022年创立）发布了四风险框架——恶意使用、AI竞赛、组织风险、失控AI——以及2023年5月由数百名教授和公司领导人签署的关于灭绝风险的声明。CAIS 2026年的发布物：前沿模型评估的AI仪表板、远程劳动指数（与Scale AI合作）、超级智能战略论文、AI前沿通讯。另一个不同的实体：NIST人工智能标准与创新中心（CAISI）——面向美国政府的自愿协议和未分类能力评估，重点关注网络、生物和化学武器风险。CAIS将组织风险列为四个顶层风险之一：安全文化、严格审计、多层次防御和信息安全是基础，但通常为了部署速度而做出权衡。加利福尼亚州SB-53（如果签署）将是美国第一个州级灾难性风险法规。

**类型：** 学习
**语言：** Python（标准库，四风险清单与缓解匹配器）
**先决条件：** 第15阶段·19（RSP），第15阶段·20（PF + FSF）
**时间：** 约45分钟

## 问题

第19课和第20课涵盖了实验室内部的扩展策略。第21课涵盖了独立能力评估。本课涵盖第三个视角：塑造公共讨论和灾难性AI风险监管基线的民间社会与政府组织。

两个不同实体很重要。CAIS是一个非营利研究组织，发布思考AI风险的框架并协调公开声明。CAISI是美国政府下属NIST的一个中心，负责与实验室进行自愿协议和未分类能力评估。名称相似但使命不重叠。从业者应该了解两者。

实际内容：CAIS的四风险框架是文献中被引用最广泛的社会规模风险分类法。安全文化和组织风险是其中之一，这也是从业者最直接可控的一个。SB-53（加利福尼亚州）如果签署，将是美国第一个州级灾难性风险法规；该法案的框架很重要，因为州级法规历来引领美国科技政策中的联邦行动。

## 核心概念

### CAIS — 人工智能安全中心

- 成立：2022年在旧金山，由Dan Hendrycks及其同事创立（"Zhang"指早期合作者，非现任联合创始人；当前领导层见CAIS网站）。
- 状态：501(c)(3)非营利组织。
- 2023年重要成果：关于灭绝风险的声明，由数百名研究人员和CEO联署。声明称："减轻来自AI的灭绝风险应成为全球优先事项，与流行病和核战争等其他社会规模风险同等重要。"
- 2026年成果：前沿模型评估的AI仪表板、远程劳动指数（与Scale AI合作）、超级智能战略论文、AI前沿通讯。

### 四风险框架

CAIS的框架将灾难性AI风险分为四个顶层类别：

1. **恶意使用**：不良行为者利用AI造成伤害（生物武器合成、虚假信息、网络攻击）。
2. **AI竞赛**：实验室、公司或国家之间的竞争压力将部署推过安全节点。
3. **组织风险**：实验室内部动态（安全文化失败、审计不足、安全资源不足）导致不良部署。
4. **失控AI**：能力足够强的AI追求与人类福祉相冲突的目标。

这不是唯一的分类法；它是被引用最多的。这些类别并非互斥——一个组织在竞赛中为了速度而牺牲审计所产生的失控AI，同时满足全部四个类别。

### 组织风险的所在

在这四个类别中，组织风险对从业者来说最具可操作性。实验室的安全文化、审计严格性、防御层次和信息安全决定了其模型是否真正配备了第10–18课中的控制措施，或者这些控制措施只是无人验证的清单条目。

具体的组织风险杠杆：

- **安全文化**：团队成员是否感到能够在没有职业成本的情况下提出担忧？CAIS调查发现这是其他杠杆的有力预测指标。
- **严格审计**：外部和内部。仅内部审计会产生乐观的报告。
- **多层次防御**：没有单个层次是足够的（第15阶段的贯穿主题）。
- **信息安全**：模型权重泄露、评估数据泄露、监控绕过技术泄露。第19课中的RAND SL-4是一个具体标准。

### CAISI — 人工智能标准与创新中心

- 在NIST内部运作。
- 与前沿实验室进行自愿协议。
- 发布未分类能力评估，重点关注网络、生物和化学武器风险。
- 与CAIS不同；缩写碰撞；检查URL（nist.gov）以确认您正在阅读哪一个。

CAISI的角色是METR私下实验室接触（第21课）的公共、面向政府的对应方。CAISI报告是未分类的；METR报告通常受NDA保护。阅读两者的从业者能获得更全面的图景。

### 加利福尼亚州SB-53

加利福尼亚州参议院法案（2025–2026会期）涉及前沿模型的灾难性风险。起草的关键条款：

- 触发州级义务的特定能力阈值。
- AI实验室员工的举报人保护。
- 灾难性事故的事件报告要求。

如果签署，这将是美国第一个州级灾难性风险法规。无论签署状态如何，该法案的框架将影响其他州立法机构如何处理这一问题。加利福尼亚州的从业者应跟踪法案状态；其他地方的从业者应阅读它以了解美国州级法规可能的样子。

### 社会规模风险不是单一层次的问题

第15阶段的贯穿主题——纵深防御——也适用于社会层面。没有单一组织、法规或框架能消除灾难性风险。生态系统只有在以下情况下才能运转：

- 实验室发布扩展策略（第19、20课）。
- 外部评估者提供测量（第21课）。
- 民间社会跟踪并宣传（CAIS）。
- 政府运行自愿项目和基线法规（CAISI, SB-53）。
- 从业者构建多层次控制措施（第10–18课）。

这是该阶段的最终综合：之前的每一课都是堆栈中的一层，其完整性比任何单层强度都更重要。

## 使用它

`code/main.py` 实现了一个小型风险清单工具。给定一个拟议的部署，它会根据四个风险类别标记该部署并返回缓解清单。它是该框架的阅读辅助工具，不能替代人类判断。

## 发布

`outputs/skill-societal-risk-review.md` 审查部署的社会规模风险态势：涉及四个类别中的哪些，已采取哪些缓解措施，组织风险暴露程度如何。

## 练习

1. 运行 `code/main.py`。输入三个不同规模的人工部署。确认四风险标签与您的预期相符；找出一个工具标记不足或过度的案例。

2. 全面阅读CAIS的四风险论文。选择一个风险类别，写两段文字说明你认为该类别中最重要的2026年发展是什么。

3. 阅读加州SB-53草案的当前版本。找出你认为能够增强灾难风险态势的一项条款，以及你认为削弱该态势的一项条款。并证明你的判断。

4. 选择一个你熟悉的AI生产部署（你自己的或已发布的）。根据组织风险子杠杆对其进行评分：安全文化、审计严谨性、多层防御、信息安全。哪个最薄弱？使其达到标准需要多少成本？

5. 草拟2028年版的四风险框架，反映额外一年的能力和额外一年的部署经验。你会添加、删除或重新组织什么？
