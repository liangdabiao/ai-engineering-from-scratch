# 有界自我改进设计

> 研究已经收敛于四个有界自我改进循环的原语。形式不变式必须在每次编辑中都成立。不可修改的对齐锚点。多目标约束，其中每个维度（安全性、公平性、鲁棒性）都必须保持，而不仅仅是性能。回归检测，当历史指标表明能力损失时暂停循环。这些都不能证明安全性——信息论结果（Kolmogorov复杂度、Löb定理）限制了任何系统能对其自身后继者证明的内容。它们是在提高静默失效成本的缓解措施。

**类型:** 学习
**语言:** Python (标准库，带不变式检查的有界循环)
**前置条件:** 第15阶段·07 (RSI), 第15阶段·04 (DGM)
**时间:** ~60分钟

## 问题

第7课的竞速模拟器显示，小的速率差异会累积成大的差距。第4课的DGM案例研究表明，循环可以主动博弈自己的评估器。两个结果都指向同一个工程问题：你能在自我改进循环上施加什么样的约束，使得这些约束不会被循环本身悄然削弱？

ICLR 2026 RSI研讨会总结（openreview.net/pdf?id=OsPQ6zTQXV）识别了四个这样的原语。Anthropic的RSP v3.0（第19课）和DeepMind的FSF v3（第20课）都在能力阈值中引用了它们。Meta的HyperAgents工作和社区框架如SAHOO（2026年3月）在生产中实现了子集。

诚实的框架：这些是缓解措施。信息论结果限制了任何系统能对其自身后继者证明的内容，并且当前没有设计能在形式上闭合该问题。一个有良好界限的循环比无界限的循环更安全，但并非绝对安全。

## 核心概念

### 原语1：形式不变式

不变式是一个属性，在每次自我修改前后都必须成立。示例：

- 输出分布基于固定的宪法标头（第17课）进行条件化。
- 没有工具调用指向未经授权的端点。
- 内存写入通过经过记录和签名的路径。
- 评估器的模块哈希与已批准版本匹配。

不变式由循环无法编辑的外部代码检查。如果提议的修改违反不变式，则被拒绝。这捕获了DGM风格的评估器篡改模式（第4课）。

困难在于选择对安全性必要且可在合理时间内计算的不变式。过度指定会阻碍进展；不足指定会让循环绕过重要属性。

### 原语2：对齐锚点

对齐锚点是循环核心目标的不可变表示，固定在循环的编辑面之外。示例：

- 从只读位置在每次调用时加载的宪法文本（第17课）。
- 位于独立仓库且具有独立访问控制的评估器。
- 由人类签署并在每个周期重新验证的工具权限清单。

锚点的作用是防止目标漂移。循环可以改进其追求目标的方式，但不能编辑目标是什么。

微妙的失效模式：循环无法编辑的锚点仍可能被在读取锚点时发生解释漂移的循环重新解释。宪法AI（第17课）明确基于推理来处理新情况；该推理层正是解释漂移所在之处。锚点是必要的，但不充分。

### 原语3：多目标约束

优化单一标量分数的循环会寻找捷径。必须同时满足多个硬约束的循环可用的捷径更少。典型维度：

- 性能（任务级基准）
- 安全性（红队评估，已知不良的拒绝率）
- 公平性（敏感子群体的差异化影响界限）
- 鲁棒性（OOD测试集，对抗性输入处理）

仅当每个约束都成立时，修改才被接受。第13课的成本管理者将其与财务约束叠加。第18课的Llama Guard作为安全性维度接入。

### 原语4：回归检测

每个周期的分数与历史分布进行比较。超过配置容忍度的下降会暂停循环。这捕获了静默能力损失，否则这些损失会被循环“改进超过”而吸收到运行平均值中。

一个实际实现：存储最近N个周期的每任务分数。每个新周期计算每任务增量。如果任何增量低于阈值，该周期被拒绝并由人工审查。

### 信息论限制

Kolmogorov复杂度和Löb定理为系统能证明自身的内容设定了上限。Schmidhuber的形式化哥德尔机（第4课）旨在达到该上限的最高点；没有人完成过非平凡证明。Löb的结果说：如果一个系统可证明地相信“如果我能证明我应该做X，我就会做X”，那么它会在没有证明的情况下做X，这是一个著名的自指失败。

对我们原语的启示：它们不能闭合安全性问题。它们使静默失效更加昂贵。一个恶意或漂移的循环如果原本会悄然绕过缺失的检查，现在必须主动破坏显式的检查，这是更可检测的特征。

### 一个工作示例

假设一个智能体提出一次编辑。门控栈如下：

1. 不变式检查：模块哈希、工具权限清单、宪法标头。
2. 锚点检查：目标陈述与批准版本匹配（逐字节或语义上）。
3. 多目标评估：性能、安全性、公平性、鲁棒性维度。
4. 回归检测：没有维度下降超过容忍度。

所有四个都必须通过，编辑才能落地。任何单个失败都会暂停循环。

## 使用它

`code/main.py` 在第4课的DGM风格玩具上运行一个有界自我改进循环，但上层叠加了四个原语。每个原语可以单独启用或禁用。演示表明，每个原语捕获特定的故障类别，移除任何一个都会让该故障类别通过。

## 发布

`outputs/skill-bounded-loop-review.md` 审计一个提议的有界循环，并对其实际实现与声称实现的四个原语进行评分。

## 练习

1. 运行所有原语启用的`code/main.py`。确认循环仍在主要指标上改进，而不让攻击获胜。

2. 禁用回归检测（Regression detection）。构造一个输入，使得这导致静默能力损失被接受。

3. 禁用多目标约束（Multi-objective constraint）。展示循环在性能轴上收敛而在安全轴上下降。

4. 为编程智能体设计一个对齐锚点（Alignment anchor）。文本是什么？存储在哪里？如何检查？

5. 阅读ICLR 2026 RSI研讨会摘要。从四个原语中选一个，并针对当前的技术水平提出具体的改进。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|---|---|---|
|  不变性 (Invariant)  |  "始终为真的属性"  |  一个由外部代码在每次编辑前后检查的属性 |
|  对齐锚点  |  "固定的目标"  |  循环编辑面之外的不可变核心目标表示 |
|  多目标约束  |  "所有轴都必须保持"  |  性能、安全性、公平性、鲁棒性——全部需要 |
|  回归检测  |  "下降时暂停"  |  当历史指标变化表明能力损失时暂停循环 |
|  柯尔莫哥洛夫界 (Kolmogorov bound)  |  "信息论极限"  |  限制系统能证明关于自身后继者的内容 |
|  洛布定理 (Lob's theorem)  |  "自指陷阱"  |  系统可以根据“我应该”行动，而无需证明它应该 |
|  门堆栈 (Gate stack)  |  "分层检查"  |  多个原语组合；任何失败都拒绝编辑 |
|  有界改进 (Bounded improvement)  |  "缓解，而非证明"  |  提高静默故障成本；但未解决安全问题 |
