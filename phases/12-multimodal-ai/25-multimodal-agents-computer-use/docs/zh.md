# 多模态代理与计算机使用（顶点项目）

> 2026年的前沿产品是一个多模态代理(Multimodal Agent)，它能够读取截图、点击按钮、导航网页界面、填写表单并端到端完成工作流。SeeClick与CogAgent（2024）证明了GUI基础(GUI-grounding)原语。Ferret-UI增加了对移动端的支持。ChartAgent引入了图表视觉工具使用。VisualWebArena与AgentVista（2026）是前沿追逐的基准——即便是Gemini 3 Pro和Claude Opus 4.7在AgentVista的困难任务上也仅得分约30%。这个顶点项目整合了第12阶段的所有线索：感知（高分辨率视觉语言模型），推理（带工具使用的大语言模型），基础（坐标输出），长时序记忆和评估。

**类型：** 顶点项目
**语言：** Python（标准库，动作模式+代理循环骨架）
**先决条件：** 第12阶段·05（LLaVA），第12阶段·09（Qwen-VL JSON），第14阶段（代理工程）
**时间：** ~240分钟

## 学习目标

- 设计一个多模态代理循环：感知→推理→行动→观察→重复。
- 构建一个GUI基础输出模式（点击坐标、输入文本、滚动、拖拽），视觉语言模型可以将其输出为JSON。
- 比较纯截图代理 vs 无障碍树代理 vs 混合代理。
- 在一个小型VisualWebArena切片上设置多模态代理基准评估。

## 问题

一个订票网站工作流：“帮我找一趟4月15日飞往东京的航班，靠过道座位，价格低于800美元，并预订。”

一个多模态代理需要：

1. 截取浏览器截图。
2. 将截图+URL+目标解析为计划。
3. 发出结构化动作：点击（在x,y处），输入“Tokyo”（在元素E处），向下滚动，选择（单选按钮）。
4. 将动作应用于浏览器。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是一次多模态视觉语言模型调用。视觉语言模型的输出必须是可解析的JSON。错误会在各步中累积，因此恢复能力很重要。

## 核心概念

### GUI基础——基本概念

GUI基础是：给定一张截图和一条自然语言指令，输出要点击的（x，y）坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是大规模首个开放成果：在合成+真实GUI数据上微调视觉语言模型，将坐标输出为纯文本标记。可行。

CogAgent（arXiv:2312.08914）增加了1120x1120高分辨率编码，用于密集界面。得分：网页导航约84%。

Ferret-UI（arXiv:2404.05719）专注于移动界面，集成了iOS无障碍数据。

输出格式通常是JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc`有助于恢复：如果坐标在截图之间漂移，语义提示让系统重新定位基础。

### 动作模式

一个典型的动作模式有6-10种动作类型：

- `click`: (x, y)
- `click`: (text, x?, y?)
- `click`: (direction, amount)
- `click`: (x0, y0, x1, y1)
- `click`: (option_index)
- `click`: (x, y)
- `click`: (url)
- `click`: (ms)
- `click`: (success, explanation)

代理每步发出一个动作。浏览器包装器执行并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，无结构化信息。最通用；适用于任何应用。
- 无障碍树：结构化DOM / iOS无障碍信息。用于基础时更可靠；在树可用时有效。
- 混合模式：两者兼用，树作为原子动作的可靠基础，截图用于语义上下文。

生产环境中的代理尽可能使用混合模式。浏览器自动化（Selenium+无障碍）始终拥有树；桌面应用有时也有。

### 长时序记忆

一个20步工作流生成20张截图。视觉语言模型的上下文很快被填满。三种压缩策略：

- 摘要链：每5步后总结已发生情况，丢弃旧截图。
- 跳帧：保留第一张、最后一张和每隔3张的截图。
- 工具记录日志：执行动作，保留所做内容的文本日志；不再查看旧截图。

Claude的计算机使用API使用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）引入了用于图表理解的视觉工具使用：裁剪、缩放、OCR、调用外部检测。代理可以输出“裁剪到区域(100, 200, 300, 400)然后调用OCR”作为工具调用。工具返回文本；视觉语言模型继续推理。

这种模式可以推广：标记集提示、区域注释和外部检测工具都适合相同的“输出工具调用，接收结构化响应”模式。

### 2026 年基准测试

- ScreenSpot-Pro。基于约 1k 网页截图的 GUI定位。开源 SOTA Qwen2.5-VL-72B 约85%。前沿模型约90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类广告）。开源 SOTA 约20%。Gemini 3 Pro 约27%。
- AgentVista (arXiv:2602.23166)。2026年最难的基准测试。横跨12个领域的现实工作流。前沿模型得分27-40%；开源模型10-20%。
- WebArena / WebShop。较旧的基准测试；已被前沿模型饱和。

### 为什么仍然困难

智能体性能瓶颈：

1. 精细尺度的视觉定位。“点击小X”在移动分辨率下经常失败。
2. 长程规划。10个动作后，智能体会偏离目标。
3. 错误恢复。当点击失败（按错按钮）时，检测并恢复的训练数据很少。
4. 跨页面上下文。在标签页或长表单之间跳转会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（截图匹配以检查动作成功）。

### 结业构建项目

结业任务：构建一个计算机使用智能体，能够：

1. 读取预订网站模拟页面的HTML和截图。
2. 规划多步序列：搜索 → 选择 → 填写表单 → 提交。
3. 输出符合动作模式的JSON动作。
4. 在固定的10个任务片段上进行评估。

本课程提供脚手架代码，易于扩展为真实的浏览器。

## 使用它

`code/main.py` 是结业构建项目的脚手架：

- 动作模式JSON定义（10个动作）。
- 以字典形式呈现的模拟浏览器状态。
- 智能体循环骨架：接收状态、输出动作、执行、循环。
- 10个任务的迷你基准测试（合成页面）以衡量端到端成功率。
- 当动作失败时的错误恢复钩子。

## 发布

本课程产出 `outputs/skill-multimodal-agent-designer.md`。针对某个计算机使用产品（领域、动作集、评估目标），设计完整的智能体循环、记忆策略、定位模式和预期基准测试得分。

## 练习

1. 用 `screenshot_region` 工具（裁剪+缩放）扩展动作模式。哪些任务受益？

2. 阅读AgentVista (arXiv:2602.23166)。描述最难的任务类别以及为什么前沿模型仍然失败。

3. 长程记忆压缩：设计一个摘要链，最多保留4张活跃截图，任意数量记录。

4. 构建错误恢复钩子：当动作失败（未找到按钮）时，智能体下一步该做什么？

5. 比较仅截图模式的Claude 4.7与混合截图+无障碍树的Qwen2.5-VL在10个网页任务上的表现。哪些任务上谁更胜一筹？

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  GUI定位  |  “点击坐标”  |  模型针对截图中的指令输出目标位置的 (x,y)  |
|  动作模式  |  “工具定义”  |  有效动作（点击、输入、滚动、拖拽）的JSON描述  |
|  无障碍树  |  “结构化DOM”  |  来自浏览器/iOS API的机器可读UI层级结构  |
|  混合智能体  |  “截图+树”  |  同时使用图像和结构化信息；比单独使用任一方式更可靠  |
|  视觉工具使用  |  “缩放/裁剪/检测”  |  智能体在规划中途调用外部视觉工具（OCR、检测）  |
|  摘要链  |  “记忆压缩”  |  周期性文本摘要替代长截图历史  |
|  VisualWebArena  |  “端到端网页基准”  |  2024年端到端网页任务基准测试  |
|  AgentVista  |  “2026年困难基准”  |  12个领域现实工作流；即使是Gemini 3 Pro也只得分约30%  |
