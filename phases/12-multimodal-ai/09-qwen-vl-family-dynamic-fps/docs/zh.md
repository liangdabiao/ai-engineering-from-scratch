# Qwen-VL 家族与动态帧率视频

> Qwen-VL 家族——Qwen-VL (2023)、Qwen2-VL (2024)、Qwen2.5-VL (2025)、Qwen3-VL (2025)——是2026年最具影响力的开源视觉语言模型系列。每一代都做出了一个决定性的架构决策，而整个开源生态系统在十二个月内就进行了复制：通过 M-RoPE 实现原生动态分辨率、带有绝对时间对齐的动态帧率采样、ViT 中的窗口注意力机制，以及结构化的智能体输出格式。到 Qwen3-VL 时，这一配方已经稳定：一个采用原生宽高比输入的 2D-RoPE-ViT 编码器、一个将视觉特征映射到大语言模型 Qwen3 的 MLP 投影器，以及将 OCR、接地和智能体行为作为一等目标的训练阶段。本课按时间顺序解读该系列，让你理解每个参数为何如此设置。

**类型：** 学习
**语言：** Python (stdlib, M-RoPE 编码器 + 动态帧率采样器)
**前置要求：** 第12阶段·第06课（补丁与打包）
**时间：** 约120分钟

## 学习目标

- 计算 M-RoPE 的三轴旋转（时间、高度、宽度），并解释为何三者缺一不可。
- 为视频选择一种动态帧率采样策略，并推理每秒令牌数与事件检测准确率之间的权衡。
- 按顺序命名 Qwen-VL 的四次代际升级，并说明每次升级带来了什么。
- 编写一个 Qwen2.5-VL 风格的 JSON 智能体输出格式，并从 VLM 响应中解析结构化的工具调用。

## 问题

Qwen-VL 于 2023 年 8 月发布，直接回应了 LLaVA-1.5 和 BLIP-2。Qwen 团队瞄准的差距有三方面：分辨率、视频和结构化输出。

分辨率：LLaVA-1.5 运行在 336x336 分辨率。对于照片来说还行，但对于中文发票或密集的电子表格截图来说毫无用处。Qwen-VL 的第一个创新是 448x448 分辨率以及接地边界框输出，让模型能够指向物体。

视频：Video-LLaMA 将逐帧编码器堆叠起来并馈送到大语言模型。它适用于短视频片段，但不适用于时间轴本身就是信号的多分钟视频。Qwen 团队希望有一个能理解时间的单一编码器。

结构化输出：LLaVA 输出自由形式的文本。智能体需要 JSON。Qwen-VL 在明确的 JSON 输出格式（包括作为文本的边界框坐标）上进行了训练。

Qwen-VL 的每一代都在扩展这三个轴之一。

## 核心概念

### Qwen-VL（2023年8月）

第一代：OpenCLIP ViT-bigG/14 作为编码器（2.5B 参数）、兼容 LLama 的 Q-Former（1 步，256 个查询）、Qwen-7B 基础模型。贡献：

- 448x448 分辨率（当时开源 VLM 的 SOTA）。
- 接地：在带有显式坐标令牌输出的图像-文本对上训练。"猫在 <box>(112, 204), (280, 344)</box>"。
- 从一开始就进行中英文多语言训练。

当时的基准测试：在英文上与 GPT-4V 竞争，在中文上占主导地位。接地监督才是真正的亮点。

### Qwen2-VL（2024年9月）——M-RoPE 和原生分辨率

Qwen2-VL 用原生动态分辨率的 ViT 编码器取代了固定分辨率加 Q-Former 的架构。关键变化：

- 原生动态分辨率。ViT 接受任何能被 28 整除的 H×W（patch 为 14，2x 空间合并）。1120x672（40x24 合并 patches）的图像产生 960 个视觉令牌。无需缩放、无需分块、无需缩略图。
- M-RoPE（多模态旋转位置编码）。每个令牌携带一个 3D 位置 (t, h, w) 而非 1D。对于图像 t=0，对于视频 t=帧索引。RoPE 按每个轴的频率旋转查询/键向量。无位置编码表。
- MLP 投影器。放弃 Q-Former；在合并的 patch 令牌上使用 2 层 MLP。
- 带动态帧率的视频。默认以 1-2 FPS 采样视频，但模型接受任意帧数。

结果：Qwen2-VL-7B 在多个多模态基准测试上与 GPT-4o 持平，并在 DocVQA 上击败了它（94.5 vs 88.4）。架构变化是决定性的举措。

### Qwen2.5-VL（2025年2月）——动态帧率 + 绝对时间

Qwen2.5-VL 的重大变化是视频。动态帧率不仅仅是“在需要时采样更多帧”。论文将其形式化为：

- 绝对时间令牌。使用实际时间戳代替位置索引（帧 0、1、2...）。"在 0:04，猫跳了起来。"模型看到 `<time>0.04</time>` 令牌与帧令牌交错。
- 动态帧率。对于慢速镜头以 1 FPS 采样，对于动作场景以 4+ FPS 采样。用户或训练器选择；M-RoPE 自适应。
- ViT 中的窗口注意力。空间注意力是窗口化的（在块内局部）以提高吞吐量；每几层使用全局注意力。
- 明确的 JSON 输出格式。在工具调用数据上训练："{\"tool\": \"click\", \"coords\": [380, 220]}"。开箱即用，适用于智能体。
- MRoPE-v2 缩放。位置根据最大输入大小缩放，因此 10 分钟的视频不会超出频率范围。

基准测试：Qwen2.5-VL-72B 在大多数视频基准测试上击败了 GPT-4o，在文档上与 Gemini 2.0 持平，并在 GUI 接地（ScreenSpot：准确率 84% vs GPT-4o 的 38%）上创下了开源模型的 SOTA。

### Qwen3-VL（2025年11月）

Qwen3-VL 是一个增量升级，它巩固而非重新发明：更大的 LLM 骨干（Qwen3-72B）、扩展的训练数据、改进的 OCR、通过 Qwen3“思考模式”增强的推理能力。ViT 和 M-RoPE 保持不变。论文侧重于数据和训练改进，而非架构。

该系列的经验教训：到 2025 年，Qwen-VL 架构已经稳定。后续世代扩展的是计算和数据，而非基本要素。

### M-RoPE 的数学原理

经典的 RoPE 通过配对坐标将查询 `q`（维度为 `d`）按位置 `m` 旋转：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE 将隐藏维度拆分为三个频带。假设 `d = 96`。将 32 维分配给时间，32 维分配给高度，32 维分配给宽度。每个频带按自己的轴位置旋转。位于 (t=5, h=10, w=20) 的 patch 获得分别应用于其三个频带的旋转 `R_t(5)`、`R_h(10)`、`R_w(20)`。

文本令牌使用 `t = text_index, h = 0, w = 0`（或归一化选择），保持兼容性。视频帧使用 `t = frame_time, h = row, w = col`。单张图像使用 `t = 0`。

好处：一个位置编码即可处理文本、图像和视频，无需分支代码或不同的位置表。

### 动态帧率采样逻辑

给定一个持续时间为 `T` 秒的视频和一个目标Token预算 `T`：

1. 计算你能负担的最大FPS：`fps_max = B / (T * tokens_per_frame)`。
2. 从满足 `fps_max = B / (T * tokens_per_frame)` 的FPS中选取一个目标FPS。
3. 如果运动剧烈（通过光流启发式或用户明确请求），选择更高的FPS。如果运动平缓，选择较低的FPS。
4. 在所选FPS下均匀采样；在帧之间插入 `fps_max = B / (T * tokens_per_frame)` 个Token。

Qwen2.5-VL隐含地训练了这一逻辑；在推理时用户通过 `fps` 参数控制。一个60秒的动作序列，以4FPS采样，每帧81个Token，总共19440个Token，在32k上下文中可以管理。

### 结构化智能体输出

Qwen2.5-VL的智能体训练明确针对结构化工具调用：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析是确定性的：对模型输出执行 JSON.parse。相比之下，自由形式的“点击(1024, 512)”需要正则表达式并处理歧义。这一转变是Qwen2.5-VL的ScreenSpot得分从Qwen2-VL的55%跃升至84%的原因。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  M-RoPE  |  "多模态RoPE"  |  3D旋转位置嵌入，在隐藏维度中包含时间、高度和宽度频带  |
|  动态FPS  |  "智能采样"  |  根据视频的运动、时长和Token预算选择的帧采样率  |
|  绝对时间Token  |  "时间戳Token"  |  `<time>t</time>` 插入序列中，使模型看到实际秒数而非帧索引  |
|  窗口注意力  |  "局部注意力"  |  空间自注意力限制在小窗口内以提高速度；周期性添加全局注意力  |
|  结构化智能体输出  |  "JSON模式"  |  训练数据监督，教导VLM输出带有坐标和工具名称的可解析JSON  |
|  min_pixels / max_pixels  |  "分辨率边界"  |  每请求的Qwen2.5-VL控制总像素数，从而控制Token数  |
|  定位  |  "指向它"  |  输出边界框坐标作为文本Token；自Qwen-VL v1以来使用  |
