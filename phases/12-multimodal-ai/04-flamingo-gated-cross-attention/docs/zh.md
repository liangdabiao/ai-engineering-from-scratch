# Flamingo与门控交叉注意力在少样本VLM中的应用

> DeepMind的Flamingo（2022）率先完成了两项任务：它展示了单一模型可以处理任意交错排列的图像、视频和文本序列；同时证明了VLM能够进行上下文学习——通过三个示例（图像、标题）对进行少样本提示，模型无需任何梯度步骤即可对新图像进行描述。其机制是：在冻结的LLM现有层之间插入门控交叉注意力层，并采用初始值为零的可学习tanh门控，以确保初始化时LLM的文本能力得以保留。本课将解析Flamingo的Perceiver重采样器和门控交叉注意力架构——它是Gemini交错输入和Idefics2视觉标记的前身。

**类型：** 学习
**语言：** Python（标准库，门控交叉注意力+Perceiver重采样器演示）
**先修知识：** 阶段12 · 03（BLIP-2 Q-Former）
**时间：** 约120分钟

## 学习目标

- 解释门控交叉注意力如何通过tanh(门控)=0在初始化时保留冻结LLM的文本能力。
- 逐步解析Perceiver重采样器：N个图像块 → 通过交叉注意力生成K个固定的"潜在"查询。
- 描述Flamingo如何处理交错图像-文本序列，并使用尊重图像位置的因果掩码。
- 复现一个少样本多模态提示结构（3个图像-标题示例，然后一个查询图像）。

## 问题

BLIP-2向冻结LLM的输入层馈送32个视觉标记。每个提示仅适用于一张图像。但如果你想处理*多张*图像与文本交错，例如"这里是图像A，为其添加标题；这里是图像B，为其添加标题；现在这里是图像C，为其添加标题"？LLM的自注意力需要在单个流中处理图像标记和文本标记，哪些位置可以关注哪些图像的问题变得复杂。

Flamingo的解决方案：完全不改变LLM的输入流。在现有LLM块之间插入额外的交叉注意力层。文本标记仍像往常一样通过LLM的因果自注意力。每隔几个LLM块，文本标记还会通过新的门控层交叉关注图像特征。门控（初始化为零）意味着在零步时，新层是空操作——模型的行为与预训练LLM完全相同。随着训练的进行，门控打开，视觉信息开始流动。

Flamingo解决的第二个问题：如何处理每个提示中可变数量的图像（0、1或多张）？Perceiver重采样器——一个小的交叉注意力模块，它接收任意数量的图像块并生成固定数量的视觉潜在标记。无论提示中有多少图像，LLM交叉注意力层看到的形状都是相同的。

## 核心概念

### 冻结的LLM

Flamingo以冻结的Chinchilla 70B LLM开始。所有700亿参数保持不变。现有的文本自注意力和前馈网络正常运行。

### Perceiver重采样器

对于提示中的每张图像，ViT生成N个块标记。Perceiver重采样器有K个固定的可学习潜在向量（Flamingo使用K=64）。每个重采样器块包含两个子步骤：

1. 交叉注意力：K个潜在向量关注N个块标记（查询来自潜在向量，键/值来自块）。
2. 潜在向量内部的自注意力和前馈网络。

经过6个重采样器块后，输出为K=64个维度为1024的视觉标记，无论ViT生成了多少个块。一张224x224的图像（196个块）和一张480x480的图像（900个块）都输出为64个重采样器标记。

对于视频，重采样器在时间上应用：每帧的块产生64个潜在向量，时间位置编码让模型区分t=0和t=N。整个视频变为T * 64个视觉标记。

### 门控交叉注意力

在冻结LLM的每M层之间（Flamingo使用M=4），插入一个新的门控交叉注意力块：

```
x_after_llm_block = llm_block(x_before)
cross = cross_attn(x_after, resampler_output)
gated = tanh(alpha) * cross + x_after
x_before_next_block = gated
```

- `alpha`是一个可学习的标量，初始化为零。
- `alpha`，因此在初始化时，门控分支贡献为零。
- 随着`alpha`远离零，交叉注意力的贡献平滑增长。
- 残差连接意味着即使门控完全打开，也不会覆盖LLM的文本表示；它只是在顶层添加视觉信息。

这是Flamingo最重要的设计选择：视觉条件是加性的、门控的，且在初始化时为零。在文本输入上，步骤0的Flamingo是一个完美的Chinchilla 70B。

### 用于交错输入的有掩码交叉注意力

在像"<图像A> 标题A <图像B> 标题B <图像C> ?"这样的提示中，每个文本标记只能看到序列中位于其之前的图像。交叉注意力掩码要求：位置`t`处的文本标记仅关注图像重采样器标记，其中图像索引`i < i_t`满足`i_t`是位置`t`之前最近的图像。"仅看到前一张图像"或"看到所有之前的图像"都是有效选择；Flamingo选择了前者。

### 上下文少样本学习

一个Flamingo提示看起来像：

```
<image1> A photo of a cat. <image2> A photo of a dog. <image3> A photo of a
```

模型看到完成模式并输出"鸟"（或image3显示的任何内容）。无梯度步骤。冻结LLM的上下文学习能力通过门控交叉注意力传递——这是该论文的要点及其重要性所在。

### 训练数据

Flamingo在三个数据集上训练：

1. MultiModal MassiveWeb (M3W)：4300万张带有交错图像和文本的网页，重建阅读顺序。
2. 图像-文本对 (ALIGN + LTIP)：44亿对。
3. 视频-文本对 (VTP)：2700万个短视频片段。

OBELICS（2023）是交错网页语料库的开放复现版本，Idefics、Idefics2以及大多数开放的"类Flamingo"模型都在其上训练。

### OpenFlamingo和Otter

OpenFlamingo（2023）是开放复现版本。架构相同（Perceiver重采样器+冻结LLaMA或MPT上的门控交叉注意力）。提供3B、4B、9B检查点。由于基础LLM较小且数据较少，质量落后于Flamingo。

Otter（2023）基于OpenFlamingo，在MIMIC-IT（多模态指令数据集）上进行指令微调，展示了门控交叉注意力也适用于指令跟随。

### 后代们

- Idefics / Idefics2 / Idefics3: Hugging Face的门控交叉注意力谱系，逐步简化（Idefics2放弃了重采样器，转而使用带有自适应池化的直接补丁令牌）。
- Flamingo到Chameleon的过渡：到2024年，许多团队转向早期融合（第12.11课）；在需要冻结骨干网络的生产环境中，Flamingo风格的门控交叉注意力仍然在使用。
- Gemini的交错输入：概念上继承了Flamingo交错格式的灵活性，但具体机制是专有的。

### 与BLIP-2的比较

|   |  BLIP-2  |  Flamingo  |
|---|---|---|
|  视觉桥  |  Q-Former仅在输入处  |  每M层间的门控交叉注意力  |
|  视觉令牌数  |  每图32个  |  每个交叉注意力层每图64个  |
|  冻结的LLM  |  是  |  是  |
|  上下文少样本  |  弱  |  强——论文的核心  |
|  交错输入  |  无原生支持  |  是，设计目标  |
|  训练数据  |  1.3亿对  |  13亿对 + 4300万交错页面  |
|  参数数量  |  训练1.88亿  |  训练约100亿（交叉注意力层）  |
|  计算资源  |  在8块A100上数天  |  在数千TPUv4上数周  |

选择BLIP-2用于预算有限的单图像VQA。选择Flamingo/Idefics2用于交错、少样本或多图像推理。

## 使用它

`code/main.py` 演示了：

1. 一个在36个虚假补丁令牌上使用8个可学习潜在变量的Perceiver重采样器（纯Python交叉注意力）。
2. 一个使用`alpha = 0`的门控交叉注意力步骤→输出等于输入（LLM不变），然后`alpha = 2.0`→视觉贡献被混合进来。
3. 一个交错掩码构建器，为"(图像1) (文本1) (图像2) (文本2)"序列生成2D注意力掩码。

## 练习

1. 计算Flamingo-9B的视觉参数数量：90亿LLM + 14亿门控交叉注意力层 + 6400万重采样器。训练的参数占总参数的多少比例？

2. 在PyTorch中实现门控残差`y = tanh(alpha) * cross + x`。通过实验表明，在初始化时，使用`alpha=0`，`y==x`精确等于。

3. 阅读OpenFlamingo第3.2节（arXiv:2308.01390），了解当每个提示具有不同图像数量时，他们如何处理批次中的多个图像。描述填充策略。

4. 为什么Flamingo的交叉注意力掩码让文本令牌只关注*最近的*前一个图像，而不是所有前面的图像？阅读Flamingo论文第2.4节并解释权衡。

5. 上下文少样本：为一个新的Flamingo变体构建一个包含4个"图像→主要物体颜色"示例的提示。描述当示例数量从0变化到8时预期的准确率模式。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  Perceiver重采样器  |  "固定潜在变量交叉注意力"  |  从可变数量的输入补丁生成K个固定令牌的模块  |
|  门控交叉注意力  |  "Tanh门控桥"  |  残差层`y = tanh(alpha)*cross + x`，可学习的alpha，初始化为0  |
|  交错输入  |  "混合序列"  |  图像和文本按阅读顺序自由混合的提示格式  |
|  冻结的LLM  |  "无LLM梯度"  |  文本LLM的权重不更新；仅重采样器+交叉注意力层训练  |
|  少样本  |  "上下文示例"  |  在提示中给出几个(图像, 答案)对；模型无需微调即可泛化  |
|  OBELICS  |  "交错网络语料库"  |  包含1.41亿网页的开放数据集，图像和文本按阅读顺序排列  |
|  Chinchilla  |  "700亿冻结基础"  |  Flamingo的冻结文本LLM，来自DeepMind的Chinchilla论文  |
|  门控调度  |  "alpha如何移动"  |  训练期间交叉注意力门打开的速度  |
|  交叉注意力频率  |  "每M层"  |  门控交叉注意力块插入的频率；Flamingo使用M=4  |
|  OpenFlamingo  |  "开放复现"  |  MosaicML/LAION 在 3-9B 规模的开放检查点；架构与 Flamingo 相同 |
