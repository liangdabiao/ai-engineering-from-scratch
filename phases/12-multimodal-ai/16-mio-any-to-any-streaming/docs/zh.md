# MIO和任意到任意流式多模态模型

> GPT-4o推出的一个产品是大多数开放模型无法复制的：一个能够听到语音、看到视频并实时回应的代理。截至2024年底，开放生态系统的答案是MIO（Wang等人，2024年9月）。MIO将文本、图像、语音和音乐进行令牌化(Tokenize)，在一个因果变换器上训练交织序列，并生成任意模态到任意模态的输出。AnyGPT（Zhan等人，2024年2月）是概念验证；MIO是规模化版本；Unified-IO 2（Allen AI，2023年12月）是其增加了视觉和动作基础的近亲。本课解读任意到任意模式——四个令牌化器(Tokenizer)，一个变换器(Transformer)，支持流式解码。

**类型：** 学习
**语言：** Python（标准库，四模态令牌分配器 + 流式解码循环）
**前置知识：** 阶段12 · 11（Chameleon），阶段6（语音和音频）
**时间：** ~120分钟

## 学习目标

- 设计一个共享词汇表，容纳文本、图像、语音和音乐令牌而不发生冲突。
- 比较SEED-Tokenizer（图像）和SpeechTokenizer残差VQ（语音）在压缩与重构之间的权衡。
- 解释用于构建任意到任意生成的四阶段课程。
- 列举三种开放的任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题

统一多模态模型说起来容易，但规模化构建却很难。2024年之前，大多数“任意到任意”系统都是流水线式的：视觉模型→文本表示→语音模型→音频。每一步都会丢失信息、增加延迟并使训练复杂化。GPT-4o的演示视频展示了一种单模型替代方案，响应时间低于1秒；而开放生态系统落后了数月。

工程挑战包括：

- 每个模态都必须有令牌化器，能够足够无损地压缩以便重构，并以变换器可以消耗的速率生成令牌。
- 单一词汇表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。至少需要四万个以上的条目。
- 训练数据必须覆盖每一种输入-输出对（文本→图像、图像→语音、语音→图像等），否则模型必须能够组合。
- 推理必须足够快地流式输出令牌，以满足对话延迟（首次音频字节时间小于500毫秒）。

## 核心概念

### 四种模态的四个令牌化器

MIO的令牌化器栈：

- 文本：标准BPE，词汇表约32000。
- 图像：SEED-Tokenizer（2023年）——离散码本的量化VAE，4096个条目，每张图像32x32个令牌。
- 语音：SpeechTokenizer残差VQ（2023年）——将16kHz波形编码为8个分层码本；第一层为粗略内容，后续层添加韵律和说话人身份。
- 音乐：类似的残差VQ（Meta的MusicGen / Encodec家族），4-8个码本。

每种模态产生整数令牌。这些令牌在共享词汇表中获得不相交的ID范围：

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

总计：约48k词汇表。输入嵌入和输出投影覆盖所有这些范围。

### 流式解码

语音生成使用残差VQ。变换器预测基础（第0层）语音令牌；一个并行解码的残差量化器预测后续层。每个第0层令牌大约对应16kHz下的50ms音频。

流式模式如下：

1. 用户对麦克风说话；实时音频令牌化器每50ms发出语音令牌。
2. MIO在令牌到达时消费它们（快速预填充 + 增量前向）。
3. 输出令牌在生成时流式输出；一个并行语音解码器将它们转换为音频样本，延迟约50-150ms。
4. 首次音频字节时间：MIO论文中约为300-500ms，接近GPT-4o的约250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和Moshi（arXiv:2410.00037）是互补的流式语音LLM设计。特别是Moshi，在单GPU上实现了160ms的往返延迟。

### 四阶段课程

MIO的训练课程：

1. 阶段1——对齐。大规模的模态对语料库：文本-图像、文本-语音、文本-音乐。每对使用自己的令牌词汇段。训练共享词汇表。
2. 阶段2——交织。多模态交织文档（带图像和视频的博客、带文字记录的播客等）。训练跨模态上下文。
3. 阶段3——语音增强。额外的音频数据，在不损失文本能力的情况下提升语音质量。
4. 阶段4——SFT。跨模态的指令微调：VQA、描述、叙述、语音到语音对话。

缺少一个阶段会降低特定能力：跳过阶段2，模型失去跨模态上下文；跳过阶段3，语音质量差。

### 视觉思维链

MIO引入了视觉思维链：模型发出中间图像令牌作为推理步骤。对于“猫在爬树吗？”这个问题，模型：

1. 发出`<image>`令牌来渲染场景（来自输入图像或草图）。
2. 发出文本分析草图。
3. 发出最终答案。

渲染的中间图像作为草稿纸。在空间推理任务上指标提升。这个想法类似于文本推理的思维链。

### 任意到任意领域的竞争者

- AnyGPT（arXiv:2402.12226）：4种模态（文本、图像、语音、音乐），类似设计。
- Unified-IO 2（arXiv:2312.17172）：增加了视觉动作输出、深度、法线。任务多样性更广，规模较小。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态特定扩散解码器。不是单模型方法。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜在空间实现任意到任意。

MIO是最接近纯令牌任意到任意的方法。AnyGPT是其概念上的前身。

### 延迟预算

对于一个对话产品，每个组件的延迟都很重要：

- 麦克风到音频标记：约50毫秒。
- 预填充（音频标记+历史记录）：8B模型上约100毫秒。
- 第一个输出标记：约50毫秒。
- 并行残差VQ+语音解码器：约100-150毫秒。

总首次音频字节时间：最低约300毫秒。GPT-4o声称约250毫秒。Moshi声称160毫秒。MIO/AnyGPT根据公开基准测试在400-600毫秒范围内。

### 为什么任意到任意仍然困难

即使到2026年，开源任意到任意模型在两个方面仍落后于闭源模型：

- 语音质量。残差VQ分词器是有损的；与ElevenLabs级别的语音相比，对话语音听起来很机械。
- 跨模态推理。要求模型“唱出你所看到的”仍然比纯视觉任务更容易失败。

这些是开放的研究问题。Qwen3-Omni（第12.20课）是2025年最先进的开源尝试。

## 使用它

`code/main.py`：

- 定义四模态词汇表分配并打印。
- 将多模态输入列表（文本、图像、音频片段、音乐）通过分词器路由器进行路由。
- 模拟文本到语音响应的流式解码，并计数延迟。
- 根据编码器、预填充和解码器延迟计算预期的首次音频字节时间。

## 练习

1. 您的产品接受语音输入并返回语音输出。端到端延迟预算目标是什么？列出消耗时间的组件。

2. SpeechTokenizer残差VQ使用8个码本。提出为什么需要并行解码残差级别（相对于顺序解码）以及它带来了多少延迟节省。

3. 您的词汇表有32k文本+4k图像+4k语音。添加8k音乐和约10个分隔符。在隐藏维度4096下，嵌入矩阵的参数成本是多少？

4. 视觉思维链输出一个中间图像。哪种类型的问题受益？哪种类型的问题因额外标记而受损？

5. 阅读Moshi（arXiv:2410.00037）。描述其“内心独白”技术，并与MIO的视觉思维链进行比较。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
| 任意到任意  |  "多模态输入/输出"  |  接受并输出文本、图像、语音和音乐任意方向的单一模型 |
| 残差VQ  |  "语音分词器堆栈"  |  多码本分词，每层增加信息；基础层是内容，后续层是韵律 |
| SEED分词器  |  "图像编码"  |  MIO使用的离散图像分词器，具有4096条目的码本 |
| 视觉思维链  |  "视觉草稿"  |  模型在最终答案前生成中间图像作为推理步骤 |
| 首次音频字节时间  |  "TTFAB"  |  从用户语音到第一个音频输出的延迟；对话体验需<500毫秒 |
| 四阶段课程  |  "训练方案"  |  对齐->交错->语音增强->SFT，按此顺序 |
