# Emu3：面向图像和视频生成的下一词元预测

> 北京智源人工智能研究院(BAAI)的Emu3 (Wang et al., September 2024) 是2024年的一项成果，它本应终结扩散模型与自回归模型之间的争论。一个单一的类Llama解码器专用Transformer，仅使用下一词元预测目标进行训练，在一个统一的词汇表（文本 + VQ图像词元 + 3D VQ视频词元）上，在图像生成方面击败了SDXL，在感知方面击败了LLaVA-1.6。没有CLIP损失。没有扩散调度。推理时使用无分类器指导来提升质量，但核心训练目标是带教师强制(Teacher Forcing)的下一词元预测。发表于《自然》期刊。本课将解读Emu3的论文——为什么更好的词元器(Tokenizer)加上规模就是一切——并与扩散方法进行对比。

**类型：** 学习
**语言：** Python (标准库，3D视频词元器数学 + 自回归采样骨架)
**前置知识：** 阶段12 · 11 (Chameleon)
**时间：** ~120分钟

## 学习目标

- 解释为什么Emu3的单损失下一词元目标有效，尽管长期以来人们认为图像质量需要扩散。
- 描述3D视频词元器：时空VQ码本是什么样的，为什么补丁(Patch)跨越时间维度。
- 比较Emu3与Stable Diffusion XL在（训练计算量、推理成本、质量上限）上的差异。
- 指出同一个Emu3模型扮演的三个角色：Emu3-Gen（图像生成）、Emu3-Chat（感知）、Emu3-Stage2（视频生成）。

## 问题

直到2024年的传统观点：图像生成需要扩散。其论点：离散图像词元丢失太多信息，无法重建细节，并且自回归采样在数千个词元上累积误差。Stable Diffusion、DALL-E 3、Imagen、Midjourney都使用某种形式的扩散。Chameleon（第12.11课）在小规模上部分否定了这一点，但在质量上并未达到SDXL的水平。

Emu3正面攻击了这一论点。其主张：更好的视觉词元器 + 足够的规模 + 下一词元损失 = 在同一模型中进行图像生成（同时也能进行感知）并击败扩散模型。

这一赌注在发表时颇具争议。两年后，开源统一生成系列（Emu3、Show-o、Janus-Pro、Transfusion）已成为研究的默认路径；生产级前沿模型似乎使用了某种变体。

## 核心概念

### Emu3词元器

关键成分是视觉词元器。Emu3训练了一个自定义的IBQ类词元器（反向瓶颈量化器，SBER-MoVQGAN系列），每个词元实现8×8的分辨率缩减。一张512×512的图像变为64×64 = 4096个词元，码本大小为32768。

这比Chameleon的每512×512图像1024个词元（K=8192）要大，但每个词元成本更低（更小的码本查找，更简单的编解码器）。关键指标：重建PSNR为30.5 dB，与Stable Diffusion的连续潜空间（32 dB）有竞争力。

对于视频：一个3D VQ词元器将一个时空补丁（4×4×4像素）编码为一个整数。一个4秒片段（8 FPS）有32帧；在256×256分辨率下，空间缩减4倍，时间缩减4倍，词元数为 (256/4) * (256/4) * (32/4) = 64 * 64 * 8 = 32,768个词元。

词元器质量是上限。Emu3的贡献部分在于“我们训练了一个非常好的词元器”。

### 单损失训练

Emu3使用一个目标：在共享词汇表上对文本词元、2D图像词元和3D视频词元进行下一词元预测。训练时权重按模态特定因子相乘以平衡贡献，但损失函数相同。

在混合数据上训练：
- 图像生成：`<text caption> <image> image_tokens </image>`
- 图像感知：`<text caption> <image> image_tokens </image>`
- 视频生成：`<text caption> <image> image_tokens </image>`
- 视频感知：类似。
- 仅文本：标准NTP。

模型从数据分布中学习何时生成图像词元与文本词元。生成过程源于模型在`<image>`标签后预测图像词元。

### 无分类器指导与温度

自回归图像生成在推理时使用无分类器指导(CFG)会好得多。Emu3使用了它：生成两次，一次使用完整描述，一次使用空描述，将logits以指导权重（通常3.0-7.0）混合。这与扩散使用的CFG技巧相同，被借用到自回归设置中。

温度很重要：过高会导致伪影，过低会导致模式崩溃。Emu3推荐温度为1.0（感知），0.8（图像生成）。

### 三个角色，一个模型

Emu3以三个功能不同的API形式提供，但底层权重集相同：

- Emu3-Gen。图像生成。输入文本，输出图像词元。
- Emu3-Chat。VQA和描述。输入图像（词元），输出文本。
- Emu3-Stage2。视频生成和视频VQA。输入文本或视频，输出文本或视频。

没有任务特定的头部。只是不同的提示模板。同一个检查点。

### 基准测试

来自Emu3论文（2024年9月）：

- 图像生成：在MJHQ-30K FID（5.4 vs 5.6）、GenEval整体（0.54 vs 0.55——统计平局）以及Deep-Eval综合评估上击败SDXL，表现相当。
- 图像感知：在VQAv2（75.1 vs 72.4）上击败LLaVA-1.6，在MMMU上大致匹配。
- 视频生成：4秒片段质量在与Sora时代公开基准模型相比时FVD具有竞争力。

这些数字并非总是胜出——Emu3在这里输一点，在那里赢一点——但“下一词元预测就是一切”的主张在多种模态下都是站得住脚的。

### 计算成本

Emu3在约3000亿多模态词元上训练，模型参数为7B。GPU小时数大致与Llama-2-7B预训练相当（在A100级别芯片上约2000-4000 GPU年）。像Stable Diffusion 3这样的扩散模型在相似的预算下训练，但需要单独的文本编码器和更复杂的流水线。

在推理时，Emu3每张图像比SDXL慢：4096个图像token，30 tok/s的速度下，生成一张512x512的图像大约需要2分钟，而SDXL只需2-5秒。推测性解码(Spectulative decoding)和键值缓存(KV-cache)优化缩小了差距，但并未完全消除。自回归图像生成计算量巨大，这是当前存在的权衡。

### 为什么重要

Emu3的深层贡献在于概念层面。如果下一个token预测(next-token prediction)能在图像生成上与扩散模型(Diffusion)相媲美，那么统一模型路径(一种损失函数，一个骨干网络，任意模态)就是可行的。未来的模型不需要独立的文本编码器、扩散调度器和VAE。一个Transformer，每个模态一个分词器，然后进行规模化扩展。

Show-o、Janus-Pro和InternVL-U都基于或挑战了这一论点。到2025年，中国实验室（如BAAI、DeepSeek）在这一方向上的发表成果比美国实验室更为积极。

## 使用它

`code/main.py`构建了两个小工具：

- 一个2D vs 3D VQ分词器计数计算器：给定（分辨率、块大小、片段长度、每秒帧数），计算图像与视频的token数量。
- 一个自回归图像token采样器，带有温度参数和分类器免引导(Classifier-Free Guidance)。

CFG实现与Emu3的方案一致——将条件logits和无条件logits按引导权重混合。

本课生成`outputs/skill-token-gen-cost-analyzer.md`。给定一个生成产品规格（图像或视频，目标分辨率，质量等级，延迟预算），它计算token数量、推理成本，并选择Emu3系列或扩散模型。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  下一个token预测  |  "NTP"  |  标准自回归损失：给定token[0..i]预测token[i+1]；任何模态经过分词后均适用  |
|  IBQ分词器  |  "逆瓶颈量化器"  |  一类VQ-VAE，具有更大的码本（32768+），重建效果优于Chameleon  |
|  3D VQ  |  "时空量化器"  |  按（时间、行、列）索引的码本；一个token覆盖一个4x4x4像素立方体  |
|  分类器免引导  |  "CFG"  |  将条件logits和无条件logits以权重γ混合；在推理时提升图像质量  |
|  统一词汇表  |  "共享token"  |  文本、图像和视频都来自同一个整数空间；模型预测接下来出现哪种模态  |
|  MJHQ-30K  |  "图像生成基准"  |  包含3万个提示的Midjourney质量基准；Emu3在此报告FID分数  |
