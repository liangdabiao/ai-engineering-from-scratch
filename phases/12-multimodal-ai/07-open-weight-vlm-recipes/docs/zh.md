# 开源权重VLM配方：什么真正重要

> 2024-2026年的开源权重VLM文献是一片消融实验的森林。Apple的MM1测试了13种图像编码器、连接器和数据混合的组合。Allen AI的Molmo证明了详细的人工描述优于GPT-4V蒸馏。Cambrian-1进行了20多种编码器对比。Idefics2正式定义了五轴设计空间。Prismatic VLMs在受控基准上比较了27种训练配方。在所有这些噪音中，有一小部分结果在各论文中保持一致：图像编码器比连接器架构更重要，数据混合比两者都重要，详细的人工描述优于蒸馏合成数据。本课将解读那些表格，这样你就不必亲自去读了。

**类型：** 学习 + 实验
**语言：** Python（标准库，消融表解析器 + 配方选择器）
**前置条件：** 第12阶段 · 05（LLaVA基线）
**时间：** ~180分钟

## 学习目标

- 命名五轴VLM设计空间：图像编码器、连接器、LLM、数据混合、分辨率策略。
- 阅读MM1 / Idefics2 / Cambrian-1消融表，并预测哪个旋钮会影响给定基准。
- 在给定计算预算和任务组合的情况下，为新的VLM选择一个配方（编码器、连接器、数据、分辨率）。
- 解释为什么在相同token数量下，详细的人工描述优于GPT-4V蒸馏。

## 问题

存在数百个开源权重VLM。"好"与"最先进"之间的差距大多不在架构上。而是数据、分辨率策略和编码器选择。当你的模型表现不佳时，知道首先转动哪个旋钮能帮你避免一个500万GPU小时的错误。

2023年的一波（LLaVA-1.5, InstructBLIP, MiniGPT-4）使用了描述对预训练 + LLaVA-Instruct-150k。基线不错。在MMMU上最高达到约35%。

2024年的一波（MM1, Idefics2, Molmo, Cambrian-1, Prismatic VLMs）进行了详尽的消融实验。结果令人惊讶且实用。

## 核心概念

### 五轴设计空间

Idefics2 (Laurençon et al., 2024) 命名了这些轴：

1. 图像编码器。CLIP ViT-L/14, SigLIP SO400m/14, DINOv2 ViT-g/14, InternViT-6B。编码器在补丁大小、分辨率和预训练目标上有所不同。
2. 连接器。MLP（2-4层）、Q-Former（32个查询+交叉注意力）、Perceiver Resampler（64个查询）、C-Abstractor（卷积+双线性池化）。
3. 语言模型。Llama-3 8B / 70B, Mistral 7B, Phi-3, Gemma-2, Qwen2.5。LLM大小是主要的参数成本。
4. 训练数据。描述对（CC3M, LAION）、交错数据（OBELICS, MMC4）、指令数据（LLaVA-Instruct, ShareGPT4V, PixMo, Cauldron）。
5. 分辨率策略。固定224/336/448, AnyRes, 本地动态。训练过程中逐步提升或保持不变。

每个生产级VLM在每个轴上都要做出选择。MMMU分数的差异大部分由轴1、4和5解释——而不是你选择了哪个连接器。

### 轴1：编码器 > 连接器

MM1第3.2节表明：从CLIP ViT-L/14切换到SigLIP SO400m/14增加了3个以上的MMMU点。将连接器从MLP切换到Perceiver Resampler增加不到1个点。Idefics2重复了这一点：SigLIP > CLIP，在相同token数量下Q-Former ≈ MLP ≈ Perceiver。

Cambrian-1的"Cambrian视觉编码器对比"（Tong et al., 2024）在视觉中心基准（CV-Bench）上测试了20多种编码器。排行榜顶部是DINOv2和SigLIP的混合；CLIP处于中游；ImageBind和ViT-MAE较低。从CLIP ViT-L到DINOv2 ViT-g/14在CV-Bench上约有5-7个点的差距。

2026年开源VLM的默认编码器是SigLIP 2 SO400m/14（用于语义+密集特征），有时会与DINOv2 ViT-g/14特征拼接（Cambrian的"空间视觉聚合器"就是这样做的）。

### 轴2：连接器设计差异不大

MM1、Idefics2、Prismatic和MM-Interleaved都得出了相同的结论：在固定视觉token数量下，连接器架构几乎不重要。在相同token预算下，对平均池化补丁使用2层MLP的性能与使用32查询Q-Former的性能相差在1个点以内。

真正重要的是token数量。更多视觉token = 更多LLM计算 = 更好的性能，但到一定程度后收益递减。每张图像64个token对于OCR来说太少。576-1024个token是大多数开源VLM的最佳点。2048+仅对文档和图表有帮助。

Q-Former与MLP是成本问题，而非质量问题：Q-Former无论图像分辨率如何，都将token限制在32-64个；MLP则输出所有补丁token。对于高分辨率输入，Q-Former节省LLM上下文；对于低分辨率，差异可以忽略。

### 轴3：LLM大小设定上限

将LLM从7B加倍到13B，在每个VLM论文中都能稳定增加2-4个MMMU点。在70B时，你在大多数基准上达到饱和。VLM的多模态推理上限就是LLM的文本推理上限——视觉编码器只能提供输入，不能替它推理。

这就是为什么Qwen2.5-VL-72B和Claude Opus 4.7在MMMU-Pro和ScreenSpot-Pro上表现卓越：语言大脑巨大。一个7B的VLM不能通过巧妙的连接器设计来替代70B的VLM。

### 轴4：数据——详细的人工描述优于蒸馏

Molmo + PixMo（Deitke et al., 2024）是2024年每个人都应该阅读的成果。Allen AI让人类标注员以1-3分钟的密集语音到文本方式描述图像，产生了71.2万张密集描述图像。训练数据中完全没有GPT-4V蒸馏。

Molmo-72B在11个基准中的11个上击败了Llama-3.2-90B-Vision。差距不在架构——而在描述质量。详细的人工描述每张图像包含的信息是简短网络描述的5-10倍，并且在事实准确性上优于GPT-4V蒸馏（后者会幻觉）。

ShareGPT4V（Chen et al., 2023）和Cauldron（Idefics2）采用了相同的策略，混合了人工和GPT-4V描述。趋势很明确：对于2026年前沿，描述密度 > 描述数量 > 蒸馏便利性。

### 轴5：分辨率及其策略

Idefics2的消融实验：384 -> 448增加1-2个点。448 -> 980（使用图像分割AnyRes）在OCR基准上再增加3-5个点。固定分辨率训练在中精度处达到平台期；分辨率提升（从224开始，以448或本地分辨率结束）训练更快且最终精度更高。

Cambrian-1进行了分辨率与token的权衡：在固定计算量下，你可以在低分辨率下获得更多token，或者在高分辨率下获得更少token。高分辨率在OCR上胜出；低分辨率多token在一般场景理解上胜出。

2026年生产配方：第一阶段在384固定分辨率下训练，第二阶段使用动态分辨率（最高1280）用于OCR密集型任务。

### Prismatic 受控比较

Prismatic VLMs（Karamcheti 等，2024）是控制所有轴的那篇论文。相同的 13B LLM，相同的指令数据，相同的评估——每次只有一个轴变化。结果：

- 每图像视觉令牌计数解释了约60%的方差。
- 编码器选择解释了约20%。
- 连接器架构解释了约5%。
- 其他所有（数据混合、调度器、学习率）解释了剩余的约15%。

这是一个粗略的分解，但它是文献中对“我应该先消融什么”这个问题最清晰的答案。

### 2026年的选择器

基于这些证据，2026年新项目的默认开放VLM方案如下：

- 编码器：使用原生分辨率的SigLIP 2 SO400m/14并配合NaFlex，如果需要分割/定位的密集特征，则拼接DINOv2 ViT-g/14。
- 连接器：对补丁令牌使用两层MLP。除非受令牌数量限制，否则跳过Q-Former。
- LLM：Qwen2.5 / Llama-3.1 / Gemma 2，7B用于成本控制，70B用于质量，根据目标延迟选择。
- 数据：PixMo + ShareGPT4V + Cauldron，再加上特定任务指令数据。
- 分辨率：动态（长边最小256像素，最大1280像素）。
- 调度：第一阶段对齐（仅投影器），第二阶段全微调，第三阶段特定任务微调。

这些默认设置中的每一个都可以追溯到本课末尾引用的论文中经过测量的消融实验。

## 使用它

`code/main.py` 是一个消融表解析器和方案选择器。它编码了 MM1 和 Idefics2 的消融表（精简版），并允许您查询：

- “给定预算 X 和任务 Y，哪种方案胜出？”
- “如果我在 7B Llama 上将 SigLIP 换成 CLIP，预期的 MMMU 增益是多少？”
- “我应该首先消融哪个轴才能获得 80% 置信度的答案？”

输出是一个带有预期基准增益和“优先消融”建议的排名方案列表。

## 发布

本课产出 `outputs/skill-vlm-recipe-picker.md`。给定目标任务组合、计算预算和延迟目标，它会生成一个完整的方案（编码器、连接器、LLM、数据混合、分辨率调度），并附带证明每个选择的消融实验引用。防止工程师在每次启动新 VLM 项目时重造 Idefics2 消融表。

## 练习

1. 阅读 MM1 第 3.2 节。对于固定 2B LLM 且预算为 5000 万张图像，哪种编码器胜出？如果换成 13B LLM，答案会变化吗？为什么？

2. Cambrian-1 发现，拼接 DINOv2 + SigLIP 在视觉中心基准测试上优于单独使用任意一种，但在 MMMU 上没有增加任何信号。预测哪些基准测试会受益，哪些保持不变。

3. 你的目标是在 2B LLM 上构建一个移动端UI智能体。选择编码器、连接器、分辨率和数据混合。用具体的消融表证明每个选择的合理性。

4. Molmo 发布了 4B 和 72B 模型。4B 与封闭的 7B VLM 竞争力相当；72B 在 11/11 基准测试上击败了 Llama-3.2-90B-Vision。这告诉你关于 LLM 规模平台假设的什么信息？

5. 设计一个消融表，在 7B VLM 上分离数据混合质量和编码器质量。最少需要多少次训练运行？提出四个轴设置。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  消融  |  “转动一个旋钮”  |  训练多次运行，每次在恰好一个设计空间轴上不同，保持其他所有不变  |
|  连接器  |  “桥接”/“投影器”  |  将视觉编码器输出映射到LLM令牌空间的可训练模块（MLP、Q-Former、Perceiver） |
|  详细人工标题  |  “密集标题”  |  一段多句人工编写的描述（通常80-300令牌），比网页替代文本更丰富 |
|  蒸馏  |  “GPT-4V 标题”  |  由更强的专有VLM生成的训练数据；方便但容易继承幻觉 |
|  AnyRes / 动态分辨率  |  “高分辨率路径”  |  通过分块或M-RoPE输入比编码器原生分辨率更大的图像的策略 |
|  分辨率提升  |  “课程”  |  从低分辨率开始并逐渐提高的训练调度，加速对齐学习 |
|  视觉中心基准  |  “CV-Bench / BLINK”  |  强调细粒度视觉感知而非语言密集型推理的评估 |
|  PixMo  |  “Molmo的数据”  |  Allen AI的71.2万张密集标题图像数据集；人类语音转录为密集标题 |
