# 视频-语言模型：时间令牌与定位

> 视频不是一堆照片的堆叠。一段5秒的视频片段具有因果顺序、动作动词和事件时序，图像模型无法表示这些。Video-LLaMA（Zhang等人，2023年6月）发布了首个具有视听定位的开源视频-LLM。VideoChat和Video-LLaVA扩展了这一模式。到2025年，Qwen2.5-VL的TMRoPE缩小了与前沿专有模型的差距。每个系统以不同的方式处理时间令牌——每个片段使用Q-former，每帧使用concat-pool，每个令牌使用TMRoPE。本课分析这些模式，构建均匀vs动态帧采样器，并在时间定位任务上进行评估。

**类型：** 构建
**语言：** Python（标准库，帧采样器 + 时间定位评估器）
**前置条件：** 阶段12 · 08（LLaVA-OneVision）
**时间：** ~180分钟

## 学习目标

- 解释为什么时间位置编码能独立于视觉编码器改变视频VLM的性能。
- 比较均匀、动态FPS和事件驱动帧采样在每秒令牌数对比定位准确度上的表现。
- 描述每个片段Q-former（Video-LLaMA）、每帧池化（Video-LLaVA）与每个令牌M-RoPE（Qwen2.5-VL）的设计。
- 列举四个视频基准：VideoMME、TempCompass、EgoSchema、Video-MMMU。

## 问题

一段30 FPS的1分钟视频有1800帧。每帧196个视觉令牌（ViT-B，224分辨率），那就是352k个令牌——超过了任何2024年时期LLM的上下文长度。

存在三种缩减策略：

1. 子采样帧（根据内容1-8 FPS）。
2. 对每帧的补丁令牌进行激进池化（3x3或4x4双线性池化）。
3. 通过Q-former压缩，它接受16帧片段并输出64个令牌。

每种权衡都不同。子采样丢失时间细节。池化丢失空间细节。Q-former两者都损失一点，但节省令牌。

时间位置编码是另一个维度：模型如何知道第5帧在第6帧之前？选项包括简单的一维时间RoPE（Video-LLaMA）、学习的时间嵌入（Video-LLaVA）和TMRoPE（Qwen2.5-VL，全三维）。

## 核心概念

### Video-LLaMA：每个片段的Q-former + 音频分支

Video-LLaMA（2023）是首个开源视频-LLM。架构：

- 16帧片段，2 FPS（即8秒）。
- 每帧ViT特征 -> 视频Q-former，它交叉注意力所有16帧 -> 32个学习查询 -> LLM。
- 并行音频分支：波形 -> ImageBind音频编码器 -> 音频Q-former -> 32个查询 -> LLM。

优点：视听联合推理。缺点：固定片段长度，无任意时间定位。

### VideoChat和Video-LLaVA

VideoChat保留了Video-LLaMA的理念，但去除了音频并简化了。Video-LLaVA（Lin等人，2023）在图像和视频帧上训练了一个单一的视觉编码器（“投影前对齐”），提供统一表示。两者都是冻结CLIP编码器 + MLP + LLM。

两者都无法处理长视频。它们都是8-16帧的系统。

### Qwen2.5-VL和TMRoPE

Qwen2.5-VL引入了TMRoPE——时间-模态旋转位置嵌入。每个补丁令牌携带一个(t, h, w)位置，其中t是实际时间戳（不是帧索引）。

与简单时间嵌入的关键区别：

- 绝对时间，而非索引。模型看到“在4.2秒”而不是“在第15帧”。
- 每个令牌旋转，而非每个片段。每个视觉令牌由其时间戳独立旋转。
- 兼容动态FPS。如果你这里以2 FPS采样，那里以4 FPS采样，TMRoPE原生处理不均匀间隔。

TMRoPE使得回答“猫在多少秒跳跃？”的问题成为可能。模型可以输出“在4.2秒”。Video-LLaMA只能说“在片段早期”。

### 帧采样策略

均匀：在持续时间内均匀采样N帧。简单，但丢失运动峰值。

动态FPS：基于运动强度自适应采样。光流或帧差法选取高运动片段进行更密集的采样。Qwen2.5-VL在此训练。

事件驱动：运行轻量级检测器，在动作发生处采样更多。VideoAgent使用。

关键帧+上下文：在镜头边界及少量相邻帧采样。用于电影内容。

### 每帧池化

在1 FPS且每帧576令牌时，一段5分钟片段有172,800个令牌。用Qwen2.5-VL-72B的128k上下文可以处理，但代价高昂。

3x3双线性池化减少至每帧64令牌 -> 5分钟19,200令牌。对大多数任务是最佳点。

更激进地池化（6x6 -> 每帧16令牌）用于空间细节不太重要的智能体工作流。

### 四大视频基准测试

- VideoMME：综合视频理解，短+中+长视频。
- TempCompass：细粒度时序推理，"之前"/"之后"问题。
- EgoSchema：长程第一人称视频。
- Video-MMMU：多模态多学科视频问题。

完整的视频-VLM评估覆盖所有四个。它们侧重不同维度——TempCompass全关于排序，EgoSchema关于3分钟以上的推理，VideoMME跨越不同时长。

### 定位输出格式

时序定位的输出格式：

- 自由文本："猫在大约4秒标记处跳跃。" 易于解析但不精确。
- 结构化JSON：`{"event": "jump", "start": 4.1, "end": 4.3}`。Qwen2.5-VL训练此格式。
- 基于Token：特殊`{"event": "jump", "start": 4.1, "end": 4.3}`Token与答案交错。Qwen2.5-VL的内部格式。

基于Token对下游使用最准确。Qwen2.5-VL的JSON输出格式可直接解析。

### 2026年最佳实践

对于2026年的视频VLM：

- 编码器：SigLIP 2配合M-RoPE或TMRoPE（Qwen2.5-VL）。
- 帧采样：动态FPS（1-4取决于运动）带最大帧数上限。
- 每帧池化：3x3双线性。
- 输出：带时间+事件字段的结构化JSON。
- 基准测试：通用场景用VideoMME+TempCompass；长程用EgoSchema。

## 使用它

`code/main.py`包括：

- 均匀和动态FPS帧采样器。
- 一个玩具时序定位评估器：给定时间T的"真实"事件和模型输出，带容差的评分准确度。
- 跨Video-LLaMA（16帧，Q-former）、Video-LLaVA（8帧，MLP）、Qwen2.5-VL（动态FPS+TMRoPE）的比较。

本课程生成`outputs/skill-video-vlm-frame-planner.md`。给定一个视频任务（监控、动作识别、时序定位、摘要），它选择帧采样器、池化因子、输出格式和预期准确度层级。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  时序定位  |  "时间定位答案"  |  VLM输出事件发生时的具体时间戳范围  |
|  TMRoPE  |  "时间-多模态RoPE"  |  带绝对时间戳的3D旋转位置，由Qwen2.5-VL使用  |
|  动态FPS  |  "运动感知采样"  |  在高运动片段采样更多帧，在静态片段采样更少帧  |
|  帧池化  |  "每帧空间压缩"  |  在LLM之前用双线性插值减少每帧的块数  |
|  视频Q-former  |  "剪辑压缩器"  |  将N帧映射到K个可学习查询的交叉注意力瓶颈  |
|  VideoMME  |  "视频基准"  |  综合短/中/长视频基准测试，2500+样本  |
