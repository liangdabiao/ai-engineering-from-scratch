# 从CLIP到BLIP-2 — Q-Former作为模态桥梁

> CLIP对齐了图像和文本，但无法生成标题、回答问题或进行对话。BLIP-2（Salesforce，2023）通过一个小的可训练桥梁解决了这个问题：32个可学习的查询向量通过交叉注意力关注冻结的ViT特征，然后直接插入冻结的LLM的输入流。188M参数的桥梁连接了11B的LLM和ViT-g/14。直到2026年，每个基于适配器的VLM——MiniGPT-4、InstructBLIP、LLaVA的变体——都是它的后代。本节课将解读Q-Former的架构，解释其两阶段训练，并构建一个将视觉标记输入冻结文本解码器的简易版本。

**类型：** 构建
**语言：** Python（标准库，交叉注意力+可学习查询演示）
**前置要求：** 第12阶段·02（CLIP），第7阶段（Transformers）
**时间：** ~180分钟

## 学习目标

- 解释为什么在冻结的视觉编码器和冻结的LLM之间使用可训练的瓶颈在成本和稳定性上优于端到端微调。
- 实现一个交叉注意力块，其中一组固定的可学习查询关注外部图像特征。
- 逐步介绍BLIP-2的两阶段预训练：表示学习（ITC + ITM + ITG）然后是生成式学习（带冻结解码器的LM损失）。
- 比较Q-Former与LLaVA中使用的更简单的MLP投影器，并论证每种选择的优势场景。

## 问题

你有一个冻结的ViT，每张图像生成256个维度为1408的块标记。你有一个冻结的7B LLM，它期望维度为4096的标记嵌入。直接的桥梁——从1408到4096的线性层——是可行的，但将所有256个块标记输入LLM的上下文需要每张图像额外的256个标记。在32张图像的批次中，仅视觉模态就消耗了8192个标记。

BLIP-2的问题：能否将256个标记的图像表示压缩到更少的标记（比如32个），同时保留足够的信息，让LLM能够对图像进行描述、回答问题和推理？并且能否在不触碰冻结主干的情况下训练这个桥梁，使训练成本仅涉及桥梁的参数？

答案：Q-Former。32个可学习的"查询"向量，它们对ViT的块标记进行交叉注意力，生成一个由LLM消费的32标记视觉摘要。总计188M参数。在与LLM接触之前，使用对比、匹配和生成目标进行训练。

## 核心概念

### 可学习查询

Q-Former的核心技巧：不让LLM的文本标记关注图像块，而是引入一组新的32个可学习查询向量`Q`，让*它们*关注图像块。查询是模型的参数——在训练过程中学习，并且每个图像都使用相同的32个查询。

在交叉注意力之后，每个查询都持有图像的压缩摘要——"描述主要物体"，"描述背景"，"计数物体"等。查询并不严格专精于语义标签；它们学习任何能使下游损失下降的编码。

### 架构

Q-Former是一个小型transformer（12层，约100M参数），有两条路径：

1. 查询路径：32个查询向量经过自注意力（彼此之间），然后经过冻结ViT块标记上的交叉注意力，最后经过FFN。
2. 文本路径：一个类似BERT的文本编码器与查询路径共享自注意力和FFN权重。文本路径的交叉注意力被禁用。

在训练时，两条路径同时运行。查询和文本通过共享的自注意力交互，这意味着查询可以在需要文本的任务（ITM、ITG）中根据文本进行调整。在VLM交接的推理时，只有查询通过，生成32个视觉标记。

### 两阶段训练

BLIP-2进行两阶段预训练：

阶段1：表示学习（无LLM）。三种损失：
- ITC（图像-文本对比）：池化后的查询标记与文本CLS标记之间的CLIP风格对比。
- ITM（图像-文本匹配）：二分类器——该图像-文本对是否匹配？使用困难负样本挖掘。
- ITG（图像引导的文本生成）：以查询为条件的因果LM头部。迫使查询编码可生成文本的内容。

只有Q-Former进行训练。ViT被冻结。不涉及LLM。

阶段2：生成式学习。附加一个冻结的LLM（OPT-2.7B或Flan-T5-XL等）。通过一个小型线性层将32个查询输出投影到LLM的嵌入维度。将它们前置到文本提示前面。仅在串联的提示+图像+标题序列上训练线性投影和Q-Former的LM损失。

在阶段2之后，Q-Former + 投影构成完整的视觉适配器。推理时：图像 → ViT → Q-Former → 线性投影 → 前置到文本 → 冻结的LLM生成输出。

### 参数经济性

BLIP-2带有ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（188M，训练）= 总计8B，训练188M。仅Q-Former就占整个栈参数的约2.4%。训练成本反映了这一点：在几块A100上训练几天 vs 端到端训练数周。

质量：BLIP-2在零样本VQA上匹配或超越了Flamingo-80B，同时体积小50倍。桥梁有效。

### InstructBLIP与指令感知Q-Former

InstructBLIP（2023）通过额外输入扩展了Q-Former：指令文本本身。在交叉注意力时，查询现在可以同时访问图像块和指令。查询可以每条指令专门化（"数汽车"，"描述情绪"），而不是学习一个固定的摘要。在保留任务上获得基准提升。

### MiniGPT-4与仅投影器方法

MiniGPT-4保留了Q-Former，但只训练输出线性投影，同时冻结其他所有部分。成本低，但代价是质量——查询是BLIP-2的，不是你的。适合快速迭代，但不是最佳架构。

### 为什么LLaVA更简单

LLaVA（2023，第12.05课）用简单的2层MLP替换了Q-Former，该MLP将每个ViT块标记投影到LLM空间——对于24x24网格，每张图像576个标记，全部输入LLM。压缩更差，但让LLM能够关注原始块。这在当时是有争议的；到2023年底，它成为主流，因为视觉指令数据（LLaVA-Instruct-150k）证明MLP可以被训练以保留足够信号。权衡：LLaVA的上下文填充更快，但它自然地扩展到多图像和视频。

到2026年，该领域出现分野：在Token预算至关重要的情况下（长视频、多张图像），Q-Former生存下来；而在追求每个Token原始质量的场景下，MLP投影器占据主导。

### 门控交叉注意力：Flamingo，祖先

Flamingo（第12.04课）早于BLIP-2，使用了相同的交叉注意力思想，但应用于每个冻结的LLM层，而不是作为一个单一的桥梁。BLIP-2表明，可以仅压缩到输入层，且仍然有效。Gemini和Idefics结合了两者：交错输入Token加上可选的用于上下文少样本的门控交叉注意力。

### 2026年的后代

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4以及大多数视频-语言模型（出于Token预算原因）。
- 感知器重采样器：Flamingo的变体（第12.04课）；Idefics系列、Eagle、OmniMAE。
- MLP投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

所有四种方法都有效。决定性的问题是：你是受Token预算约束，还是受每个Token的质量约束。

## 使用它

`code/main.py`构建了一个标准库式的Q-Former风格交叉注意力：

1. 模拟256个图像块Token（维度128）。
2. 实例化32个可学习查询（维度128）。
3. 执行缩放点积交叉注意力（查询来自查询，键/值来自块）。
4. 通过线性层投影到LLM维度（512）。
5. 输出32个LLM可用的视觉Token。

所有数学运算均用纯Python实现（对向量进行嵌套循环）。虽为玩具示例，但形状正确。打印注意力权重矩阵，以便查看每个查询从哪些块中提取信息。

## 练习

1. 在PyTorch中实现交叉注意力模块。验证当有32个查询和256个键/值时，注意力权重矩阵为32×256，且每行在softmax后求和为1。

2. 在BLIP-2的第一阶段，Q-Former同时运行三个损失函数：ITC、ITM、ITG。用伪代码写出每个损失函数的前向签名。哪一个需要文本编码器路径处于激活状态？

3. 比较参数量：Q-Former（12层，隐藏层768）vs 一个2层MLP投影器（1408→4096，两层）。在多大的LLM规模下，1.88亿参数的Q-Former成本才能在训练效率上得到回报？

4. 阅读BLIP-2论文（arXiv:2301.12597）第3.2节中关于Q-Former初始化的内容。解释为什么从BERT-base（而非随机）初始化能加速收敛。

5. 对于一个10分钟的视频，以1 FPS采样得到60帧，计算每帧的Token成本：（Q-Former → 32 Token/帧）vs（MLP投影器 → 576 Token/帧）。哪一个能放入128k Token的LLM上下文窗口？

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  Q-Former  |  "Querying transformer"  |  带有32个可学习查询向量的小型Transformer，对冻结的ViT特征进行交叉注意力  |
|  可学习查询  |  "视觉的软提示"  |  一组固定的参数，作为交叉注意力的查询端；每个模型学习，所有输入共享  |
|  交叉注意力  |  "查询来自这里，键/值来自那里"  |  查询、键和值来自不同来源的注意力；查询如何从ViT块中提取信息  |
|  ITC  |  "图像-文本对比"  |  CLIP风格的损失，应用于Q-Former池化后的查询与文本CLS之间  |
|  ITM  |  "图像-文本匹配"  |  基于硬负例挖掘对的二元分类器；迫使查询区分细粒度不匹配  |
|  ITG  |  "图像引导的文本生成"  |  因果LM损失，其中文本在查询条件下生成；迫使查询编码可解码为文本的内容  |
|  两阶段预训练  |  "表示学习然后生成"  |  第一阶段单独训练Q-Former（ITC/ITM/ITG）；第二阶段附加冻结的LLM，仅训练投影层和Q-Former  |
|  冻结主干  |  "不微调"  |  视觉编码器和LLM的权重固定；只有桥梁进行训练  |
|  投影头  |  "线性映射到LLM维度"  |  将Q-Former输出映射到LLM嵌入维度的最终线性层  |
|  感知器重采样器  |  "Flamingo的版本"  |  类似的可学习查询交叉注意力，被Flamingo用于每一层，而非作为单一桥梁  |
