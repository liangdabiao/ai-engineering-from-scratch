# 具身VLA：RT-2, OpenVLA, π0, GR00T

> 首个从网站读取食谱并在厨房机器人上执行它的模型是RT-2（Google DeepMind，2023年7月）。RT-2将动作离散化为文本词元，在Web数据加机器人动作数据上共同微调VLM，并证明了Web规模的视觉-语言知识可迁移到机器人控制。OpenVLA（2024年6月）发布了开源的7B参考模型。Physical Intelligence的π0系列（2024-2025）加入了流匹配动作专家。NVIDIA的GR00T N1（2025年3月）为人形机器人提供了大规模双系统（系统1/系统2）控制。VLA原语——视觉-语言-动作，一个能看、能读、能动作的单一模型——是连接本阶段理解模型与第15阶段自主系统的桥梁。

**类型：** 学习
**语言：** Python（标准库，动作分词器 + VLA推理骨架）
**前置知识：** 第12阶段·05（LLaVA），第15阶段（自主系统，提及）
**时间：** ~180分钟

## 学习目标

- 描述动作分词化：离散区间编码（RT-2），FAST高效动作词元，连续流匹配动作（π0）。
- 解释为何在Web+机器人数据上共同微调能将通用知识迁移到新任务。
- 比较OpenVLA（开源7B Llama+VLM）、π0（流匹配）和GR00T N1（双系统）在相同机器人任务上的表现。
- 说出Open X-Embodiment数据集及其作为RT-X训练语料库的作用。

## 问题

自20世纪70年代以来，能根据自然语言指令做家务的机器人一直是研究目标。2020年代的答案：视觉-语言-动作（VLA）模型。与用于VQA的VLM架构相同，但输出是动作（关节扭矩、末端执行器位姿、离散命令）而非文本。

VLA特有的挑战：

1. 动作空间是连续的（关节角度、力）且高维（7自由度手臂+3自由度夹爪=10维，30Hz）。
2. 机器人的训练数据稀缺。Open X-Embodiment约有100万条轨迹；Web文本-图像有50亿+。
3. 控制频率至关重要。30Hz控制回路意味着每个动作预算33ms。
4. 安全性。错误的动作可能损坏硬件、人类或财产。

## 核心概念

### 动作分词化（RT-2）

RT-2的技巧：将每个关节目标表示为量化后的文本词元。将归一化的[-1, 1]范围离散化为256个区间，每个区间映射到一个词汇ID。每个控制步中，10自由度动作变成10个词元。

在混合数据上共同微调PaLM-X VLM：

- Web图像-文本对（图像描述、VQA）。
- 机器人演示，动作作为词元。

模型看到“拿起红色方块”（语言）→图像（视觉）→10词元动作序列（离散化关节目标）。Web预训练保留了通用知识迁移：RT-2能执行“向快速移动的物体移动”指令，尽管“快速移动”不在训练数据中。

RT-2论文中推理速度为3-5 Hz，受限于VLM的自回归解码。

### OpenVLA——开源7B参考模型

OpenVLA（Kim等，2024年6月）是开源权重的RT-2等价物。7B Llama骨干，DINOv2 + SigLIP双视觉编码器，256区间的动作分词化。

在Open X-Embodiment（22个机器人上的97万条轨迹）上训练。支持LoRA微调以适应新机器人。

推理：量化的A100上4-5 Hz。对于慢速操作足够快，但不适合高频控制。

### FAST分词器——更快的动作解码

Pertsch等（2024年）指出，离散区间分词化效率低——大多数动作聚集在区间空间的狭小区域内。FAST（频域动作序列分词器）通过DCT压缩动作序列并量化系数。

30步动作轨迹变成约10个FAST词元，而非300个离散区间词元。推理速度提升3-5倍且无质量损失。

### π0与流匹配动作

Physical Intelligence的π0（Black等，2024年10月）用流匹配动作专家替代离散动作词元：

- 一个小型动作Transformer读取VLM的隐状态，通过修正流输出连续50步的动作序列。
- 动作头以流匹配损失训练；VLM预训练保持不变。
- 推理：约5步去噪步骤中输出完整动作序列，实际控制频率50 Hz。

π0声称：在广泛的操作任务上胜过OpenVLA和Octo。连续动作公式保留了离散化破坏的平滑性。

π0.5和π0-FAST是增量升级。π0-FAST结合了FAST分词化与流匹配。

### GR00T N1——人形机器人的双系统

NVIDIA的GR00T N1（2025年3月）专为人形机器人（>30自由度，全身）而建：

- 系统2：大型VLM读取场景+指令，以约1 Hz频率产生高层子目标。
- 系统1：小型动作头Transformer，以50-100 Hz频率产生低层关节命令，条件于子目标。

这种分割对应卡尼曼的快慢思维：系统2规划，系统1执行。优点：VLM大小的慢速规划不阻塞快速控制；系统1保持小巧以降低延迟。

GR00T N1.7（2025年底）改进了数据规模。GR00T使用来自Omniverse的仿真到现实数据进行微调。

### 开放X-具身(Open X-Embodiment)

训练数据。RT-X（2023年10月）汇集了22个数据集，涵盖22个机器人的100万条轨迹。开放X-具身(Open X-Embodiment)是大家使用的语料库：

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table。
- 每个样本：（机器人状态、相机视角、指令、动作序列）。
- 训练规范：统一动作空间，归一化关节范围，调整相机尺寸。

OpenVLA和π0在开放X-具身(Open X-Embodiment)上训练。通过LoRA微调100-1000个任务特定演示，弥补与任何特定机器人之间的领域差距。

### 协同微调 vs 仅机器人数据

协同微调将网络VQA数据与机器人轨迹混合。比例很重要：过多的VQA会导致模型忘记动作；过多的机器人数据会导致模型失去通用知识。

RT-2的比例：约1:1。OpenVLA：约0.5:1（网络对机器人）。π0：类似。精确比例是需要根据数据集大小调整的超参数。

仅机器人训练会产生任务特定模型，这些模型无法处理分布外的指令。协同微调是“拿起红色立方体（演示中）”与“从左边拿起第三大的物体（新表述）”之间的区别。

### 安全与动作限制

每个生产级VLA都配备：

- 硬关节限位（不能超出规格的扭矩）。
- 速度限制（软裁剪）。
- 工作空间边界（末端执行器不能离开桌面）。
- 对新任务的人工审核批准。

这些作为控制层检查位于VLA之外。VLA的输出是建议，而非命令。

## 使用它

`code/main.py`：

- 实现256箱动作标记化和去标记化。
- 勾勒基于DCT+量化的FAST标记器。
- 比较不同（离散箱、FAST、连续流）每个动作步的标记数。
- 打印RT-2 → OpenVLA → π0 → GR00T的演变总结。

## 发布

本课产生`outputs/skill-vla-action-format-picker.md`。给定一个机器人任务（操作、导航、人形全身），在离散箱+RT-2、FAST+OpenVLA、流匹配+π0或双系统+GR00T之间进行选择。

## 练习

1. 一个10自由度手臂以30 Hz控制速率运行。离散箱标记化（256箱）每秒产生多少个标记？一个7B的VLM能跟上吗？

2. FAST标记化将30步轨迹压缩到约10个标记。如果轨迹包含高频运动（例如击鼓），用户会损失什么？

3. π0的流匹配头在大约5步内去噪。比较其吞吐量与OpenVLA自回归解码（4-5 Hz）。

4. GR00T的系统1/系统2划分借鉴了卡尼曼。提出一个不同的划分（系统3？），可能有助于双足行走。

5. 阅读开放X-具身(Open X-Embodiment)第4节关于数据集策划。列出防止领域泄露的三个策划规则。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  VLA  |  "视觉-语言-动作"  |  接收图像和指令并输出动作命令的模型  |
|  动作标记化  |  "离散箱"  |  将连续关节目标量化为每维度256个箱，每个箱是一个词汇ID  |
|  FAST标记器  |  "频率动作标记"  |  DCT+量化，将30步轨迹压缩到约10个标记  |
|  协同微调  |  "混合网络+机器人"  |  在网络VQA数据和机器人演示上共同训练，以保留通用知识  |
|  流匹配动作头  |  "π0连续输出"  |  通过修正流输出50步动作序列的小型变压器  |
|  系统1/系统2  |  "双系统控制"  |  大型VLM缓慢规划，小型动作头快速行动；GR00T模式  |
|  开放X-具身  |  "RT-X数据集"  |  100万轨迹的跨机器人数据集；训练语料库  |
