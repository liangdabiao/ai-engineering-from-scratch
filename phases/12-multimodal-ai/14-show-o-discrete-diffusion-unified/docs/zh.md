# Show-o 与离散扩散统一模型

> Transfusion混合了连续和离散表示。Show-o（Xie等人，2024年8月）则反其道而行之：文本令牌使用因果式下一个令牌预测，图像令牌使用MaskGIT风格的掩码离散扩散。两者位于同一个Transformer中，配以混合注意力掩码。结果统一了VQA、文本到图像、图像修复以及混合模态生成于一个骨干网络、每个模态一个分词器、一个损失公式（将下一个令牌预测扩展到掩码预测）。本课程讲解Show-o的设计——为何掩码离散扩散是一种并行、少步图像生成器——并与Transfusion和Emu3进行对比。

**类型：** 学习
**语言：** Python（标准库，掩码离散扩散采样器）
**先修知识：** 阶段12 · 13（Transfusion）
**时间：** 约120分钟

## 学习目标

- 解释掩码离散扩散：一种通过均匀掩码令牌然后让Transformer恢复它们的调度方案。
- 比较并行图像解码（Show-o、MaskGIT）与自回归图像解码（Chameleon、Emu3）在速度和质量上的差异。
- 列举Show-o在一个检查点中处理的三个任务：T2I、VQA、图像修复。
- 选择一个掩码调度方案（余弦、线性、截断）并说明其对样本质量的影响。

## 问题

Transfusion的双损失训练有效但具有更复杂的动态——连续扩散损失与离散NTP（下一个令牌预测）损失处于不同的数值尺度。平衡损失权重是一项超参数搜索。该架构有效但复杂。

Show-o的答案：保持两个模态都是离散的（如Chameleon），但通过掩码离散扩散并行生成图像，而不是顺序生成。训练目标变为单一的掩码令牌预测，自然地推广了下一个令牌预测。

## 核心概念

### 掩码离散扩散（Masked Discrete Diffusion）（MaskGIT）

原始Chang等人（2022）的MaskGIT技巧非常优雅。从一个完全掩码的图像开始（每个令牌都是特殊的`<MASK>` id）。每一步并行预测所有掩码令牌，然后保留最自信的Top-K预测并重新掩码其余部分。经过大约8-16次迭代后，所有令牌都被填充。每一步要解掩的令牌数量是通过调度调整的——余弦调度效果很好。

训练很简单：从[0, 1]均匀采样一个掩码比例，将其应用于图像的VQ（向量量化）令牌，训练Transformer恢复被掩码的令牌。这与BERT对文本所做的完全一样，只是规模扩展到图像生成。

### Show-o：一个Transformer，混合掩码

Show-o将MaskGIT置于因果语言模型Transformer内部。注意力掩码为：

- 文本令牌：因果（标准大语言模型）。
- 图像令牌：图像块内完全双向（因此被掩码的令牌在预测期间可以看到所有其他图像令牌）。
- 文本到图像：文本关注先前的图像，图像关注先前的文本。

训练交替进行：
1. 标准NTP（下一个令牌预测）在文本序列上。
2. T2I样本：文本→图像，带有掩码图像令牌，掩码令牌预测损失。
3. VQA样本：图像→文本，带有掩码文本令牌（实际上就是NTP）。

统一损失是`<MASK>`令牌上的交叉熵，它既涵盖文本NTP（只有最后一个令牌被“掩码”）也涵盖图像掩码扩散（随机子集被掩码）。

### 并行采样

Show-o大约在16步内生成一张图像，而不是约1000步（每个令牌自回归）或约20步（扩散）。每一步并行预测所有掩码令牌；提交最自信的Top-K；重复。

比较：
- Chameleon / Emu3（令牌自回归）：N_tokens 次前向传播，通常每张图像1024-4096次。
- Transfusion（连续扩散）：约20步，每一步是一次完整的Transformer前向传播。
- Show-o（掩码离散扩散）：约16步，每一步是一次完整的Transformer前向传播。

在相似规模的模型下，Show-o比Chameleon更快，步数与Transfusion大致相当，但每步成本更低（离散词汇logits对比连续MSE损失）。

### 单个检查点中的任务

Show-o在推理时支持四种任务，通过提示格式选择：

- 文本生成：标准自回归文本输出。
- VQA：图像输入，文本输出。
- T2I：文本输入，通过掩码离散扩散输出图像。
- 图像修复：部分令牌被掩码的图像，填充缺失部分。

图像修复能力来自于掩码预测训练，无需额外工作。掩码VQ令牌网格中的一个区域，将剩余部分加上文本提示输入，预测被掩码的令牌。

### 掩码调度

每一步要解掩的令牌数量的调度影响质量。Show-o推荐余弦调度：

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

在步骤0时，所有令牌被掩码（比例1.0）。在步骤T时，无掩码。余弦调度将质量集中在预测信息最丰富的中等比例上。线性调度也有效，但快速达到平台期。

### Show-o2

Show-o2（2025年后续工作，arXiv 2506.15564）扩展了Show-o：更大的大语言模型基础，更好的分词器，改进的掩码调度。相同的架构模式。

### Show-o的定位

在2026年的分类中：

- 离散令牌 + NTP: Chameleon, Emu3. 简单但推理速度慢。
- 离散令牌 + 掩码扩散: Show-o, MaskGIT, LlamaGen, Muse. 并行采样，但仍有分词器的损失。
- 连续 + 扩散: Transfusion, MMDiT, DiT. 质量最高，训练更复杂。
- 连续 + VLM中的流匹配: JanusFlow, InternVL-U. 最新。

按任务选择：当你想要一个开放模型同时具备T2I（文本到图像）+ 填充 + VQA（视觉问答）且速度合理时，选Show-o；当质量最重要且你能承受双损失管道时，选Transfusion。

## 使用它

`code/main.py` 模拟了Show-o的采样：

- 一个包含16个VQ令牌的简易网格。
- 一个模拟的“transformer”，根据提示和当前未遮蔽的令牌预测logits。
- 基于余弦调度进行8步的并行掩码采样。
- 打印中间状态（掩码模式演化）和最终令牌。

运行它，观察掩码逐步消融。

## 练习

1. 掩码离散扩散大约需要16步采样。为什么不是1步？如果在第0步就解开所有遮蔽，会出现什么问题？

2. 填充功能在掩码扩散中是免费的。提出一个产品用例（真实或假设），其中Show-o的填充性能优于专用模型。

3. 余弦调度与线性调度：追踪T=8时每步未遮蔽令牌的数量。哪种更平衡？

4. 一张512x512的Show-o图像对应1024个令牌。在词汇量K=16384下，模型输出1024 * log2(16384) = 14,336比特（约1.75 KiB）数据。Stable Diffusion输出512*512*24比特 = 6,291,456比特（约768 KiB）的原始像素。压缩比是多少，它换来了什么质量？

5. 阅读LlamaGen (arXiv:2406.06525)。LlamaGen的类条件自回归图像模型与Show-o的掩码方法有何不同？

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  掩码离散扩散  |  "MaskGIT风格"  |  训练预测被遮蔽的令牌；推理时，迭代地解开最确信的预测  |
|  余弦调度  |  "解遮蔽调度"  |  在推理步骤中遮蔽比率衰减；将置信度增长集中于中间范围  |
|  并行解码  |  "一次全部令牌"  |  每一步在一次前向传播中预测所有被遮蔽令牌的完整序列，然后提交前K个  |
|  混合注意力  |  "因果 + 双向"  |  遮蔽在文本令牌上是因果的，在图像块内是双向的  |
|  填充  |  "填充生成"  |  以某些令牌被遮蔽的图像为条件，预测缺失的令牌；从训练目标中免费获得  |
|  提交率  |  "每步Top-K"  |  每次迭代声明“完成”的令牌数量；控制推理与质量的权衡  |
