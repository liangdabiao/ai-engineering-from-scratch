# 视觉变换器与补丁-令牌原语

> 在任何多模态处理之前，图像必须变成变换器可以处理的令牌序列。2020年的ViT论文通过16x16像素的补丁、线性投影和位置嵌入解决了这一问题。五年后，所有2026年的前沿模型（Claude Opus 4.7原生支持2576px，Gemini 3.1 Pro，Qwen3.5-Omni）仍然以此方式开始——编码器从ViT变为DINOv2再到SigLIP 2，增加了寄存器令牌，位置方案变为2D-RoPE，但原语保持不变。本课从头到尾解读补丁-令牌流水线，并用stdlib Python构建它，以便第12阶段的其余部分对“视觉令牌”有一个具体的心理模型。

**类型：** 学习
**语言：** Python (stdlib, 补丁分词器 + 几何计算器)
**前置要求：** 第7阶段 (变换器), 第4阶段 (计算机视觉)
**时间：** ~120分钟

## 学习目标

- 将HxWx3图像转换为带有正确位置编码的补丁令牌序列。
- 计算给定(补丁大小、分辨率、隐藏维度、深度)下ViT的序列长度、参数数量和FLOPs。
- 说出将ViT从2020年研究升级到2026年生产的三个改进：自监督预训练(DINO / MAE)、寄存器令牌和原生分辨率打包。
- 为下游任务在CLS池化、均值池化和寄存器令牌之间做出选择。

## 问题

变换器操作于向量序列。文本已经是序列（字节或令牌）。图像是像素的2D网格，带有三个颜色通道——而不是序列。如果展平每个像素，224x224 RGB图像将变成150,528个令牌，此时的自我注意力不可行（序列长度呈二次增长）。

2020年之前的做法是将CNN特征提取器绑在前面：ResNet产生7x7的2048维向量特征图，将这49个令牌送入变换器。这种方法可行，但继承了CNN的偏差（平移等变性、局部感受野），并失去了变换器对规模的适应性。

Dosovitskiy等人(2020)提出了一个直白的问题：如果我们跳过CNN呢？将图像分成固定大小的补丁（例如16x16像素），将每个补丁线性投影为向量，添加位置嵌入，然后将序列送入普通变换器。当时这被视为异端——没有卷积的视觉。凭借足够的数据（JFT-300M，然后是LAION），它在ImageNet上击败了ResNet，并持续改进。

到2026年，ViT原语已成为无可争议的基础。每个开放权重的VLM的视觉塔都是其某种后代（DINOv2, SigLIP 2, CLIP, EVA, InternViT）。问题不再是“我们应该使用补丁吗？”，而是“什么补丁大小、什么分辨率调度、什么预训练目标、什么位置编码。”

## 核心概念

### 补丁作为令牌

给定形状为`x`的图像`(H, W, 3)`和补丁大小`P`，将图像切割成`(H/P) x (W/P)`个不重叠的补丁网格。每个补丁是一个`P x P x 3`像素立方体。将每个立方体展平为`3 P^2`维向量。应用形状为`W_E`的共享线性投影`(3 P^2, D)`，将每个补丁映射到模型的隐藏维度`D`。

对于ViT-B/16的标准配置：
- 分辨率224，补丁大小16 → 网格14x14 → 196个补丁令牌。
- 每个补丁是`16 x 16 x 3 = 768`个像素值，投影到`D = 768`。
- 添加一个可学习的`16 x 16 x 3 = 768`令牌 → 序列长度197。

补丁投影在数学上等同于一个2D卷积，其核大小为`P`，步长为`P`，输出通道数为`D`。这就是生产代码实际实现它的方式——`nn.Conv2d(3, D, kernel_size=P, stride=P)`。“线性投影”框架是概念性的，而核框架是高效的。

### 位置嵌入

补丁没有固有顺序——变换器将它们视为一个袋子。早期的ViT添加了可学习的1D位置嵌入（每个位置一个768维向量，共197个）。这有效，但将模型绑定到训练分辨率：在推理时，如果改变网格，必须插值位置表。

现代视觉骨干使用2D-RoPE（Qwen2-VL的M-RoPE，SigLIP 2的默认）或因子化的2D位置。2D-RoPE基于补丁的行、列索引旋转查询和键向量，因此模型从旋转角度推断相对2D位置。没有位置表。模型在推理时处理任意网格大小。

### CLS令牌、池化输出和寄存器令牌

什么是图像级表示？有三种选择共存：

1. `[CLS]`令牌。在补丁序列前添加一个可学习向量。经过所有变换器块后，CLS令牌的隐藏状态就是图像表示。继承自BERT。由原始ViT、CLIP使用。
2. 均值池化。对补丁令牌的输出隐藏状态取平均。由SigLIP、DINOv2和大多数现代VLM使用。
3. 寄存器令牌。Darcet等人(2023)观察到，没有显式汇入令牌训练的ViT会发展出高范数的“伪影”补丁，劫持自我注意力。添加4-16个可学习的寄存器令牌吸收这一负载，并提高密集预测（分割、深度）的质量。DINOv2和SigLIP 2都带有寄存器。

选择对下游任务很重要。CLS适用于分类。对于将补丁令牌输入LLM的VLM，则完全跳过池化——每个补丁成为LLM的输入令牌。寄存器在交接前被丢弃（它们是脚手架，而不是内容）。

### 预训练：监督、对比、掩码、自蒸馏

2020年的ViT使用JFT-300M上的监督分类进行预训练。很快被取代：

- CLIP (2021)：在400M对图像-文本上进行对比学习。第12.02课。
- MAE (2021, He等人)：掩码75%的补丁，重建像素。自监督，适用于纯图像。
- DINO (2021) / DINOv2 (2023)：使用师生模型进行自蒸馏，无标签，无描述。2023年的DINOv2 ViT-g/14是最强的纯视觉骨干，并且是“密集特征”用例的默认选择。
- SigLIP / SigLIP 2 (2023, 2025)：使用sigmoid损失和NaFlex实现原生宽高比的CLIP。2026年开放权重VLM（Qwen, Idefics2, LLaVA-OneVision）中占主导地位的视觉塔。

预训练的选择决定了骨干适合什么：CLIP/SigLIP用于与文本的语义匹配，DINOv2用于密集视觉特征，MAE作为下游微调的起点。

### 缩放定律

ViT缩放 (Zhai等人2022) 确立了ViT的质量在模型大小、数据大小和计算方面遵循可预测的规律。在固定计算下：
- 更大的模型 + 更多的数据 → 更好的质量。
- 补丁大小是序列长度与保真度之间的杠杆。补丁14（DINOv2/SigLIP SO400m的典型值）比补丁16每幅图像提供更多令牌；对OCR和密集任务更好，但速度较慢。
- 分辨率是另一个重要杠杆。从224到384再到512几乎总是有帮助，但FLOPs成本呈二次增长。

ViT-g/14 (1B参数，补丁14，分辨率224 → 256个令牌) 和SigLIP SO400m/14 (400M参数，补丁14) 是2026年开放权重VLM的两个主力编码器。

### ViT的参数数量

完整计算见`code/main.py`。对于ViT-B/16在224分辨率下：

```
patch_embed = 3 * 16 * 16 * 768 + 768  =  591k
cls + pos    = 768 + 197 * 768          =  152k
block        = 4 * 768^2 (QKVO) + 2 * 4 * 768^2 (MLP) + 2 * 2*768 (LN)
             = 12 * 768^2 + 3k          =  7.1M
12 blocks    = 85M
final LN    = 1.5k
total       ≈ 86M
```

在加载检查点之前，用这种方法估算每个ViT。骨干网(Backbone)的大小决定了任何下游VLM中的VRAM下限。

### 2026年生产配置

2026年大多数开源VLM配备的编码器是原生分辨率(NaFlex)下的SigLIP 2 SO400m/14。它具有：
- 4亿参数。
- 补丁大小14，默认分辨率384 → 每张图像729个补丁令牌(Patch Token)。
- 图像级任务使用均值池化(Mean Pool)；所有729个补丁都流入LLM进行VQA。
- 4个寄存器令牌(Register Token)，在交给LLM之前丢弃。
- 带有图像级缩放的2D-RoPE，用于原生宽高比。

## 使用它

`code/main.py` 是一个补丁分词器和几何计算器。它接受(图像 H, W, 补丁 P, 隐藏维度 D, 深度 L)并报告：

- 网格形状和补丁化后的序列长度。
- 一个合成的8x8像素玩具图像的令牌序列（遍历展平+投影路径）。
- 参数数量分解为补丁嵌入(Patch Embedding)、位置嵌入(Position Embedding)、Transformer模块和头部。
- 目标分辨率下每次前向传播的FLOPs。
- 跨ViT-B/16 @ 224、ViT-L/14 @ 336、DINOv2 ViT-g/14 @ 224、SigLIP SO400m/14 @ 384的比较表。

运行它。将参数数量与已发表的数据匹配。调整补丁大小和分辨率，感受令牌数量的代价。

## 发布

本节课产生`outputs/skill-patch-geometry-reader.md`。给定一个ViT配置（补丁大小、分辨率、隐藏维度、深度），它生成令牌数量、参数数量和VRAM估算及理由。每当你为VLM选择视觉骨干网(Visual Backbone)时，使用这个技能——它可以防止“令牌爆炸，我的LLM上下文被填满”的意外。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  补丁(Patch)  |  "16x16像素方形"  |  输入图像中固定大小的非重叠区域；成为一个令牌  |
|  补丁嵌入(Patch Embedding)  |  "线性投影"  |  一个共享的学习矩阵（或步长为P的Conv2d），将展平的补丁像素映射到D维向量  |
|  CLS令牌(CLS Token)  |  "类别令牌"  |  预置的可学习向量，其最终隐藏状态代表整个图像；2026年可选  |
|  寄存器令牌(Register Token)  |  "沉没令牌"  |  额外的可学习令牌，吸收ViT在预训练过程中产生的高范数注意力伪影  |
|  位置嵌入(Position Embedding)  |  "位置信息"  |  每个位置的向量或旋转，使序列顺序感知；2D-RoPE是现代默认选择  |
|  网格(Grid)  |  "补丁网格"  |  给定分辨率和补丁大小下的(H/P) x (W/P)二维补丁数组  |
|  NaFlex  |  "原生灵活分辨率"  |  SigLIP 2特性：单个模型无需重新训练即可服务于多种宽高比和分辨率  |
|  骨干网(Backbone)  |  "视觉塔"  |  预训练的图像编码器，其补丁令牌输出输入到VLM中的LLM  |
|  池化(Pooling)  |  "图像级摘要"  |  将补丁令牌转换为一个向量的策略：CLS、均值、注意力池化或基于寄存器  |
|  补丁14 vs 16  |  "更细 vs 更粗的网格"  |  补丁14每张图像产生更多令牌，OCR保真度更好，速度较慢；补丁16是经典默认  |
