# ColPali与原生视觉文档RAG

> 传统RAG将PDF解析为文本，切分成块，嵌入块，存储向量。每一步都会丢失信息：OCR丢失图表数据，分块破坏表格行，文本嵌入忽略图形。ColPali（Faysse等人，2024年7月）提出了一个更简单的问题：为什么非得提取文本？直接通过PaliGemma嵌入页面图像，使用ColBERT风格的后期交互进行检索，保留文档携带的所有布局、图形、字体和格式信息。已发布的基准测试：在视觉丰富的文档上，端到端准确率比文本RAG提高20-40%。ColQwen2、ColSmol和VisRAG扩展了这一模式。本课阅读原生视觉RAG论文，并构建一个类似ColPali的微型索引器。

**类型：** 构建
**语言：** Python（标准库，多向量索引器 + MaxSim评分器）
**前置条件：** 第11阶段（LLM工程 — RAG基础），第12阶段·05（LLaVA）
**时间：** ~180分钟

## 学习目标

- 解释双编码器检索（每个文档一个向量）与后期交互检索（每个文档多个向量）的区别。
- 描述ColBERT的MaxSim操作以及ColPali如何将其从文本令牌泛化到图像块。
- 构建一个微型ColPali风格的索引器：页面 → 块嵌入 → 对查询术语嵌入进行MaxSim → 前k个页面。
- 在发票/财务报告用例上比较ColPali + Qwen2.5-VL生成器与文本RAG + GPT-4。

## 问题

对PDF的文本RAG丢弃了文档的大部分内容。财务报告的第三季度收入增长通常出现在图表中；医疗报告的发现位于带注释的图像中；法律合同的签名块是一个布局事实，而不是文本事实。

文本RAG流水线：

1. PDF → 通过OCR/pdftotext转换为文本。
2. 文本 → 300-500令牌的块。
3. 块 → 双编码器嵌入（一个向量）。
4. 用户查询 → 嵌入 → 余弦相似度 → 前k个块。
5. 块 + 查询 → 大语言模型。

五个有损步骤。图表未被捕获。表格被跨块分割。多列布局被展平。图形注释消失。

ColPali的修复：跳过OCR，直接嵌入页面图像。使用ColBERT风格的后期交互进行检索，以便模型可以在查询时关注细粒度块。

## 核心概念

### ColBERT（2020年）

ColBERT（Khattab & Zaharia，arXiv:2004.12832）是一种文本检索方法。它不是每个文档一个向量，而是每个令牌一个向量。在查询时：

- 查询令牌获得自己的嵌入（N_q个向量）。
- 文档令牌获得嵌入（N_d个向量，通常缓存）。
- 分数 = 对查询令牌求和，每个查询令牌取与文档令牌余弦相似度的最大值：Σ_i max_j cos(q_i, d_j)。

这就是MaxSim操作。每个查询令牌“选择”其最佳匹配的文档令牌。最终分数是总和。

优点：召回率高，处理术语级语义。缺点：每个文档N_d个向量，存储成本高。

### ColPali

ColPali（Faysse等人，arXiv:2407.01449）将ColBERT模式应用于图像。

- 每个页面由PaliGemma（ViT + 语言）编码成块嵌入：每页N_p个向量。
- 每个用户查询（文本）被编码成查询令牌嵌入：N_q个向量。
- 分数 = Σ_i max_j cos(q_i, p_j)，即对查询文本令牌和页面图像块进行MaxSim。
- 通过总分检索前k个页面。

在文档摄取时：用PaliGemma嵌入每个页面，存储所有块嵌入。在查询时：嵌入查询令牌，计算对所有存储页面嵌入的MaxSim，返回前k个页面。

优点：端到端在视觉丰富文档上比文本RAG提高20-40%。每个块向量捕获局部布局和内容。

缺点：每页N_p个块 × 4字节浮点数 × D维向量 = 存储快速增长。通过PQ/OPQ量化缓解。

### ColQwen2和ColSmol

ColQwen2（illuin-tech，2024-2025）将PaliGemma替换为Qwen2-VL。更好的基础编码器，更好的检索。

ColSmol是用于本地/边缘的较小规模变体。约1B参数的ColSmol检索器可在消费级GPU上运行。

### VisRAG

VisRAG（Yu等人，arXiv:2410.10594）是一种不同的变体：不是对块进行MaxSim，而是用视觉语言模型将每个页面池化为单个向量，然后进行双编码器检索。更快的索引和更小的存储，但召回率较弱。

质量与成本的权衡：ColPali注重质量，VisRAG注重规模。

### M3DocRAG

M3DocRAG（Cho等人，arXiv:2411.04952）将多模态检索扩展到多页多文档推理。跨文档检索页面，为视觉语言模型组合多页上下文。

### ViDoRe — 基准测试

ColPali的配套基准测试。视觉文档检索评估。任务包括财务报告、科学论文、行政文档、医疗记录、手册。衡量指标：nDCG@5。

ColPali-v1 在 ViDoRe 上得分约为 80% nDCG@5；相同文档上的文本-RAG 得分约为 50-60%。

### 端到端 RAG 流水线

对于视觉原生 RAG：

1. 摄取：PDF → 页面图像 → PaliGemma 编码 → 存储所有补丁嵌入。
2. 查询：用户文本 → 查询令牌嵌入 → 对所有索引页面进行 MaxSim → 前 k 个页面。
3. 生成：前 k 个页面图像 + 查询 → VLM（Qwen2.5-VL 或 Claude）→ 答案。

任何地方都不需要 OCR。图表、图、字体、布局都流入答案。

### 存储计算

一份 50 页的财务报告，每页 729 个补丁，128 维嵌入：

- ColPali：50 * 729 * 128 * 4 字节 = 约 18 MB 原始，PQ 后约 4 MB。
- 文本-RAG：50 个块 * 768 维 * 4 字节 = 约 150 kB。

ColPali 每文档存储约大 30 倍。在规模下，OPQ / PQ 将其降低到约 5-10 倍，通常可接受。

### 文本-RAG 仍然胜出的情况

- 没有布局信号的纯文本文档（维基文章、聊天日志）。文本-RAG 更简单且存储更便宜。
- 数百万页的档案，存储成本占主导。
- 严格的法规要求提取 OCR 文本连同检索。

对于 2026 年所有其他情况——财务报告、科学论文、法律合同、医疗记录、UX 文档——视觉原生 RAG 胜出。

## 使用它

`code/main.py`：

- 玩具补丁编码器：将“页面”（特征向量的小网格）映射到补丁嵌入数组。
- MaxSim 评分器：计算查询令牌嵌入集与页面补丁集之间的 ColBERT 风格分数。
- 索引 5 个玩具页面，运行 3 个查询，返回带得分的前 k 个。

## 练习

1. 一份 200 页的年报，每页 729 个补丁，128 维嵌入，4 字节浮点数。计算原始存储和 PQ 压缩（8 倍）后的存储。

2. MaxSim 是 Σ_i max_j cos(q_i, p_j)。这个求和捕获了简单平均相似度所不能捕获的什么？

3. ColPali 将页面索引为补丁集。如果我们改为在单词级别索引（如 ColBERT 所做），会发生什么变化？权衡？

4. 为 100 万页的语料库设计端到端流水线，每查询延迟预算为 500 毫秒。选择 ColQwen2 / VisRAG 并证明其合理性。

5. 阅读 M3DocRAG (arXiv:2411.04952)。描述多页面注意力模式及其与单页面 ColPali 检索的不同。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  后期交互  |  "ColBERT 风格"  |  使用每令牌或每补丁嵌入 + MaxSim 的检索，而不是单个文档向量  |
|  MaxSim  |  "补丁上取最大"  |  对于每个查询令牌，选择最高相似度的文档令牌；跨查询求和  |
|  双编码器  |  "单向量"  |  每文档一个向量；更快但失去粒度  |
|  多向量  |  "每文档多向量"  |  每文档/页面存储 N_p 个向量；存储成本增加但召回率提高  |
|  补丁嵌入  |  "页面特征"  |  来自 VLM 编码器的每个图像补丁的一个向量，每页面缓存  |
|  ViDoRe  |  "视觉文档基准"  |  ColPali 的视觉文档检索基准套件  |
|  PQ 量化  |  "乘积量化"  |  保持向量相似性同时将存储缩小约 8 倍的压缩  |
