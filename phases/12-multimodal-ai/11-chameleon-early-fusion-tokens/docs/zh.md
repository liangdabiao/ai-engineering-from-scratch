# 变色龙（Chameleon）与早期融合纯令牌多模态模型

> 到目前为止，我们见到的每一个视觉语言模型都将图像和文本分开处理。视觉令牌来自视觉编码器，流入投影器，然后在大型语言模型内部与文本相遇。视觉和文本词汇从未重叠。变色龙（Meta，2024年5月）提出了一个问题：如果它们重叠会怎样？训练一个VQ-VAE，将图像转换为来自共享词汇的离散令牌序列。现在每个多模态文档都是一个序列——文本令牌和图像令牌交错排列，使用单一的回归损失。副作用：模型可以生成混合模态输出——在单次推理调用中交替生成文本令牌和图像令牌。本节课将解读早期融合论文，并端到端地构建一个简化版本。

**类型：** 构建
**语言：** Python（标准库，VQ-VAE分词器+交错解码器）
**前提条件：** 阶段12·05，阶段8（生成式AI）
**时间：** 约180分钟

## 学习目标

- 解释为什么共享词汇和单一损失改变了模型的能力。
- 描述VQ-VAE如何将图像分词化为与Transformer的下一令牌目标兼容的离散序列。
- 列出变色龙的训练稳定性技巧：QK-Norm、失活放置、层归一化顺序。
- 比较变色龙与BLIP-2的Q-Former方法，并描述每种方法何时是合适的选择。

## 问题

基于适配器的视觉语言模型（LLaVA、BLIP-2、Qwen-VL）将文本和图像视为两种不同的东西。文本令牌通过`embed(text_token)`；图像通过`visual_encoder(image) → projector → ... pseudo_tokens`。模型有两个输入路径，在中间合并。

三个后果：

1. 大型语言模型只能消费图像，不能生成图像。输出仅为文本。
2. 混合模态文档（例如文章中交替的段落和图像）处理起来很别扭——要么在模型外部解析多模态输入，要么进行链式生成。
3. 分布不匹配。视觉令牌和文本令牌位于隐藏空间的不同区域，导致微妙的对齐问题。

变色龙否定了这一前提：图像只是来自共享词汇的离散令牌序列。在交错文档上训练模型，使用一个损失、一个回归解码器，就能免费获得混合模态生成能力。

## 核心概念

### VQ-VAE作为图像分词器

分词器是一个向量量化变分自编码器。其架构：

- 编码器：卷积神经网络+视觉Transformer，将图像映射成空间特征图，例如32x32的特征，维度为256。
- 码本：一个由K个向量（变色龙使用8192个）组成的学习词汇表，维度也是256。
- 量化：对于每个空间特征，通过L2距离查找最近的码本条目。用整数索引替换连续特征。
- 解码器：卷积神经网络，将量化后的特征映射回像素。

训练：变分自编码器重构损失+承诺损失+码本损失。码本索引形成了图像的离散字母表。

对于变色龙：一幅图像变成32*32=1024个令牌，这些令牌来自一个包含8192个词汇的词汇表。与文本令牌（来自大型语言模型的BPE词汇表，例如32000个）拼接。最终词汇表：40192。Transformer看到一个序列、一个损失。

### 共享词汇

变色龙的词汇结合了文本令牌、图像令牌和模态分隔符。每个令牌有一个单独的ID。输入嵌入层将每个ID映射到D维隐藏向量。输出投影将隐藏向量映射回词汇表logits。Softmax选择下一个令牌，无论是什么模态。

分隔符很重要：`<image>`和`</image>`标记包围图像令牌序列。在生成时，如果模型生成了`<image>`，下游软件就知道接下来的1024个令牌是VQ索引，需要发送给解码器进行像素渲染。

### 混合模态生成

推理过程是在共享词汇表上的下一令牌预测。示例提示：“画一只猫并描述它。”变色龙生成：

```
<image> 4821 1029 2891 ... (1024 image tokens) </image>
The cat is orange, sitting on a windowsill...
```

模型自主选择顺序——它可能先生成图像再生成文本，或先文本后图像，或交错进行。相同的解码器、相同的损失。

与适配器视觉语言模型（仅文本生成）相比，变色龙重新提出了模型输出模态的问题。

### 训练稳定性——QK-Norm、失活、层归一化顺序

早期融合训练在大规模下不稳定。变色龙的论文记录了三个技巧：

- QK-Norm。在注意力内部，点积之前对查询和键投影应用层归一化。防止深度下logit幅度爆炸。被多个2024年后的大型模型使用。
- 失活放置。在每个残差加法之后都进行失活，而不仅仅是在注意力和MLP之后。当来自图像令牌的梯度可能占主导时，需要更强的正则化。
- 层归一化顺序。残差分支上的预层归一化（标准），再加上最后一个块的跳跃连接上的额外层归一化。稳定了最后一层的梯度流。

没有这些技巧，34B参数的变色龙训练会在多个检查点发散。有了它们，模型收敛。训练方法本身与架构同等重要。

### 分词器的重构上限

VQ-VAE是有损的。对于每个512x512图像，使用8192个码本条目和1024个令牌，重构PSNR上限约为26-28 dB。这足以进行可识别的图像生成，但明显比连续空间扩散（Stable Diffusion 3达到32+ dB）差。

分词器是瓶颈。更好的分词器（MAGVIT-v2、IBQ、SBER-MoVQGAN）提高了上限。Emu3（第12.12课）仅通过更好的分词器就达到了SDXL级生成质量。

### 变色龙 vs BLIP-2 / LLaVA

变色龙（早期融合，共享词汇）：
- 一个损失，一个解码器。
- 生成混合模态输出。
- 分词器是质量上限。
- 缺点：推理时每个生成的图像都需要VQ-VAE解码器。

BLIP-2 / LLaVA（晚期融合(late fusion)，分离塔(separate towers)）：
- 视觉输入，仅文本输出。
- 复用预训练的大语言模型。
- 无词元器瓶颈理解。
- 低成本：单次前向传播。

根据任务选择。如果需要图像生成，选择Chameleon系列。如果只需要理解，适配器视觉语言模型(adapter-VLM)更简单，且能复用更多预训练计算资源。

### Fuyu和AnyGPT

Fuyu（Adept，2023）是一种相关方法：完全跳过独立的视觉编码器，将原始图像块直接作为词元通过大语言模型的输入投影送入，无需词元器。比Chameleon更简单，但失去了共享词汇表输出生成的能力。

AnyGPT（Zhan等人，2024）将Chameleon扩展到四种模态：文本、图像、语音、音乐。每种模态使用相同的VQ-VAE技巧，共享Transformer。任意到任意生成。更多内容在第12.16课中。

## 使用它

`code/main.py` 构建了一个玩具端到端早期融合(early-fusion)模型：

- 一个微型VQ-VAE风格的量化器，将8x8块映射到码本索引(codebook indices)(K=16)。
- 一个共享词汇表，包含（文本ID 0..31）+（图像ID 32..47）+（分隔符48，49）。
- 一个在合成标题+图像词元序列上训练的玩具自回归解码器（二元语法表）。
- 采样循环，根据提示交替生成文本和图像词元。

代码有意保持Transformer微小（二元语法），以便你能端到端追踪信号流。

## 练习

1. Chameleon使用K=8192个码本条目(codebook entries)和每张512x512图像1024个词元。估算与24位RGB图像相比的压缩比。它是有损的吗？损失程度如何？

2. 在相同VQ-VAE密度下，一张4K图像（3840x2160）会产生多少个图像词元？Chameleon风格的模型能否在一次推理调用中生成4K图像？哪个最先失效——上下文(context)、词元器质量(tokenizer quality)还是KV缓存(KV cache)？

3. 用纯Python实现QK-Norm。给定64维的查询(query)和键(key)，展示LayerNorm前后的点积。为什么幅度控制在深层很重要？

4. 阅读Chameleon第2.3节关于训练稳定性的内容。描述论文中在34B模型未使用QK-Norm时观察到的确切失效模式。什么是“范数爆炸”特征？

5. 扩展玩具解码器，使其在仅文本提示下生成混合模态响应。给定训练数据分布60%文本优先/40%图像优先，测量模型选择图像优先 vs 文本优先的频率。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  早期融合(early fusion)  |  “统一词元(unified tokens)”  |  图像从一开始就转换为离散词元，共享Transformer词汇表 |
|  VQ-VAE  |  “图像词元器(image tokenizer)”  |  卷积神经网络(CNN)+视觉Transformer(ViT)+码本(codebook)，将图像映射为Transformer可预测的整数索引 |
|  共享词汇表(shared vocabulary)  |  “一个字典(one dictionary)”  |  覆盖文本+图像+模态分隔符的单一词元ID空间 |
|  QK-Norm  |  “注意力稳定器(attention stabilizer)”  |  在查询和键的点积之前应用层归一化(LayerNorm)，防止范数爆炸(norm blowup) |
|  混合模态生成(mixed-modality generation)  |  “文本+图像输出(text + image output)”  |  一次性自主生成交错文本和图像词元的推理 |
|  码本大小(codebook size)  |  “K个条目(K entries)”  |  VQ-VAE可以量化到的离散向量数量；在压缩与保真度之间权衡 |
|  词元器上限(tokenizer ceiling)  |  “重建极限(reconstruction limit)”  |  解码VQ词元可达到的最佳峰值信噪比(PSNR)；限制了模型的图像质量 |
