# 多令牌预测 (Multi-Token Prediction, MTP)

> 从 GPT-2 到 Llama 3 的每个自回归 LLM 都在每个位置上训练一个损失：预测下一个令牌。DeepSeek-V3 在每个位置上增加了一个损失：预测再下一个令牌。额外的 140 亿参数（在 671B 模型上）通过梯度流蒸馏回主模型，训练后的 MTP 头在推理时被重新用作推测解码草稿生成器，接受率达 80% 以上。由此免费获得了 1.8 倍的生成吞吐量。本节课程将根据 DeepSeek 技术报告构建顺序 MTP 模块，计算损失和共享头参数布局，并解释为什么 MTP 保持因果链，而 Gloeckle 等人的原始并行 MTP 打破了它。

**类型：** 构建
**语言：** Python (标准库)
**前置条件：** 阶段 10 · 04（预训练一个小型 GPT）、阶段 10 · 15（推测解码）
**时间：** 约 60 分钟

## 学习目标

- 阐述 MTP 训练目标，并推导跨预测深度的联合损失。
- 解释 Gloeckle 等人 (2024) 的并行 MTP 头与 DeepSeek-V3 的顺序 MTP 模块之间的区别，以及为什么顺序设计能保持因果链。
- 计算向预训练运行添加 MTP 模块带来的参数和内存开销。
- 从头实现一个 MTP 模块：共享嵌入、每深度 Transformer 块、投影和共享输出头。

## 问题

下一个令牌预测是标准 LLM 训练目标。每个隐藏状态都被监督去预测恰好一个东西：紧随其后的那个令牌。这是一个出奇弱的信号。一个序列中的大部分信息延伸超过一个令牌——结构、连贯性、事实性、算术流程。模型必须通过累积数万亿个令牌上的许多单令牌信号来学习这些。

MTP 提出：如果每个隐藏状态都被监督去同时预测多个未来令牌会怎样？Gloeckle 等人（Meta，2024）证明了这有帮助。他们的实现是在主干之上放置了几个独立的输出头，每个头预测不同的偏移量。并行、简单，但这些头看到的是相同的隐藏状态，没有任何层次化精炼——而且预测没有因果链，因此它们不能用于推测解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为顺序模块，在每个预测深度保持因果链。模型从 `t+1` 预测 `h_i^(0)`，然后从结合了 `h_i^(0)` 与 `E(t+1)` 嵌入的新隐藏状态 `h_i^(1)` 预测 `t+2`，依此类推。每个深度都有自己的小型 Transformer 块。共享嵌入和共享输出头使参数开销保持适度。在 DeepSeek-V3 的规模下，MTP 模块在 671B 主模型权重之上额外增加了 140 亿参数。这 2% 的开销换来了更密集的训练信号以及在推理时现成的推测解码草稿生成器。

本节课程从头构建一个单一的 MTP 模块和 D 深度损失。数学很简洁。实现代码约 150 行。

## 核心概念

### 顺序式 MTP 方案

DeepSeek-V3 在主模型之上添加了 `D` 个 MTP 模块。每个模块 `k`（对于 `k = 1..D`）预测深度 `k` 处的令牌——也就是说，给定一个前缀直到位置 `i` 的 `t_{i+k}`。

模块 `k` 包含：

- 一个带有自身注意力和 MLP 的 Transformer 块 `T_k`。
- 一个投影矩阵 `T_k`，用于组合前一深度的隐藏状态与下一深度真实令牌的嵌入。
- 共享嵌入 `T_k`（与主模型相同）。
- 共享输出头 `T_k`（与主模型相同）。

在训练时，对于前缀直到位置 `i`，每深度的隐藏状态为：

```
h_i^(0) = main model backbone at position i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

每深度的预测为：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

每深度的损失是针对真实令牌 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的权重因子——DeepSeek-V3 在前 10% 训练中采用 0.3，之后采用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序式而非并行式

Gloeckle 的原始并行 MTP 有 D 个输出头，每个直接应用于 `h_i^(0)`。每个头从相同的骨干隐藏状态预测 `t_{i+k}`。这样训练没问题，但预测并不相互条件。你不能用 `head_1` 的输出来帮助 `head_2`——这些头是并行触发的。

DeepSeek-V3 的顺序设计从 `h_i^(k-1)` 加上实际下一个令牌嵌入 `E(t_{i+k})` 构建 `h_i^(k)`。这保持了因果链：要预测 `t_{i+k+1}`，深度 `t_{i+k}` 处的模块可以看到 `k+1` 处的内容。这在结构上与自回归解码器消费自身输出的方式相同——使得 MTP 模块可直接用作推测解码草稿生成器。

推理时：将 `h_i^(k-1)` 和草稿 `t_{i+k}` 输入模块 `k+1`，得到 `t_{i+k+1}` 的预测。重复。这完全是一个 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率超过 80%，速度提升约 1.8 倍。

### 参数统计

对于隐藏层大小为 `h`、词汇量为 `V` 的模型：

- 主模型：数十亿参数，加上一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的头。无额外参数。
- 共享嵌入：复用主模型的嵌入。无额外参数。
- 每个 MTP 模块：
  - 投影 `V * h`：`M_k`。
  - Transformer 块 `V * h`：注意力（MHA 为 `M_k`）加 MLP（对于比例为 8/3 的 SwiGLU 通常是 `(2h) * h = 2h^2`）。每个块约 `T_k`。

每个模块的额外参数总计：`~14h^2`。对于 DeepSeek-V3 的 `h = 7168`，D = 1 个模块：纸质 `~14 * 7168^2 = ~720M` 参数。DeepSeek-V3 报告为 140 亿——差异主要在于 MTP 模块中的专家层也是 MoE。

### 推测解码的回报

在预训练期间，MTP 模块会使训练变慢约 10%（更多前向计算，额外损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到 D+1 个监督目标。在 DeepSeek-V3 的消融实验中，对 MMLU、GSM8K、MATH、HumanEval 的测量效果是一致的几个百分点的提升。

2. 推理时免费的推测解码草稿。MTP 模块已经训练好来预测接下来的几个令牌。作为草稿网络重新利用，它提供了超过 80% 的接受率。在这个水平上，N=3 或 N=5 的推测解码带来了 1.8 倍的吞吐量。10% 的训练时间成本在第一次运行推理时就得到回报。

### 与 EAGLE 的关系

EAGLE在预训练之后单独训练一个小型草稿模型。MTP将草稿融入预训练。这两种方法通过不同的流程实现了相似的接受率：

| 维度 | EAGLE-3 | MTP (DeepSeek-V3) |
|-----------|---------|------------------|
| 训练时机 | 预训练之后 | 预训练期间 |
| 与现有权重的向后兼容性 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2个Transformer层 | 1个Transformer块+投影 |
| 接受率 | 0.88-0.92 | 深度1下0.80+ |
| 加速之外的收益 | 仅投机解码 | 更密集的训练信号+加速 |

## 动手构建

`code/main.py` 端到端构建单个MTP模块：共享嵌入、投影、Transformer块、共享输出头。然后计算短合成序列上每个深度的交叉熵损失，并按组件打印参数数量。32个token的玩具词汇表使数字可读。

### 步骤1：共享嵌入表

一个`vocab_size x hidden`表被主模型以及每个深度的每个MTP模块使用。不是第二个副本——字面上是同一个张量。

### 步骤2：每个深度的组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # concat along feature dim, then project down to hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # vector addition stand-in
    projected = matvec(M_k, concat)
    return projected
```

真正的DeepSeek-V3将两个RMSNormed向量拼接成`[2h]`，并用`h x 2h`矩阵进行投影。玩具版本为了标准库简洁性使用向量相加。

### 步骤3：深度k的Transformer块

自注意力加MLP。在玩具版本中，单层线性注意力块和SwiGLU MLP保持了结构可见性而不使用numpy。

### 步骤4：共享输出头

重用主模型的输出投影。在词汇表上的logits。

### 步骤5：每个深度的损失

softmax(logits)与偏移`k`处的真实token之间的交叉熵。使用`lambda / D`缩放因子跨深度聚合。

### 步骤6：参数统计

打印总参数数量、共享（嵌入、头）数量以及每个模块的额外数量。显示MTP额外参数与主模型大小的比率。

## 使用它

MTP已集成到DeepSeek-V3（2024年12月）和DeepSeek-R1系列中。在推理时：

- DeepSeek自己的服务栈直接使用MTP模块作为投机解码器。
- vLLM和SGLang截至2026年4月有DeepSeek-V3 MTP的集成路径。
- AMD的ROCm SGLang教程展示了特定的MTP投机解码配置，在V3检查点上测得1.8倍的加速。

在新的预训练运行中使用MTP的时机：

- 你控制完整的预训练流程，并希望获得更密集的训练信号。
- 你确定将大规模部署模型，并希望免费获得投机解码。
- 你的隐藏层大小至少为4096。在1B规模下，开销的负面影响大于收益。

何时不使用：

- 微调现有的预训练密集模型。MTP模块未被训练。
- 研究模型，你希望有一个干净的基线进行比较。MTP改变了架构。

## 练习

1. 运行`code/main.py`。显示每个深度的损失随着合成信号增强而单调递减。修改合成使用固定模式，并验证深度1和深度2的损失都收敛。

2. 计算密集70B模型（隐藏8192，80层）使用D=1 MTP模块的参数开销。与DeepSeek-V3报告的14B开销进行比较。解释为什么DeepSeek的数字更高：MTP Transformer块继承了相同的MoE结构，增加了每个模块的参数数量。

3. 在玩具版本中实现D=2：添加第二个MTP模块，接收h^(1)并预测`t_{i+2}`。验证联合损失和参数统计与DeepSeek论文中的公式19-21匹配。

4. 将玩具版本切换为并行MTP（Gloeckle风格）：在主模型隐藏状态之上添加D个输出头，每个预测不同的偏移。在同一合成信号上，测量每个深度的损失与顺序版本的对比。对于k>1，顺序版本应该产生更低的深度k损失，因为它依赖于中间预测。

5. 将训练好的MTP模块用作EAGLE风格的草稿：在推理时调用模块k提出`t_{i+k}`。在保留序列上测量这些草稿token相对于主模型预测的接受率。如果在玩具数据上达到50%以上，就重现了经验性的MTP作为草稿的性质。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  MTP模块  |  "额外损失块"  | 一个小型Transformer块加上投影，用于预测主模型之前`k`个位置的token  |
|  预测深度  |  "哪个偏移"  | 整数`k`，使得模块`k`从前缀通过位置`i`预测`t_{i+k}`  |
|  并行MTP  |  "Gloeckle风格"  | 在相同骨干隐藏状态上的D个独立头，无条件链  |
|  顺序MTP  |  "DeepSeek-V3风格"  | 每个模块以先前深度的隐藏状态加上下一个token的嵌入为条件；保留因果链  |
|  共享输出头  |  "重用主头"  | MTP模块调用主模型的LM head，而非单独的输出投影  |
|  共享嵌入  |  "重用主表"  | 各处使用相同的词汇嵌入表；无重复参数  |
|  投影矩阵M_k  |  "合并隐藏状态+下一个token"  | 一个`h x 2h`线性层，将先前的隐藏状态和目标token嵌入折叠为下一个深度的输入  |
|  联合损失L_MTP  |  "平均额外损失"  | 每个深度的交叉熵损失的算术平均值，按`lambda`缩放  |
|  深度1处的接受率  |  "MTP草稿正确的频率"  | D=1 MTP模块的top-1预测等于主模型top-1预测的比率；在DeepSeek-V3上超过80%  |
|  Lambda加权  |  "额外损失重要性"  | 每深度缩放因子；训练开始时0.3，后来在DeepSeek-V3上为0.1  |
