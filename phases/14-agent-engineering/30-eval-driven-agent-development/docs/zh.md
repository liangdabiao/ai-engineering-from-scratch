# 评估驱动的智能体开发

> Anthropic 的指导方针：“从简单的提示开始，通过全面评估优化，只在必要时添加多步骤代理系统。”评估不是最后一步。它是驱动第14阶段每个其他选择的外循环。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第14阶段全部内容。
**时间：** 约60分钟

## 学习目标

- 命名三个评估层——静态基准、自定义离线、在线生产——并说明各自用途。
- 解释评估器-优化器紧密循环。
- 描述2026年最佳实践：评估与代码共存，在CI中运行，门控PR。
- 将第14阶段的每一课与其产生的评估案例联系起来。

## 问题

代理能通过演示。但在生产环境中失败的方演示无法预测。基准回答的是“这个模型是否广泛胜任？”而不是“这个代理是否为我的产品推送正确的补丁？”答案是：三个评估层持续运行，每个护栏和学到的规则都映射到评估案例。

## 核心概念

### 三个评估层

1. **静态基准** —— SWE-bench Verified 用于代码（第19课），WebArena/OSWorld 用于浏览/桌面（第20课），GAIA 用于通用型（第19课），BFCL V4 用于工具使用（第06课）。用于跨模型比较和回归门控。污染是真实存在的：SWE-bench+ 发现32.67%的解决方案泄露。始终报告 Verified / +-audited 分数。

2. **自定义离线评估** —— 你产品的形态：
   - LLM 作为评判者（Langfuse, Phoenix, Opik —— 第24课）。
   - 基于执行的评估（运行补丁，检查测试）。
   - 基于轨迹的评估（比较动作序列与黄金序列；OSWorld-Human 显示顶级代理是黄金序列的1.4-2.7倍）。

3. **在线评估** —— 生产环境：
   - 会话回放（Langfuse）。
   - 护栏触发的警报（第16、21课）。
   - 每步成本/延迟跟踪（第23课 OTel spans）。

### 评估器-优化器（Anthropic）

紧密循环：

1. 提议生成输出。
2. 评估器判断。
3. 直到评估器通过时才精炼。

这是 Self-Refine（第05课）的泛化。任何你关心的代理流程都可以包装在评估器-优化器中以提高可靠性。

### 2026年最佳实践

- 评估与代码共存。
- 在每个PR的CI中运行。
- 根据评估分数门控合并（例如“与主分支相比无回归>5%”）。
- 每个护栏都映射到一个评估案例。
- 每个学到的规则（Reflexion, pro-workflow learn-rule）都映射到一个失败案例。

### 评估驱动开发的失败之处

- **无基线。** 没有已知最新正确结果的评估是不可读的。存储基线。
- **无依据的LLM评判。** 评判者也会产生幻觉。CRITIC模式（第5课）——评判者依据外部工具。
- **过度拟合评估。** 针对评估进行优化会偏离生产用途。轮换用例。
- **不可靠的评估。** 非确定性用例导致误报。固定随机种子，快照状态。

## 动手构建

`code/main.py` 是一个标准库评估框架：

- 用例注册表，带有分类（基准测试、自定义、在线）。
- 一个受测的脚本化智能体。
- 评估器-优化器循环：提议、评判、优化，直到通过或达到最大轮次。
- CI门禁：汇总通过率+与基线的回归对比。

运行它：

```
python3 code/main.py
```

输出：每个用例的通过/失败、回归标志、CI门禁裁决。

## 使用它

- 在与你的智能体代码相同的仓库中编写评估用例。
- 通过CI在每个PR上运行它们。
- 在回归时让构建失败。
- 随时间跟踪通过率。
- 将每个生产故障关联到一个新用例。

## 发布

`outputs/skill-eval-suite.md` 为一个智能体产品构建了一个三层评估套件，带有CI门禁和回归跟踪。

## 练习

1. 选取你生产环境中的一个故障。编写一个能复现它的评估用例。你的智能体现在能通过它吗？
2. 为你的领域构建一个LLM评判标准，包含三个维度（事实性、语气、范围）。对50个会话进行评分。
3. 将评估套件接入CI。在大于等于5%回归时让构建失败。
4. 添加一个轨迹效率指标：你的智能体花费的步数与黄金轨迹相比如何？
5. 将阶段14的每一课映射到你的评估套件中的一个用例。有缺失的吗？那就是需要填补的差距。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  静态基准测试  |  "现成评估"  |  SWE-bench, GAIA, AgentBench, WebArena, OSWorld  |
|  自定义离线评估  |  "领域评估"  |  在你产品形态上的LLM-as-judge / exec / 轨迹  |
|  在线评估  |  "生产评估"  |  会话回放、护栏警报、成本/延迟跟踪  |
|  评估器-优化器  |  "提议-评判-优化"  |  迭代直到评判通过  |
|  CI门禁  |  "合并阻挡器"  |  在评估回归时让构建失败  |
|  基线  |  "已知最新正确结果"  |  用于检测回归的参考分数  |
|  轨迹效率  |  "步数比黄金"  |  智能体步数除以人类专家最小步数  |
