# Tree of Thoughts 与 LATS：深思熟虑的搜索

> 单一思维链轨迹没有回溯余地。ToT（Yao 等人，2023）将推理转化为树，并对每个节点进行自我评估。LATS（Zhou 等人，2024）将 ToT 与 ReAct 和 Reflexion 统一在蒙特卡洛树搜索下。24点游戏从4%（思维链）提升到74%（ToT）；LATS 在 HumanEval 上达到 92.7% pass@1。

**类型：** 构建
**语言：** Python（标准库）
**前置条件：** 阶段14·01（智能体循环），阶段14·03（Reflexion）
**时间：** ~75分钟

## 学习目标

- 将推理框架化为搜索：节点是“想法(Thoughts)”，边是“扩展(Expansions)”，价值是“有多大希望”。
- 实现一个标准库风格的 ToT 广度优先搜索树，并带有自我评估评分。
- 扩展为一个玩具 LATS 蒙特卡洛树搜索循环，包含选择(Select)/扩展(Expand)/模拟(Simulate)/反向传播(Backpropagate)。
- 判断何时搜索值得付出代币倍数（24点游戏、代码生成），以及何时单一轨迹就足够（简单问答）。

## 问题

思维链是线性行走。如果第一步错了，后续每一步都基于错误的前提。在24点游戏（使用四个数字通过加减乘除得到24）中，GPT-4 思维链准确率只有4%。模型过早选择了错误的子表达式，且无法恢复。

推理需要的是能够提出多个候选方案，评估它们，选择有希望的，并在遇到死胡同时回溯。这就是搜索。Tree of Thoughts 和 LATS 是两个经典形式。

## 核心概念

### Tree of Thoughts（Yao 等人，NeurIPS 2023）

每个节点是一个连贯的中间步骤（一个“想法”）。每个节点可以扩展出 K 个子想法。LLM 通过评分提示对每个节点进行自我评估。搜索探索树——广度优先、深度优先或束搜索。

```
                     (root: "find 24 from 4 6 4 1")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- Score: HIGH
              /   \              |                  |
          ...    ...          ...                finish
```

自我评估是关键部分。论文展示了三种变体：`sure / likely / impossible` 分类、`1..10` 数值评分和候选者投票。三者都在24点游戏上大幅超越思维链（GPT-4 从4%提升到74%）。

### LATS（Zhou 等人，ICML 2024）

LATS 将 ToT、ReAct 和 Reflexion 统一在蒙特卡洛树搜索下。LLM 扮演三个角色：

- **策略**：提出候选下一步动作（ReAct 风格）。
- **价值函数**：对部分轨迹评分（ToT 风格自我评估）。
- **自我反思器**：失败时，写自然语言反思（Reflexion 风格）并用于重新播种未来 rollout。

环境反馈（观察）融入价值函数，使搜索基于真实工具结果而非模型意见。论文发表时结果：HumanEval pass@1 92.7%（GPT-4，SOTA），WebShop 平均 75.9（GPT-3.5，接近基于梯度的微调）。

### 蒙特卡洛树搜索，最小化

每次迭代的四个阶段：

1. **选择**——使用 UCT（上限置信区间树）从根节点走到叶子节点。
2. **扩展**——通过策略生成 K 个子节点。
3. **模拟**——从子节点使用策略 rollout，用价值函数（或环境奖励）对叶子评分。
4. **反向传播**——更新路径上的访问次数和价值估计。

UCT 公式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。第一项是利用，第二项是探索。针对任务调整 `c`。

### 成本现实

搜索会爆炸式消耗代币。ToT 在24点游戏上使用的代币是思维链的100–1000倍。LATS 类似。这不是免费的；将搜索保留用于：

- 单一轨迹明显不足的任务（24点游戏、复杂代码）。
- 墙钟时间不如正确性重要的任务。
- 有廉价且可靠价值函数的任务（代码的单元测试、数学的明确目标）。

如果你的任务有单一正确答案和嘈杂评估器，搜索往往会使情况更糟——它会找到一个得分高但错误的答案。

### 2026年定位

大多数生产环境智能体不运行 LATS。它们运行带有工具验证的 ReAct（CRITIC，第05课）。搜索出现在专业领域：

- 以测试作为价值函数的编码智能体（HumanEval 风格）。
- 探索多个查询路径的深度研究智能体。
- LangGraph 子图中的规划密集型工作流。

AlphaEvolve（第11课）是2025年的极端：对代码的进化搜索，机器可检查的适应度，前沿收益（56年来首次4x4矩阵乘法改进）。

## 练习

1. 运行玩具 LATS，UCT c=0.1 vs c=2.0。跟踪有何变化？
2. 将价值函数替换为更嘈杂的评分器（添加随机抖动）。蒙特卡洛树搜索还能找到最佳叶子吗？它能容忍的最小信噪比是多少？
3. 实现束搜索 ToT（每层保留 top-k）并与广度优先比较。在严格的代币预算下哪个更好？
4. 阅读 LATS 第5.1节。复现 HumanEval 的 rollout 计数：要达到报告的 pass@1 需要多少次 rollout？
5. 阅读 LATS 论文关于“何时 LATS 帮助较小”的讨论。写一段决策规则，将任务形状映射到搜索策略。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
| 思维树(ToT)  |  "分支思维链(CoT)"  |  Yao等人 — 带有自我评估的思维树节点 |
| LATS  |  "用于大语言模型的蒙特卡洛树搜索(MCTS)"  |  Zhou等人 — 将ToT、ReAct和Reflexion统一在MCTS框架下 |
| UCT  |  "上置信界(Upper confidence bound)"  |  平衡利用(Q)和探索(ln N / n)的选择公式 |
| 值函数(Value function)  |  "该状态有多好"  |  通过提示大语言模型得到的得分或环境奖励；用于反向传播 |
| 策略(Policy)  |  "动作提议者"  |  类似ReAct的生成器；产生候补的下一个想法/动作 |
| 推演(Rollout)  |  "模拟轨迹"  |  使用策略从节点走到叶节点，用值函数评分 |
| 反向传播(Backpropagate)  |  "更新祖先节点"  |  将叶节点的奖励沿路径向上传递，更新访问次数和Q值 |
| 搜索成本(Search cost)  |  "标记爆炸"  |  在"24点"游戏上比CoT高100-1000倍；采用前需预算 |
