# 为何选择Transformer——RNN的问题

> RNN逐个处理token，而Transformer一次性处理所有token。这一架构抉择改变了2017年后深度学习的所有缩放曲线。

**类型：** 学习
**语言：** Python
**前置要求：** 阶段3（深度学习核心）、阶段5·09（序列到序列）、阶段5·10（注意力机制）
**时间：** ~45分钟

## 问题

在2017年之前，地球上每个最先进的序列模型——语言、翻译、语音——都是循环神经网络。LSTM和GRU在翻译基准测试中称霸了五年。它们是当时唯一可用的工具。

它们有三个致命弱点。顺序计算意味着无法沿时间轴并行化：token `t+1`需要来自token `t`的隐藏状态。一个1024 token的序列意味着在每周期可进行1,000,000次浮点运算的GPU上需要1024个串行步骤。在专为并行设计的硬件上，训练墙钟时间随序列长度线性增长。

梯度消失意味着50个token之前的信息已经通过50个非线性变换压缩。门控循环单元（LSTM、GRU）减轻了这种压缩，但从未消除。长距离依赖——"我去年夏天在去京都的飞机上读的那本书是……"——经常失败。

固定宽度的隐藏状态意味着编码器在解码器看到任何内容之前将整个源序列压缩成一个单一向量。无论源序列是5个token还是500个token，瓶颈的形状都一样。

2017年的论文《Attention Is All You Need》提出了一个激进的想法：完全放弃循环。让每个位置并行地关注其他所有位置。通过一次大的矩阵乘法进行训练，而不是1024次顺序乘法。

到2026年，这一结果主导了所有模态。语言（GPT-5、Claude 4、Llama 4）、视觉（ViT、DINOv2、SAM 3）、音频（Whisper）、生物学（AlphaFold 3）、机器人学（RT-2）。相同的模块，不同的输入。

## 核心概念

![RNN sequential compute vs Transformer parallel attention](../assets/rnn-vs-transformer.svg)

**循环作为瓶颈。** RNN计算`h_t = f(h_{t-1}, x_t)`。每一步依赖前一步。在`h_5`之前无法计算`h_4`。在现代拥有10,000+并行核心的GPU上，对于长序列，这浪费了99%的硅片。

**注意力作为广播。** 自注意力同时计算每对`(i, j)`的`output_i = sum_j(a_ij * v_j)`。整个N×N注意力矩阵在一次批量矩阵乘法中填充。没有任何步骤依赖其他步骤。GPU喜欢它。

**加速不是常数。** 它是`O(N)`串行深度与`O(1)`串行深度之间的差异。实际上，在同等硬件上，当N=512时，Transformer每个epoch的训练速度比RNN快5-10倍，并且随着序列长度增加，差距持续扩大，直到达到注意力的`O(N²)`内存墙（后来被Flash Attention修复——见第12课）。

**Transformer的代价。** 注意力的内存随`O(N²)`缩放。对于2K上下文，没问题。对于128K上下文，需要滑动窗口、RoPE外推、Flash Attention分块或线性注意力变体。循环在时间和内存上都是`O(N)`；Transformer用时间换内存，再通过并行性赢回时间。

**归纳偏置的转变。** RNN假设局部性和最近性。Transformer不假设任何东西——每对都是注意力的候选者。这就是为什么Transformer需要更多数据才能训练好，但一旦拥有足够数据就能扩展得更远。Chinchilla（2022）将其形式化：给定足够的token，相同参数量的Transformer总是优于RNN。

## 动手构建

这里没有神经网络——我们通过数值模拟核心瓶颈，让您在自己的笔记本上感受差距。

### 步骤1：测量串行深度

参见`code/main.py`。我们构建两个函数。一个将序列编码为加法链（串行，如RNN）。一个将其编码为并行规约（广播，如注意力）。相同的数学，不同的依赖图。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # can't parallelize: h depends on previous h
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # every x is independent
```

我们对两者在长达100,000个元素的序列上计时。RNN版本是O(N)且单CPU流水线。即使在纯Python中，注意力风格的规约在长度≥1,000时胜出，因为Python的`sum()`是用C实现的，且迭代时没有每步的解释器开销。

### 步骤2：计算理论操作

两个算法都进行N次加法。区别在于*依赖深度*：在下一步开始前必须顺序完成的操作数。RNN深度=N。注意力深度=log(N)（使用树规约）或1（使用并行扫描）。决定GPU时间的不是操作数，而是深度。

### 步骤3：长序列上的经验缩放

我们打印一个计时表，使O(N)差异显而易见。在2026年的Mac笔记本上，长度低于1000的序列快得无法测量。100,000的序列显示出清晰的线性扫描。将其扩展到16,384个token的Transformer和等效的12层LSTM，就能理解为什么2016年训练墙钟时间是一个障碍。

## 使用它

2026年何时仍选择RNN：

|  情况  |  选择  |
|-----------|------|
|  流式推理，一次一个token，恒定内存  |  RNN或状态空间模型（Mamba, RWKV）  |
|  极长序列（>1M token），注意力内存爆炸  |  线性注意力, Mamba 2, Hyena  |
|  无矩阵乘法加速器的边缘设备  |  深度可分离RNN在FLOPs/瓦特上仍占优  |
|  其他所有情况（训练、批推理、上下文至128K）  |  Transformer  |

像Mamba这样的状态空间模型（SSM）本质上是具有结构化参数化的RNN，通过选择性扫描实现了两全其美：`O(N)`扫描内存、并行训练。它们恢复了Transformer质量的90%，同时具有更好的长上下文缩放。到2026年，大多数前沿实验室训练混合SSM+Transformer模型（例如Jamba、Samba）——循环并未消亡，它只是一个组件。

## 练习

1. **简单。** 从`code/main.py`获取`rnn_style`，并将标量隐藏状态替换为长度为64的隐藏状态向量。重新测量。串行开销随隐藏状态维度增长多少？
2. **中等。** 在纯Python中实现并行前缀和（Hillis-Steele扫描）。验证其在长度1024上产生与串行扫描相同的数值输出。计算深度。
3. **困难。** 将注意力风格的规约移植到GPU上的PyTorch。在从64到65,536的序列长度范围内计时两者。绘制曲线图并解释形状。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|-----------------------|
| 递归(Recurrence) | "RNN是顺序的" | 步骤`t`依赖于步骤`t-1`的计算，强制沿时间轴串行执行。 |
| 串行深度(Serial depth) | "图有多深" | 依赖操作的最长链；即使在无限硬件上也受限于墙上时钟时间。 |
| 注意力(Attention) | "让令牌互相查看" | 加权和`sum_j a_ij v_j`，其中`a_ij`来自位置i和j之间的相似度得分。 |
| 上下文窗口(Context window) | "模型能看到多少" | 注意力层可以输入的令牌数量；二次内存成本在此缩放。 |
| 归纳偏置(Inductive bias) | "架构中内置的假设" | 关于数据样式的先验；CNN假设平移不变性，RNN假设近因性。 |
| 状态空间模型(State-space model) | "具有代数基础的RNN" | 通过结构化状态空间矩阵参数化的递归，支持并行训练。 |
| 二次瓶颈(Quadratic bottleneck) | "为什么上下文成本这么高" | 注意力内存 = 序列长度的`O(N²)`；Flash Attention隐藏了常数，但没有隐藏缩放。 |
