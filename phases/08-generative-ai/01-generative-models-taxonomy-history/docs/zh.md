# 生成模型——分类与历史

> 每一个图像模型、文本模型、视频模型和3D模型都可归入五类中的一类。选错一类，你将与数学搏斗数周；选对一类，该领域过去十二年的进展将清晰地呈现在你脑海中。

**类型：** 学习
**语言：** Python
**先决条件：** 阶段2（机器学习基础）、阶段3（深度学习核心）、阶段7·14（Transformer）
**时间：** ~45分钟

## 问题

生成模型的任务只有一个：给定从某个未知分布`p_data(x)`中抽取的训练样本，输出看起来来自同一分布的新样本。人脸、句子、MIDI文件、蛋白质结构——如果你粗略地看，它们都是同一个问题。

棘手之处在于`p_data`存在于一个具有数百万维度的空间中（一张512x512的RGB图像约786k维度），样本位于该空间内的一个薄流形上，而你只有大约1000万个样本。暴力求解密度是不可能的。每种生成模型都是一种折中，将一个难题换成一个稍微不那么难的问题。

在过去的十二年里，有五个系列存活了下来。了解每个系列所做的折中，就能明白为什么它在某些任务上胜出，而在其他任务上失败。

## 核心概念

![Five families of generative models — taxonomy by what they model](../assets/taxonomy.svg)

**1. 显式密度，可处理。** 将`log p(x)`写成一个可以实际求和的表达式。自回归模型（PixelCNN、WaveNet、GPT）对`p(x) = ∏ p(x_i | x_<i)`进行分解。归一化流（RealNVP、Glow）通过一个可逆变换将`p(x)`构造为一个简单基分布。优点：精确似然，干净的训练损失。缺点：自回归推理是顺序的（长序列慢），流需要可逆架构（架构受限）。

**2. 显式密度，近似。** 从下界（ELBO）近似`log p(x)`并优化该下界。VAE（Kingma 2013）使用带有变分后验的编码器-解码器。扩散模型（DDPM、Ho 2020）训练一个去噪器，隐式优化加权ELBO。扩散是2026年图像、视频和3D的主流骨干。

**3. 隐式密度。** 完全跳过密度；学习一个生成器`G(z)`产生样本和一个判别器`D(x)`区分真假。GAN（Goodfellow 2014）。推理速度快（一次前向传播），但训练时极不稳定。即使在2026年，StyleGAN 1/2/3在固定域的照片级真实感（人脸、卧室）方面仍是最先进的。

**4. 基于分数/连续时间。** 直接学习对数密度`∇_x log p(x)`的梯度（分数）。Song & Ermon（2019）表明分数匹配将扩散推广到随机微分方程。流匹配（Lipman 2023）是2024-2026年的热点：免模拟训练，更直的路径，采样速度比DDPM快4-10倍。Stable Diffusion 3、Flux、AudioCraft 2都使用了流匹配。

**5. 基于离散码的自回归。** 通过VQ-VAE或残差量化器将高维数据压缩成短序列的离散标记，然后使用Transformer对标记序列建模。Parti、MuseNet、AudioLM、VALL-E、Sora的patch分词器都采用这种方法。这其实是第一类加上一个学习到的分词器。

## 五问筛查

当一篇新的生成模型论文发布时，在阅读方法部分之前，回答以下五个问题。

1. **建模什么？** 像素、潜在变量、离散标记、3D高斯、网格、波形？
2. **密度是显式还是隐式？** 他们是否写出了`log p(x)`？
3. **采样：一步到位还是迭代？** 迭代意味着推理较慢；一步到位通常意味着对抗或蒸馏。
4. **条件：无条件、类别、文本、图像、姿态？** 这决定了损失和架构框架。
5. **评价：FID、CLIP分数、IS、人类偏好、任务准确率？** 每一种都有已知的失效模式（见第14课）。

在本阶段的每节课中，你都将重新回答这五个问题。到结束时，它们将成为条件反射。

## 动手构建

本课的代码是一个轻量级可视化：使用三种简单方法（核密度估计、离散直方图和最近样本“类GAN”生成器）从样本中拟合一维高斯混合模型，以便你能在单个屏幕上看到显式密度与隐式密度之间的差异。

运行 `code/main.py`。它从双峰高斯混合中抽取2000个样本，然后打印：

```
explicit density (histogram): p(x in [-0.5, 0.5]) ≈ 0.38
approximate density (KDE):     p(x in [-0.5, 0.5]) ≈ 0.41
implicit (nearest-sample gen): 20 new samples printed, no p(x)
```

注意：前两种方法允许你询问“这个点有多可能？”而第三种不能。这就是*显式与隐式*的区别，它对之后的每节课都很重要。

## 使用它

到2026年，哪个家族适合哪项任务？

|  任务  |  最佳家族  |  原因  |
|------|-------------|-----|
|  逼真人脸，窄领域  |  StyleGAN 2/3  |  仍然最锐利，推理最快。  |
|  通用文生图  |  隐扩散 + 流匹配  |  SD3, Flux.1, DALL-E 3。  |
|  快速文生图  |  整流流 + 蒸馏  |  SDXL-Turbo, SD3-Turbo, LCM。  |
|  文生视频  |  扩散Transformer + 流匹配  |  Sora, Veo 2, Kling。  |
|  语音 + 音乐  |  基于token的自回归（AudioLM, VALL-E, MusicGen）或流匹配（AudioCraft 2）  |  离散token扩展成本低。  |
|  3D场景  |  高斯泼溅拟合，扩散先验  |  3D-GS用于重建，扩散用于新视角。  |
|  密度估计（无采样）  |  流  |  唯一能给出精确 `log p(x)` 的家族。  |
|  仿真 / 物理  |  流匹配，分数SDE  |  直线路径，平滑向量场。  |

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|-----------------------|
|  生成模型  |  “它生成新内容”  |  学习一个 `p_data(x)` 的采样器，可选地暴露 `log p(x)`。  |
|  显式密度  |  “你可以评估它”  |  模型提供封闭形式或可处理的 `log p(x)`。  |
|  隐式密度  |  “GAN风格”  |  只有采样器——无法评估给定点的 `p(x)`。  |
|  ELBO  |  “证据下界”  |  `log p(x)` 的可处理下界；VAE和扩散模型优化它。  |
|  分数  |  “对数密度的梯度”  |  `∇_x log p(x)`；扩散和SDE模型学习这个场。  |
|  流形假设  |  “数据生活在表面上”  |  高维数据集中在低维流形上；降维有效的原因。  |
|  自回归  |  “预测下一个部分”  |  将联合概率分解为条件概率的乘积。  |
|  隐变量  |  “压缩编码”  |  低维表示，解码器可从中重建输入。  |

## 生产备注：五个家族，五种推理形态

每个家族对应不同的推理服务器成本曲线。生产推理文献将LLM推理分为预填充+解码；同样的分解也适用于此：

- **自回归（桶1和5）。** 顺序解码主导延迟；KV缓存、连续批处理和推测性解码都直接适用。
- **VAE / 扩散 / 流匹配（桶2和4）。** 没有LLM意义上的解码。成本 = `num_steps × step_cost`，`step_cost` 是在全潜分辨率下的Transformer或U-Net前向传播。生产中的调节旋钮是步数（DDIM / DPM-Solver / 蒸馏）、批大小和精度（bf16 / fp8 / int4）。
- **GAN（桶3）。** 一次前向传播。无调度，无KV缓存。TTFT ≈ 总延迟。这就是StyleGAN在窄领域用户体验上仍然胜出的原因。

当你在论文摘要中看到“比扩散更快”时，将其翻译为“更少的步数 × 相同的步成本”或“相同的步数 × 更低的步成本”。其他都是营销话术。
