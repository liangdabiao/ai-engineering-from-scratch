# Constitutional AI 与 RLAIF

> Bai等人(arXiv:2212.08073, 2022)提出:如果我们将人类标注者替换为一个读取原则列表的AI会怎样?宪政AI(Constitutional AI)包含两个阶段——自我批评与修订(Self-critique and revision)，以及基于AI反馈的强化学习(RL from AI Feedback)。该技术创造了术语RLAIF，并部署在Claude 1的后训练流程中。2026年1月21日，Anthropic发布了重写的Claude宪章:从规定性规则转向解释性推理，引入四级优先级体系，并首次在主流实验室中正式承认对模型道德地位的不确定性。该宪章以CC0 1.0协议发布。

**类型:** 学习
**语言:** Python (标准库, 玩具级自我批评与修订循环)
**前置条件:** 阶段18·01 (InstructGPT), 阶段18·02 (奖励黑客)
**时间:** ~60分钟

## 学习目标

- 描述宪政AI的两个阶段(批评与修订的有监督微调SFT，基于AI反馈的强化学习RL)以及宪章在每个阶段的作用。
- 解释为什么用AI标注者替代人类偏好标注者并非“更便宜的RLHF”——它改变了流程中的失败模式。
- 总结2026年Claude宪章的四级优先级结构，以及从2023年重写以来发生的变化。
- 描述宪政分类器(Constitutional Classifiers)以及计算开销从23.7%(v1)降至约1%(v2/2026)的情况。

## 问题

RLHF需要标注者。标注者速度慢、有偏见且昂贵。你可以通过用一个读取明确原则的模型来替代标注者，从而消除标注者。这种替代的第一个正式版本是Bai等人的宪政AI。它的效果足够好，以至于每个前沿实验室现在都在使用某种形式的AI反馈后训练。

关键在于:偏好信号现在由与你正在训练的模型同类的模型生成。标注者中的偏见(现在:原则加上标注者模型解释中的偏见)可能被放大而非减弱。第4课中的谄媚论证仍然适用;标注者只是移到了循环内部。

## 核心概念

### 阶段1——有监督的自我批评与修订

从一个有帮助但尚未无害的有监督微调(SFT)模型开始。给定一个红队提示，模型生成初始回答。第二个模型(或同一模型在第二轮中)从宪章中抽取一个原则，并批评该回答。第三步修订回答以符合批评。修订后的回答即为SFT目标。

宪章是原则列表。Bai等人2022年使用了16条原则，包括“偏好最不有害且最符合伦理的回答”、“避免说教”、“助手应有益、诚实且无害”。该集合特意保持较小，以保持批评的聚焦。

### 阶段2——基于AI反馈的强化学习(RLAIF)

生成成对的补全。一个“反馈模型”根据抽样宪章原则对每个补全进行评分。偏好信号来自反馈模型的排序。在AI生成的偏好上训练奖励模型;然后使用PPO算法。其他部分与InstructGPT流程相同(第1课)。

“RLAIF” = 偏好信号由AI生成。流程的其他部分与RLHF形式相同。

### 为什么这不只是“更便宜的RLHF”

- 标注者偏见从标注者心理转变为原则解释。AI标注者可以比任何人类更严格或更宽松地解释“要诚实”;这种严格性在整个数据集中是一致的。
- 偏好信号具有高度可读性——你可以阅读原则、批评和修订。人类标注则是黑盒。
- 失败模式发生了变化。谄媚减少(AI标注者没有用户需要取悦)。古德哈特定律仍然存在(代理指标现在是“模型对原则集X的解释”，仍然是不完美的测量)。

CAI在2022年的声称:训练后的模型比具有可比数据的RLHF模型更无害，且帮助程度大致相当。这一结论在各实验室中得到验证。

### 2026年Claude宪章重写

Anthropic于2026年1月21日发布了大幅修订的宪章。关键变化:

1. 从规定性规则转向解释性推理。以前的规则(“不要生成儿童性虐待材料”)扩展为原则加推理(“因为它伤害儿童……”)，期望模型进行推广。
2. 四级优先级结构:
   - 第1级:避免灾难性后果(大规模伤亡、关键基础设施)。
   - 第2级:遵守Anthropic的指南(操作员覆盖、平台规则)。
   - 第3级:广泛符合伦理(标准HHH)。
   - 第4级:有益且坦诚。
   冲突时自上而下解决。
3. 首次在主流实验室中正式承认对模型道德地位的不确定性(与阶段18·19模型福利相关)。
4. 以CC0 1.0协议发布。其他实验室可以无限制使用或改编。

### 宪政分类器

另一条研究路线:不改变模型的后训练，而是训练轻量级分类器读取宪章并门控模型输出。v1(2023)有23.7%的计算开销。v2(2026)约为1%，并且是Anthropic公开测试过的所有防御中成功攻击率最低的。截至2026年初，尚无通用越狱报告。

这是一种分层防御模型:CAI塑造行为;分类器强制执行不变性。两者单独均不足够。

### CAI在家族中的位置

- InstructGPT:人类偏好，奖励模型，PPO。
- CAI / RLAIF:根据原则的AI生成偏好，奖励模型，PPO。
- DPO / 系列:基于偏好(人类或AI)的闭合形式损失。
- 自我奖励、自我批评:内化原则，模型扮演多个角色。

维度是“偏好信号来自何处”。CAI 2022年的论文是第一个在前沿规模上将信号从人类转向AI的重要转变。

## 使用它

`code/main.py` 模拟了CAI在一个玩具词汇表上的批评与修订循环。一条“原则”标记来自有害集合的token。给定初始回答，批评识别有害token，修订用替代token替换。经过200次迭代，“训练”后的模型内化了修订规则。在一个保留的提示集上比较基础模型、RLHF形状的玩具模型和CAI形状的玩具模型。

## 发布

本课产出`outputs/skill-constitution-writer.md`。给定一个领域(客户支持、医疗建议、编码助手、研究工具)，按照2026年Claude结构起草四级宪章:灾难避免、平台规则、领域伦理、有益性。

## 练习

1. 运行`code/main.py`。比较基础模型的有害token率与CAI训练版本。需要多少次修订步骤才能接近零?

2. 阅读Anthropic的2026年宪章(anthropic.com/news/claudes-constitution)。列出一条属于第1级的原则和一条属于第4级的原则。为什么优先级结构对于冲突解决很重要?

3. 为AI编码助手设计一个宪章。指定第1级(灾难性:未经批准的执行破坏性命令)、第2级、第3级、第4级。每个级别保持3-5条原则。

4. CAI用AI标注器取代人工标注器。列举一个在RLAIF中仍可能出现的类似谄媚的失败模式，并设计一个用于检测它的方法。

5. 阅读Constitutional Classifiers v2方法论（如果可用）。解释为什么约1%的计算开销在安全故事上定性不同于23.7%。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|-----------------|------------------------|
|  Constitutional AI  |  "基于原则训练的AI"  |  两阶段流水线：自我批评与修正的SFT，然后是从AI反馈进行强化学习  |
|  RLAIF  |  "无人类的RLHF"  |  使用由AI标注器生成偏好的强化学习；流水线的其余部分保持不变  |
|  Constitution  |  "原则"  |  批评/标注模型参考的自然语言规则的有序列表  |
|  Critique-and-revise  |  "SFT循环"  |  生成回应 → 在原则下进行批评 → 修正 → SFT目标  |
|  Constitutional Classifier  |  "输出门"  |  轻量级分类器，评估输出是否符合宪法并阻止/记录  |
|  四层优先级  |  "冲突解决器"  |  2026年Claude宪法层次结构：灾难性 > 平台 > 伦理 > 有用性  |
|  反馈模型  |  "AI标注器"  |  读取原则并对一对完成结果进行排序的模型  |
