# 谈判与议价

> 智能体(Agent)就资源、价格、任务分配和条款进行谈判。2026年的基准测试集已明确：NegotiationArena (arXiv:2402.05863) 表明，大语言模型(LLM)通过角色操控("绝望")可将收益提高约20%；"Measuring Bargaining Abilities" (arXiv:2402.15813) 显示，买方比卖方更难，且规模扩大并无帮助——其 **OG-Narrator** (确定性报价生成器 + LLM叙述者) 将成交率从26.67%提升至88.88%；大规模自主谈判竞赛 (arXiv:2503.06416) 进行了约18万次谈判，发现 **隐藏思维链(chain-of-thought-concealing)** 的智能体通过向对手隐藏推理过程而获胜；Bhattacharya等人2025年在哈佛谈判项目指标上的排名显示，Llama-3最有效，Claude-3最激进，GPT-4最公平。本课实现了合同网协议(Contract Net Protocol，即FIPA的前身，第02课)，搭建了LLM风格的买方/卖方，运行了OG-Narrator风格的分解，并衡量了成交率随每种结构选择的变化。

**类型：** 学习+构建
**语言：** Python (标准库)
**前置条件：** 第16阶段 · 02 (FIPA-ACL遗产)，第16阶段 · 09 (并行群体网络)
**时间：** 约75分钟

## 问题

两个智能体需要就某个价格达成一致。仅凭纯语言提示，2024-2026年的大语言模型以惊人的低成交率达成交易(在arXiv:2402.15813中关于严格参数化议价的情况下约为27%)。规模扩大并不能解决这个问题：GPT-4在议价方面并不比GPT-3.5结构上更优；它只是在议价的*语言*方面更优。

根本问题在于，大语言模型混淆了两项工作——决定报价和叙述报价。OG-Narrator将两者分离：一个确定性报价生成器计算数值步骤；大语言模型仅负责叙述。成交率跃升至约89%。

这反映了一个经典的多智能体发现：将机制与通信层解耦是取胜之道。合同网协议(Contract Net Protocol，FIPA 1996；Smith 1980)是参考任务市场机制。将大语言模型插入叙述槽，就得到了一个现代的大语言模型驱动任务市场。

## 概念

### 合同网，概述

Smith 1980年提出的合同网协议：一个**管理者(manager)**广播**招标书(call for proposals, cfp)**；**投标者(bidders)**以包含其报价的**提议(propose)**消息响应；管理者选择获胜者并向获胜者发送**接受提议(accept-proposal)**，向失败者发送**拒绝提议(reject-proposal)**。获胜者执行工作。可选消息：**拒绝(refuse)** (投标者拒绝提议)。FIPA将其编码为`fipa-contract-net`交互协议。

### 为什么OG-Narrator获胜

"Measuring Bargaining Abilities of Language Models" (arXiv:2402.15813) 观察到：

- 大语言模型常常违反议价规则(以无意义的价格报价，忽视对方的最佳替代协议ZOPA)。
- 它们的锚定效果很差(接受糟糕的首次报价；以象征性的而非战略性的金额进行还价)。
- 仅靠规模并不能解决这些问题。更大的模型能生成更合理的语言，但策略错误相似。

OG-Narrator的分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ offer generator  │ price → │  LLM narrator    │ → message
           │  (deterministic) │        │  (writes the     │
           │                  │        │   human-style    │
           └──────────────────┘        │   accompaniment) │
                                       └──────────────────┘
```

报价生成器是经典谈判策略：鲁宾斯坦议价模型、Zeuthen策略或简单的价格以牙还牙。大语言模型负责叙述。消息包含确定性价格和自然语言框架。

成交率跃升的原因是：
- 价格保持在谈判区间内。
- 锚定是战略性的，而非情绪化的。
- 大语言模型做了它擅长的事：写作。

### NegotiationArena的发现

arXiv:2402.05863提供了标准基准测试。主要发现：

- 大语言模型通过采用角色(如"我很着急必须在周五前卖掉这个")可将收益提高约20%——角色操控是一种真实策略。
- 公平/合作的智能体会被对手利用；防御需要明确的对抗姿态。
- 对称配对在大约40%的基准场景中会收敛到不公平的结果。

这并非"大语言模型是不好的谈判者"，而是"大语言模型谈判时过于像人类，包括那些易被利用的部分。"

### 隐藏思维链(Chain-of-thought concealment)

大规模自主谈判竞赛 (arXiv:2503.06416) 对多种大语言模型策略进行了约18万次谈判。获胜者向对手隐藏了他们的推理过程：

- 如果一个智能体在公开的草稿区打印"我只会出到$75; my reservation price is $70"，对手就能看到。
- 获胜者私下计算策略；输出通道仅包含报价和最少的必要叙述。

这是2026年对经典博弈论( Aumann 1976关于理性与信息)的回响：透露你的私人估值会损失收益。大语言模型并未直觉地认识到这一点，乐于在推理痕迹中写出自己的保留价，而这些痕迹对对手可见。

工程启示：将私人草稿区上下文与公开消息上下文分开。这不是可选项。

### Bhattacharya等人2025年——模型排名

根据哈佛谈判项目指标(原则性谈判、BATNA尊重、利益互惠)：

- **Llama-3** 在达成交易方面最有效(成交率+收益)。
- **Claude-3** 是最激进的谈判者(高锚定、后期让步)。
- **GPT-4** 是最公平的(配对间收益差异最小)。

这是2025年的一份快照。重点不在于2026年4月哪个模型胜出——而是不同基础模型具有持久的谈判风格。异质集成(第15课)将其作为一种多样性来源。

### 通过合同网+LLM进行任务分配

合同网在现代LLM多智能体中的重新使用：

1. 管理者代理将任务分解为多个单元。
2. 通过`cfp`向工作者代理广播任务描述。
3. 每个工作者返回一个报价：`cfp`，其中价格可以是代币、计算单位或美元。
4. 管理者选择胜出者（单个或多个，取决于任务）并授予任务。
5. 被拒绝的工作者可以自由参与其他任务的竞标。

这种方式可良好扩展到超过100个工作者，因为协调是广播与响应式，而非同步聊天。已用于生产环境：微软Agent框架的编排模式、某些LangGraph实现。

### 大语言模型-利益相关者交互式谈判

NeurIPS 2024 (https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf) 引入了多方可评分博弈，具有**秘密分数**和**最低接受门槛**。每个利益相关者都有私有效用；大语言模型必须从消息中推断这些效用。这是从两方讨价还价到N方联盟形成的推广。与具有异构工作者能力的生产任务市场相关。

### 叙述与机制分离规则

在所有2024-2026年的谈判基准测试中，一致的工程规则是：

> 让大语言模型叙述。不要让大语言模型计算报价。

如果报价需要是一个数字（价格、预计到达时间、数量），则根据谈判状态确定性生成，并让大语言模型生成表述框架。如果报价需要是一个提案结构（任务分解、角色分配），则让大语言模型起草，但在发送之前根据模式和约束检查进行验证。

## 动手构建

`code/main.py` 实现：

- `ContractNetManager`，`ContractNetTask`，`Bid` — 管理者 + 竞标者，广播招标，收集提案，授予。
- `ContractNetManager` — OG-Narrator买家：向中点确定性地让步（Zeuthen风格）。
- `ContractNetManager` — 确定性卖家还价策略（两种风格的结构性真实值）。
- `ContractNetManager` — 模拟全大语言模型谈判者：高方差选择价格，通常超出ZOPA。
- 度量：1000次试验中的成交率，每次试验采样新的保留价格。

运行：

```
python3 code/main.py
```

预期输出：朴素大语言模型成交率约65-75%；OG-Narrator成交率约85-95%；15-25个百分点的差距是报价生成与叙述分离的结构性优势。此外，还有一个包含三个竞标者和一个任务的合同网任务市场分配示例。

## 发布

生产谈判检查清单：

- **分离草稿。** 私有状态永远不进入对方的上下文。这是不可协商的。
- **确定性报价生成。** 价格、数量、预计到达时间：计算，不要提示。
- **根据模式验证所有收到的报价。** 在协议边界拒绝超出ZOPA的报价。
- **限制轮次。** 最多3-5轮；陷入僵局时升级到调解者。
- **持续度量成交率和收益方差。** 成交率下降是症状——通常是提示漂移或对方攻击。
- **记录所有被拒绝的提案及其确定性理由。** 对于合同网管理者，失败的竞标者需要了解原因。

## 练习

1. 运行`code/main.py`。确认OG-Narrator在成交率上优于朴素大语言模型。差距多少？
2. 实现**基于角色的收益改进**（arXiv:2402.05863）——买家仅在叙述中采用“本周急需购买”的角色，报价生成器不变。成交率或收益是否有变化？
3. 实现思维链**隐藏**：维护一个私有草稿字符串，不传递给对方。如果不小心泄露（通过交换通道模拟），会发生什么？
4. 将合同网扩展为带保留价的N竞标者拍卖。当所有竞标都超过保留价时，管理者如何在最低价和最高质量之间决策？你选择哪个授予规则？为什么？
5. 阅读Bhattacharya等人2025年关于哈佛谈判项目指标的文章。实现两种不同风格（激进 vs 公平）的谈判者。在对称和非对称配对下度量收益方差。
