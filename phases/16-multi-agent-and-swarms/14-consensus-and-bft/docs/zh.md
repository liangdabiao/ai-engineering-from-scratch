# 智能体共识与拜占庭容错

> 经典分布式系统BFT遇到随机LLM。在2025-2026年，出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）根据置信度探测为每个投票加权；**DecentLLMs**（arXiv:2507.14928）采用无领导方式，并行提出工作节点，并通过几何中位数聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类（Hierarchical Structure Clustering）相结合，将节点分为核心和边缘两类。来自“Can AI Agents Agree?”（arXiv:2603.01213）的诚实实证结果是：即使标量一致（scalar agreement）在当下也很脆弱——单个欺骗性智能体就能破坏混合智能体（Mixture-of-Agents）。BFT是必要的，但并不充分。本课构建一个最小BFT协议，引入三种特定于智能体的攻击（拜占庭谎言、谄媚从众、相关误差单一文化），并测量每种共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第16阶段·07（心智社会与辩论），第16阶段·13（共享内存）
**时间：** 约75分钟

## 问题

你有N个LLM智能体，每个都产生一个答案。它们意见不一致。多数投票选错了，因为两个智能体是相关的（相同的基础模型、相同的训练数据、相同的失败模式）。第三个智能体恰好以一种新方式出错——因此多数是虚假多数。

现在加入一个欺骗性智能体：它故意撒谎。或者一个谄媚智能体：它总是同意最后一个发言的人。在经典BFT中，假设拜占庭节点占比`f < n/3`且行为任意。2026年的现实是：LLM节点即使诚实也是随机的，跨模型相关，并受彼此输出的影响。你不能把它们当作独立的伯努利投票者。

经典BFT（PBFT，1999）并没有错——它是不完整的。它能处理任意比特翻转。但它不能处理“三个诚实智能体因为共享训练数据而产生相同的幻觉”。本课从PBFT的基础出发，并叠加了三种2025-2026年的适应性改进。

## 概念

### 经典BFT能给你什么

实用拜占庭容错算法（Castro & Liskov，OSDI 1999）能容忍`f < n/3`个拜占庭节点。该协议包含三个阶段（预准备、准备、提交）和两个原语（签名消息、法定证书）。在`n >= 3f + 1`个诚实或恶意节点之间达成单一值的一致。

其保证很强，但假设：

1. **独立故障。** 拜占庭节点不协调。
2. **诚实节点真正诚实。** 诚实输出的正确性不成问题；协议只对齐分歧。
3. **问题有真实答案。** 对错误事实达成共识仍然是共识。

LLM智能体违反了所有三个假设。运行相同基础模型的两个智能体共享故障。一个“诚实”的LLM仍然会产生幻觉。而在模糊问题上，“真相”取决于智能体的决定——没有外部预言机。

### 三种特定于LLM的攻击

**拜占庭谎言。** 一个智能体故意输出错误答案。如果`f < n/3`，经典BFT可以处理。

**谄媚从众。** 一个智能体在投票前读取其他智能体的答案，并与最后一个发言者保持一致。并非恶意，但与最强声音相关。经典BFT无法阻止，因为该智能体通过了所有签名检查。

**相关误差单一文化。** 三个智能体共享一个基础模型。它们产生相同的错误幻觉。多数是错误的。经典BFT无济于事，因为这三个智能体“诚实”地达成了一致。

### 2025-2026年的应对方案

**CP-WBFT**（arXiv:2511.10400）——置信度探测加权BFT。每个投票者为其答案附加一个置信度探测（自我报告的概率，或独立校准模型的预测）。投票权重按置信度缩放。报告在完全图上的BFT改进达+85.71%。缓解：谄媚从众（谄媚智能体对其自选立场通常自信度较低）。

**DecentLLMs**（arXiv:2507.14928）——无领导。工作智能体并行提出，评估智能体对提案打分，最终答案是打分位置的几何中位数（geometric median）。当`f < n/2`时具有鲁棒性。缓解：拜占庭谎言和相关误差（几何中位数对异常值鲁棒，并向密集簇而非模型偏差均值靠拢）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权BFT。投票权重根据响应质量以及从历史学习到的信任分数分配。将智能体聚为核心和边缘；核心智能体必须首先达成共识，边缘智能体随后跟随。缓解：可扩展性（核心共识小而快）并部分缓解单一文化（核心可选择多样化）。

### 实证：“Can AI Agents Agree?”（arXiv:2603.01213）

该论文测量了多种前沿模型上的标量一致（LLM智能体在单一数值上达成一致）。结果令人不安：

- 即使没有对手，LLM智能体在许多基准测试中标量问题的分歧率超过30%。
- 单个采用欺骗性人格的智能体就能将混合智能体（Mixture-of-Agents）共识拉离诚实基线40多个百分点。
- 分歧率与模型多样性相关——异质集成比同质集成分歧更大（好处：误差不相关），但收敛也更慢（坏处：达成一致时间更长）。

要点：BFT提供了对齐输出的机制，但它不告诉你对齐后的输出是否正确。结合验证（第16阶段·08角色专业化）、多样性（第16阶段·15辩论变体）和评估智能体（第16阶段·24基准测试）。

### 核心协议，精简版

针对LLM智能体的最小化BFT轮次：

```
1. task arrives; each agent i produces answer a_i
2. each agent attaches confidence probe c_i in [0, 1]
3. aggregator collects (a_i, c_i) from all n agents
4. aggregator groups by semantic cluster (equivalent answers)
5. aggregator computes weight for each cluster C:
     w(C) = sum_{i in C} c_i
6. winner = cluster with max weight, if max > threshold * sum(c_i)
   else: retry or escalate
7. minority clusters logged with provenance for post-hoc audit
```

语义聚类是LLM特有的技巧。两个答案“研究报告4.2%”和“4.2%改进”属于同一个聚类。简单的字符串相等检查会遗漏这一点。在生产中，使用廉价的嵌入模型或显式规范化。

### 阈值调整

参数`threshold`决定了何时接受、何时重试。太低：你会接受弱多数。太高：你永远不会接受任何东西。经验范围：对`n=5-7`个智能体为0.5-0.67，对更小的`n`更高。低于阈值时，升级到人类或不同的智能体集成。

### 共识无济于事的场合

- **模糊问题。** 如果问题没有真实答案，共识是一种观点。请如实称呼它。
- **复合问题。** “编写代码并解释”——两个答案。分别对每个投票。
- **对抗性多轮。** 如果智能体可以观察先前轮次并模仿（Du 2023辩论），它们开始无视真相彼此同意。限制轮次（通常2-3轮）。

已实现的攻击模式：

1. `byzantine`: 一个智能体以高置信度撒谎。
2. `byzantine`: 一个智能体复制它看到的第一个答案，并匹配置信度。
3. `byzantine`: 三个智能体共享一个错误答案（相关错误），置信度适中。

```
python3 code/main.py
```

## 发布

在部署任何共识机制之前：

- **使用至少上述三种模式进行攻击测试**。你的协议应可预测地失败，而不是静默失败。
- **记录每个少数集群及其来源**。少数集群是相关错误的早期预警系统。
- **强制执行有限轮次**。不要“不断辩论直到达成一致”——这会奖励谄媚行为。
- **将一致性与正确性分离**。共识输出交给验证器；验证器独立于集成体。
- **监控一致性率**。急剧上升意味着趋同偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认多数投票在单一文化攻击下失败，但CPWBFT在单一文化置信度低于0.7时部分缓解了攻击。
2. 添加第四种攻击模式：**静默弃权**——一个智能体拒绝回答（“我不知道”）。每个聚合器应如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化改为嵌入相似度（使用任何开源嵌入模型）。谄媚攻击会发生什么？
4. 阅读CP-WBFT (arXiv:2511.10400)。实现置信度探测校准步骤（一个单独的校准模型检查每个智能体自报告的置信度）。测量在单一文化场景下的准确率提升。
5. 阅读“Can AI Agents Agree?” (arXiv:2603.01213)。复现一个简化的标量一致实验：三个智能体，一个标量问题，欺骗性人格提示。CPWBFT或DecentLLMs能否捕捉到它？

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  BFT  |  "拜占庭容错"  |  Castro-Lisskove 1999年针对`f < n/3`任意故障的共识协议。  |
|  拜占庭  |  "任何不良行为"  |  一个节点可以撒谎、丢弃消息、静默失败——除安全崩溃外任何行为。  |
|  置信度探测  |  "你有多确定？"  |  自报告或校准器预测的附加在投票上的概率。  |
|  语义聚类  |  "相同答案，不同措辞"  |  在计票前对等价答案进行分组。  |
|  几何中位数  |  "稳健中心"  |  使到样本点距离之和最小的点。对异常值稳健，不同于均值。  |
|  单一文化  |  "相同模型，相同失败"  |  智能体共享训练数据或基础模型时的相关错误。  |
|  谄媚趋同  |  "附和响亮的声音"  |  智能体的投票偏向于先发言/声音最大的人。  |
|  核心/边缘  |  "分层BFT"  |  WBFT拆分：先小核心共识，边缘节点跟随。限制延迟。  |
