# 心智社会与多智能体辩论

> 明斯基1986年的前提——智能是专家组成的社会——每十年就会被重新发现一次。2023年，Du等人将其转化为一个具体的算法：多个大语言模型实例提出答案，阅读彼此的答案，进行批评并更新。经过N轮，它们收敛到一个共识，在六个推理和事实性任务上超越了零样本思维链和反思。两个发现很重要：**多个智能体**和**多轮**各自独立地产生贡献。这个社会胜过单个智能体的独白；多轮交流胜过一次性投票。

**类型：** 学习+构建
**语言：** Python（标准库）
**先决条件：** 阶段16·04（原始模型）
**时间：** 约60分钟

## 问题

自洽性——多次采样同一个模型并取多数答案——是你可以附加的最经济的推理改进。它有效，但迅速饱和。你可以将样本数量翻倍，却看不到另一个有意义的跳跃。

辩论打破了饱和。不再是从一个模型采样N个独立样本，而是N个智能体阅读彼此的推理并修改。样本之间的相关性下降（它们不再是独立同分布），收敛点往往是正确的，而独立同分布投票则自信地错误。

## 概念

### 2023年Du等人的算法

来自arXiv:2305.14325（ICML 2024）：

1. 每个智能体对问题生成一个初始答案。
2. 对于轮次r = 2..R：每个智能体被展示其他智能体在第r-1轮的答案，并被要求“考虑这些，给出你更新的答案。”
3. 经过R轮后，对最终答案进行多数投票。

该论文在MMLU、GSM8K、传记、MATH和事实性基准上进行了测试。辩论一致地击败了思维链和自省。

### 两个独立旋钮

同一论文的消融实验：

- **仅增加智能体数量**（1轮，N个的多数投票）在大多数任务上胜过单个智能体，但会达到平台期。
- **仅增加轮次**（1个智能体看到自己之前的推理）几乎没有帮助——这是反思的已知弱点。
- **两者结合**产生大的跳跃。多个智能体之间的多轮交流推动了增益。

### 为什么有效

两种机制：

1. **暴露于分歧。** 当一个智能体看到另一个智能体具有不同结论的推理链时，它必须要么辩护要么更新。无论哪种方式，第r+1轮的上下文都比第r轮更丰富。
2. **相关误差减少。** 在自洽性中，所有样本来自同一模型，因此误差相关——你平均到一个自信的错误答案。不同的模型或不同的随机种子去相关。不同的*辩论观点*进一步去相关。

### 异质辩论

A-HMAD和相关的后续工作使用*不同的基础模型*用于不同的智能体。Llama + Claude + GPT的辩论减少了单一文化崩溃（第26课），因为一个模型家族的相关误差不被其他模型分享。

缺点：一个弱模型参与辩论可以将共识拖向其错误答案（参见“Should we be going MAD?”, arXiv:2311.17371）。

### NLSOM——129个智能体的扩展

Zhuge等人（“Mindstorms in Natural Language-Based Societies of Mind”，arXiv:2305.17066）将这个想法扩展到129个成员的社会。结果是：专业化和自组织随着规模而出现，系统在诸如视觉问答等任务上优于单个智能体。

### 故障模式

- **谄媚级联。** 所有智能体都服从听起来最自信的那个智能体。辩论沦为一言堂。提示对抗角色（“一个智能体必须反驳立场”）有所帮助。
- **话题漂移。** 多轮辩论会偏离原始问题。缓解措施：每轮重新注入问题。
- **计算爆炸。** N个智能体 × R轮 = N·R次LLM调用，每次调用上下文增长。5个智能体、5轮的辩论是25次调用且上下文不断增长。每个问题的成本可能超过单次思维链调用的10倍。

## 动手构建

`code/main.py` 在一个数学问题上运行3个智能体×3轮的辩论，每个智能体从一个不同的（可能是错误的）答案开始。智能体是脚本化的——每个通过按脚本化的置信度加权平均邻居的答案来“更新”。收敛在逐轮日志中可见。

演示展示了两个关键效果：

- 单轮交流使智能体更接近正确答案。
- 超过第2轮的额外轮次显示出递减的收益（与Du等人的平台期一致）。

运行：

```
python3 code/main.py
```

## 使用它

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮次数、异质性（同一模型 vs 混合）、角色分配（对称 vs 一个对抗）。它还在运行前估计token成本。

## 发布

如果你部署辩论：

- **轮次数上限设为3。** Du等人表明3轮捕获了大部分增益。更多是成本，而非质量。
- **智能体数量上限设为5。** 超过5，上下文膨胀和成本占主导。
- **默认异质。** 池中至少有两个不同的基础模型。
- **对抗槽位。** 一个智能体被提示无论如何都要反对。打破谄媚。
- **记录每一轮。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习

1. 运行`code/main.py`，然后将轮次数设置为5，观察递减收益。在哪个轮次附加的收敛停止？
2. 添加第四个具有对抗角色的智能体：始终不同意当前多数。这会破坏还是改进收敛？
3. 绘制（打印）每轮的共识分数（智能体在多数答案上的比例）。何时达到1.0，这是否等同于“正确”？
4. 阅读Du等人的第4节消融实验。使用此代码复制“仅智能体” vs “仅轮次” vs “两者”的结果。
5. 阅读“Should we be going MAD?” (arXiv:2311.17371) 并列出两种除循环赛之外的辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  心智社会  |  "明斯基的想法"  |  智能作为相互作用的专家；1986年的框架现在通过大语言模型辩论得以实现。 |
|  多智能体辩论  |  "智能体争论"  |  N个智能体提出、相互批评、经过R轮修改，多数投票。 |
|  共识  |  "他们同意"  |  不是认识论真理——只是多数答案上的比例。可能自信地错误。 |
|  轮次  |  "交换步骤"  |  一轮 = 每个智能体读取其他智能体并更新一次。  |
|  异构辩论  |  "混合模型家族"  |  使用不同的基础模型去相关错误。  |
|  谄媚级联  |  "每个人都同意声音最大的那个"  |  辩论失败，智能体无论正确与否都服从最自信的智能体。  |
|  NLSOM  |  "129智能体社会"  |  心智自然语言社会；诸葛等人的缩放版本。  |
|  相关误差  |  "相同模型，相同错误"  |  自一致性饱和的原因；跨不同观点的辩论去相关。  |
