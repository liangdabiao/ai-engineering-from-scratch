# 评估与协作基准

> 五个2025-2026基准涵盖了多智能体评估领域。**MultiAgentBench / MARBLE**（ACL 2025，arXiv:2503.01935）评估星型、链型、树型、图型拓扑结构，并使用里程碑式关键绩效指标；**图型最适合研究**，认知规划增加约3%的里程碑达成率。**COMMA** 评估多模态非对称信息协作；包括GPT-4o在内的最先进模型也难以击败随机基线。**MedAgentBoard**（arXiv:2505.12371）涵盖四类医疗任务，并发现多智能体并不总是优于单一LLM。**AgentArch**（arXiv:2509.10769）对结合工具使用、记忆和编排的企业智能体架构进行基准测试。**SWE-bench Pro**（[arXiv:2509.16941](https://arxiv.org/abs/2509.16941)）包含来自41个仓库的1865个问题，覆盖商业应用、B2B服务和开发者工具；前沿模型在Pro上的得分约为23%，而在Verified上超过70%——这揭示了数据污染的现实。Claude Opus 4.7（2026年4月）据报道在Pro上达到**64.3%**，并使用了显式的智能体团队协作（尚未公布Anthropic主要来源——视为初步结果）；Verdent（智能体框架）在Verified上达到**76.1% pass@1**（[arXiv:2509.16941](https://arxiv.org/abs/2509.16941)）。**AAAI 2026 Bridge Program WMAC**（https://multiagents.org/2026/）是2026年的社区焦点。本课基于MARBLE的指标，进行拓扑与指标对比扫描，并确立了“仅在SWE-bench Verified上通过并不代表泛化能力”的规则。

**类型：** 学习
**语言：** Python（标准库）
**前置条件：** 第16阶段·第15课（投票与辩论拓扑），第16阶段·第23课（故障模式）
**时间：** 约75分钟

## 问题

当一篇论文声称“我们的多智能体系统更好”时，问题在于：比什么好？在什么任务上好？用什么指标衡量？2023-2024年的多智能体评估是一片混乱——每个人都选择自己的指标、自己的基线和自己的任务集。2025-2026年的基准带来了结构。

没有共享的基准，你就无法有意义地比较两个多智能体系统。更糟的是，没有留出的基准，前沿模型可能会污染数据。SWE-bench Verified在2025年中左右部分被训练语料污染；前沿分数虚高；Pro被设计为无污染的现实检验。

本课列举了五个经典的2026基准，说明每个基准衡量什么，并教你如何怀疑地解读基准声明。

## 概念

### MultiAgentBench（MARBLE）——ACL 2025

arXiv:2503.01935。在研究、编程和规划任务上评估四种协作拓扑（星型、链型、树型、图型）。基于里程碑的关键绩效指标追踪部分进展，而不仅仅是最终成功。

测量结果：

- **图型**拓扑最适合研究场景；支持任意对任意的批评。
- **链型**最适合逐步改进的编程。
- **星型**最适合快速事实整合。
- **协作成本**在图型拓扑中从大约4个智能体开始出现。
- **认知规划**在所有拓扑中增加约3%的里程碑达成率。

适用场景：当你希望公平比较不同协作拓扑时。MARBLE仓库（https://github.com/ulab-uiuc/MARBLE）提供了评估器。

### COMMA——多模态非对称信息

涵盖智能体拥有不同观察模态且必须在不完全信息共享下进行协作的任务。报告的结果令人不安：包括GPT-4o在内的前沿模型在COMMA的智能体间协作中难以击败**随机基线**。这说明多智能体模态的训练和评估不足——LLM能合理处理单模态协作，但多模态协作会崩溃。

适用场景：当你的系统涉及多模态或非对称信息协作时。COMMA的零结果是一个警告：在宣称结果之前先测量。

### MedAgentBoard——领域压力测试

arXiv:2505.12371。四类医疗任务：诊断、治疗规划、报告生成、患者沟通。比较多智能体、单一LLM和传统基于规则的系统。

发现：在大多数类别上多智能体并**不优于**单一LLM。多智能体的优势很窄——当子任务明显可分离时（诊断+治疗），任务分解有帮助；当协作开销超过专业化增益时（报告生成），反而有害。

适用场景：当你的领域有明确的单一LLM基线时。如果MedAgentBoard的经验具有普遍性，许多提议的多智能体系统是过度设计的。

### AgentArch——企业架构

arXiv:2509.10769。结合工具使用、记忆和编排的企业环境。基准隔离每个层的贡献：添加工具有多大帮助？添加记忆呢？添加多智能体编排呢？

适用场景：当你设计企业智能体堆栈并需要证明每一层的合理性时。AgentArch有助于避免购买无法衡量其价值的特性。

### SWE-bench Pro——现实检验

arXiv:2509.16941。包含来自41个仓库的1865个问题，覆盖商业应用、B2B服务和开发者工具。设计为**无污染**（训练截止日期后）。前沿模型在Pro上的得分约为23%，而在Verified上超过70%。差距即污染信号。

2026年4月得分：
- Claude Opus 4.7在Pro上：**64.3%**（据报道使用了显式的智能体团队协作；尚未公布Anthropic主要来源——视为初步结果）。
- Verdent（智能体框架）在Verified上：**76.1% pass@1**（[technical report](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。
- 没有智能体框架的前沿模型在Pro上的原始得分：约23-35%（[technical report](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。

要点：“我们在SWE-bench Verified上击败了”不再是能力的证据。Pro是当前的关卡测试。智能体团队框架在Pro上产生了可衡量的增益（约30-40个百分点的差距），这是2026年多智能体协作最有力的实证论据之一。

### AAAI 2026 WMAC

AAAI 2026 Bridge Program——多智能体协作研讨会（https://multiagents.org/2026/）。2026年多智能体AI研究的社区焦点。录用论文和研讨会论文集是评估新方法的权威场所；对于生产决策，应以WMAC录用的声明为准，而非arXiv预印本。

### 怀疑地阅读基准声明——2026年检查清单

当有人声称一个多智能体结果时：

1. **哪个基准测试，哪个划分？** SWE-bench Verified 与 Pro 关系重大。在错误的划分上报告的数字毫无价值。
2. **污染检查。** 基准测试是否在模型的训练截止日期之后发布？如果不是，请谨慎对待。
3. **基线比较。** 与单 LLM 基线、随机基线、以往多智能体工作进行比较。而不是“与同一系统的未调优版本”比较。
4. **统计显著性。** N 次试验、p 值、置信区间。前沿模型方差大；单次运行具有误导性。
5. **任务多样性。** 一个任务还是多个？泛化能力对生产环境很重要。
6. **成本披露。** 每任务的令牌数、挂钟时间。一个 90% 的解决方案但成本是 20 倍，这是商业决策，而非能力宣称。

### 所有基准测试都未能很好衡量的内容

- **长周期协调。** 数天的挂钟交互时间。所有当前基准测试运行时间都较短。
- **对抗鲁棒性。** 当某个智能体恶意或被攻破时会发生什么？
- **部署中的漂移。** 基准测试是静态的；生产分布会变化。
- **成本归一化性能。** 大多数基准测试报告原始准确率，而非每美元准确率。

针对你真正关心的轴构建自己的内部基准测试通常是正确的做法。

## 动手构建

`code/main.py` 是一个非交互式演练：

- 在一个玩具任务上模拟 3 个多智能体系统。
- 为每个系统计算 MARBLE 式的里程碑指标。
- 通过从“训练”集中扣留任务进行污染检查。
- 显式地与随机基线比较。
- 打印基准测试宣称的记分卡。

运行：

```bash
python3 code/main.py
```

预期输出：系统记分卡，包含原始准确率、里程碑达成、每任务成本、与随机基线的差值，以及污染检查注释。

## 使用它

`outputs/skill-benchmark-reader.md` 读取任意多智能体基准测试宣称，并应用审查清单。输出：等级和注意事项。

## 发布

生产环境评估规范：

- **构建内部基准测试**，反映你的实际生产分布。公开基准测试提供信息但不替代。
- **在每个比较中包含随机基线**。如果你在协调任务上无法以较大优势击败随机基线，该任务可能是不良定义的。
- **报告成本和准确率。** 令牌成本和挂钟时间。运维团队两者都需要。
- **每季度重建基准测试。** 生产分布会变化；过时的基准测试具有误导性。
- **避免公开基准测试过拟合。** 如果你的团队专门针对 SWE-bench Pro 数字进行优化，你将在生产环境上退步。

## 练习

1. 运行 `code/main.py`。识别三个模拟系统中哪个具有最佳的每里程碑成本。它是否与最高原始准确率的系统一致？
2. 阅读 MultiAgentBench (arXiv:2503.01935)。针对你自己的任务领域，决定 MARBLE 会推荐四种拓扑中的哪一种。根据论文结果给出理由。
3. 阅读 SWE-bench Pro 论文。是什么使其具有抗污染能力？同样的技术能否应用到其他你关心的基准测试？
4. 阅读 COMMA 关于多模态协调的发现。设计一个简单的多模态协调任务，可以添加到你的内部基准测试中。什么才算是有用的信号？
5. 将基准测试宣称清单应用于最近一篇多智能体论文的标题结果。你会给该宣称打什么等级？
