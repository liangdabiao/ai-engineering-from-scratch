# 针对LLM的群体优化（PSO、ACO）

> 生物启发式优化正在LLM领域卷土重来。**LMPSO**（arXiv:2504.09247）使用PSO，其中每个粒子的速度是一个提示(prompt)，LLM生成下一个候选；在结构化序列输出（数学表达式、程序）上表现良好。**Model Swarms**（arXiv:2410.11163）将每个LLM专家视为模型权重流形上的一个PSO粒子，并报告在9个数据集上使用仅200个实例就比12个基线平均提升**13.3%**。**SwarmPrompt**（ICAART 2025）将PSO与灰狼(Grey Wolf)混合用于提示优化。**AMRO-S**（arXiv:2603.12933）是受ACO启发、针对多智能体LLM路由的信息素专家——**4.7倍加速**、可解释的路由证据、将推理与学习解耦的质量门控异步更新。本课在提示参数空间上实现PSO，在智能体路由上实现ACO，衡量这些经典算法为何适应当今LLM时代，以及何时不适合。

**类型：** 学习+构建
**语言：** Python（标准库）
**前置条件：** 阶段16·09（并行群体网络），阶段16·14（共识与BFT）
**时间：** 约75分钟

## 问题

你有一个在任务评估中得分62%的提示(prompt)。你想改进它。朴素的做法是无梯度的手动调整，但可扩展性差。强化学习需要奖励信号和足够的展开(rollout)来训练。通过提示进行反向传播实际上是不可能的——提示是一个离散字符串，而不是可微分参数。

经典的生物启发式优化——用于连续搜索空间的PSO和用于路径选择的ACO——正是为这种情况设计的：无梯度、基于群体、每次评估成本低。将它们与LLM配对用于无梯度搜索步骤，你就会得到一个非常实用的优化器。

相同的模式适用于多智能体系统中的智能体*路由*。ACO风格的信息素轨迹记录了哪个智能体在哪种任务类型上表现最好，让路由器利用该轨迹，并让信息素衰减，从而可以重新发现路由。

## 概念

### PSO复习（Kennedy & Eberhart 1995）

粒子群优化（Particle Swarm Optimization）：连续搜索空间中的粒子群体。每个粒子有位置`x_i`和速度`v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中`p_best`是粒子自身最佳，`g_best`是群体最佳，`w, c1, c2`是惯性 + 认知 + 社会权重，`r1, r2`是随机因子。

### LLM输出上的PSO——LMPSO

arXiv:2504.09247将PSO适配用于LLM生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个*提示(prompt)*，描述如何将当前输出修改为个人/全局最佳。LLM根据速度提示生成新输出。速度的“惯性”是一个类似“做小的增量改变”的提示。

在以下情况下效果良好：
- 输出是结构化的（可解析、可评估）。
- 适应度是自动的（测试运行、算术评估）。
- 群体规模小（约10-30个粒子），因此LLM总调用次数可控。

当适应度需要人工审查时效果不好——每次迭代的成本变得过高。

### Model Swarms

arXiv:2410.11163将PSO从输出层移到了*模型*层。每个“粒子”是一个专家LLM（参数）。群体通过无梯度更新将参数移向集体最佳。报告称：在9个数据集上，每个迭代仅使用200个实例，就比12个基线平均提升13.3%。

关键洞察是，LLM专家模型已经在共享的参数流形（适配器权重、LoRA增量）上彼此接近。在此低维子空间上的PSO既便宜又有效。

### ACO复习（Dorigo 1992）

蚁群优化（Ant Colony Optimization）：蚂蚁遍历图；每条路径有信息素轨迹。蚂蚁移动概率根据信息素强度加权。完成任务的好蚂蚁按解的质量比例沉积信息素。信息素随时间衰减。

### AMRO-S——用于智能体路由的ACO

arXiv:2603.12933使用ACO进行多智能体路由。每个任务类型是一个“目的地”；每个智能体是一条可能的路由。信息素强化产生良好输出的路由。关键贡献：

- **可解释的路由证据。** 信息素强度是人类可读的信号。
- **质量门控异步更新。** 信息素仅在质量检查通过后更新，将推理与学习解耦。
- **4.7倍加速** 在多智能体路由基准测试上。

质量门控很重要：没有它，快速但错误的智能体会积累信息素，系统就会锁定在错误的路由上。

### 何时在LLM中使用PSO/ACO

**使用PSO的情况：**
- 搜索空间是连续的或映射到连续参数（提示嵌入、LoRA权重、数值生成参数）。
- 适应度评估廉价且自动。
- 群体规模可以很小（10-30）。

**使用ACO的情况：**
- 你有路由或路径选择问题。
- 决策随时间强化（相同任务类型反复出现）。
- 你需要路由决策的可解释证据。

**两者都不使用的情况：**
- 适应度需要人工审查（每次迭代成本太高）。
- 搜索空间是离散且组合的，PSO无法覆盖（改用遗传算法）。
- 实时决策需要严格的延迟（PSO/ACO相对于单次启发式收敛较慢）。

### 为什么仿生方法仍然胜出

基于梯度的方法需要可微分的信号。大语言模型的输出和路由决策并非平凡可微。伪梯度方法（基于强化学习的学习路由器、DPO风格的提示调优器）有效但需要昂贵的训练。

PSO和ACO只需要一个*评估器*函数。如果你能够对候选输出或路由决策进行评分，就可以在该空间上进行优化。这使得适用门槛大大降低。

### 实际限制

- **群体预算.** N个粒子 × T次迭代 × 每次评估成本。对于大语言模型评估，大约$0.02 / call, a 20-particle PSO running 50 iterations costs ~$$20。请相应规划。
- **探索与利用.** 信息素衰减率和PSO惯性权重需要权衡；衰减过快→遗忘解；衰减过慢→陷入早期局部最优。
- **灾难性漂移.** 如果适应度景观发生变化（新数据分布），两种算法都可能收敛后发散。监控最佳适应度的稳定性。

## 动手构建

`code/main.py` 实现：

- `LMPSO` — 对数值提示参数（温度、top_k权重）进行PSO优化。每个粒子的“大语言模型生成”被模拟为脚本化的适应度函数。运行算法30次迭代，显示全局最优的收敛情况。
- `LMPSO` — ACO风格路由。3个智能体，4种任务类型，信息素矩阵，100个路由任务。输出随时间变化的任务类型→智能体选择分布，以展示路径形成过程。
- 对比：在相同任务流上随机路由与ACO路由。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO: 在30次迭代中，全局最优适应度从随机提升到接近最优。
- AMRO-S: 信息素表稳定在每个任务类型对应的正确智能体上；ACO路由在质量上比随机路由提升约30-40%，同时减少了延迟（重试次数更少）。

## 使用它

`outputs/skill-swarm-optimizer.md` 有助于在PSO、ACO、遗传算法和基于梯度的优化器之间进行选择，用于大语言模型/智能体优化问题。

## 发布

- **从小开始.** 10-20个粒子，20-50次迭代。只有当收敛曲线显示明显增益时才扩大规模。
- **记录每次迭代的信息素或全局最优.** 没有轨迹的群优化器调试起来很痛苦。
- **质量门控更新.** 特别是对于ACO路由：快速但错误的智能体不能积累信息素。
- **在分布偏移时重置衰减.** 当你的评估分布发生变化时，陈旧的信息素是过时的；暂时重置或加倍衰减率。
- **限制每次迭代成本.** 输出每次迭代的成本指标。每次迭代花费500美元且仅提升0.5%的PSO是不可交付的。

## 练习

1. 运行`code/main.py`。观察LMPSO收敛情况。改变群体大小5、10、20、50。在什么大小下收敛时间趋于饱和？
2. 实现一个“灾难性漂移”实验：在第30次迭代后改变适应度函数。PSO适应有多快？重置`code/main.py`是否有帮助？
3. 向AMRO-S添加质量门控：仅当评估分数>0.7时沉积信息素。与无门控版本相比，收敛性如何变化？
4. 阅读LMPSO (arXiv:2504.09247)。将论文中的“速度作为提示”映射回你的数值速度。模拟中丢失了什么？保留了什么？
5. 阅读AMRO-S (arXiv:2603.12933)。实现解耦的“推理快速路径”与异步信息素更新。在持续负载下，这对系统延迟有何影响？

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  PSO  |  "粒子群优化"  |  Kennedy-Eberhart 1995。基于种群的免梯度优化器。 |
|  ACO  |  "蚁群优化"  |  Dorigo 1992。通过信息素轨迹进行路径/路由优化。 |
|  LMPSO  |  "用于大语言模型生成的PSO"  |  arXiv:2504.09247。速度是提示；LLM生成候选。 |
|  Model Swarms  |  "对专家权重的PSO"  |  arXiv:2410.11163。在模型参数子空间上的免梯度更新。 |
|  AMRO-S  |  "用于智能体路由的ACO"  |  arXiv:2603.12933。任务类型×智能体的信息素矩阵。 |
|  p_best / g_best  |  "个体最佳 / 全局最佳"  |  每个粒子和整个群体迄今为止找到的最佳解。 |
|  Pheromone  |  "路由记忆"  |  边上的强度；随时间衰减；在质量上沉积。 |
|  Quality-gated update  |  "仅从好的运行中学习"  |  信息素沉积取决于质量检查。 |
|  Catastrophic drift  |  "分布偏移"  |  适应度景观变化；旧的个体最佳和信息素变得过时。 |
