# 投票、自一致性与辩论拓扑

> 最廉价的聚合方法：采样N个独立智能体，进行多数投票。Wang等人2022年的自一致性(Self-Consistency)通过单模型采样N次实现。多智能体用**异构**智能体扩展，以跳出单一文化——不同模型、不同提示词、不同温度、不同上下文。除多数投票外，辩论拓扑也很重要：MultiAgentBench (arXiv:2503.01935, ACL 2025) 评估了星形/链形/树形/图形协调，发现**图形最适合研究**，大约超过4个智能体后会产生“协调税(Coordination Tax)”。AgentVerse (ICLR 2024) 记录了两种涌现模式——志愿者行为(Volunteer Behaviors)和从众行为(Conformity Behaviors)，从众既是特性（达成共识）也是风险（群体思维，第24课）。本课将描绘拓扑空间，构建每种变体，并衡量协调税。

**类型：** 学习 + 构建
**语言：** Python (标准库)
**前置知识：** 阶段16 · 07 (思维社会与辩论)，阶段16 · 14 (共识与BFT)
**时间：** 约75分钟

## 问题

辩论可以提高准确性 (Du等人, arXiv:2305.14325)。也可能降低准确性。辩论是否有帮助取决于四个结构选择：

1. 谁与谁交谈 (拓扑)。
2. 多少轮次 (Du 2023：轮次和智能体数量各自独立地影响)。
3. 智能体是否异构 (不同的基础模型打破单一文化)。
4. 是否存在对抗性声音 (强化对方观点 vs. 稻草人谬误)。

那些简单地在任务上“运行5个智能体并投票”的团队，通常表现不如单个智能体。失败并非随机。它们与拓扑和异构性相关。本课就是一张拓扑地图。

## 概念

### 自一致性，单模型基线

Wang等人2022年（“自一致性改进了思维链推理”）在温度>0的条件下对同一模型采样N次，并对推理路径答案进行多数投票。在GSM8K上的结果：与单次贪心解码相比，N=40个样本带来了显著提升。自一致性是多智能体投票的单智能体前身。

局限性：自一致性使用一个基础模型。错误在构造上是相关的。如果模型存在系统性偏差，所有N个样本都会共享该偏差。

### 多智能体投票，异构扩展

用N个**不同**的智能体替换N个样本。不同的基础模型（Claude、GPT、Llama），不同的提示词，不同的工具访问权限。好处：不相关的错误。代价：不同智能体的成本不同；协调它们会增加开销。

2026年对异构辩论的规范名称是 **A-HMAD** — 对抗性异构多智能体辩论。并非普遍采用，但论文用该术语表示“不同模型进行辩论，减少了因单一文化崩溃导致的相关错误”。

### 四种拓扑

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

星形：一个中心节点，所有其他节点只与中心节点通信。相当于监督者-工作者模式，没有反向通道。
链形：线性，每个智能体看到前一个的输出。类似流水线。
树形：层次化，用于层次化智能体系统 (第06课)。
图形：任意到任意。包括全连接团和任意有向无环图。

### 协调税 (MultiAgentBench)

MultiAgentBench (MARBLE, ACL 2025, arXiv:2503.01935) 在包括研究、编码和规划等任务套件上对星形、链形、树形、图形进行了基准测试。关键测量结果：

- **图形**拓扑在研究任务上胜出。信息可以任意流动；智能体可以相互批评。
- **星形**在快速回答事实性任务上胜出。中心节点过滤并整合。
- **链形**在逐步流水线（分阶段细化）上胜出。
- **协调税**在图形拓扑中大约超过4个智能体后出现。墙钟时间和令牌成本的增长快于质量的提升。

4个智能体的上限是经验性的，而非基本限制。它反映了2026年大语言模型的上下文容量：每个智能体的上下文被其他智能体的输出填满，一旦每个人都能看到所有人，增加第N+1个智能体的边际价值就会下降。

### 多智能体辩论策略（“我们应该走向MAD吗？”）

arXiv:2311.17371 是2023年对MAD策略的综述。其他研究复制了关键发现：在相同预算下，与自一致性**结构相似**的MAD变体（独立采样+聚合）通常表现不如自一致性。MAD在智能体真正异构且辩论具有对抗结构（一个智能体反对另一个）时最有帮助。

### AgentVerse涌现模式

AgentVerse (ICLR 2024, https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf) 记录了即使在未显式设计的情况下，多智能体辩论中也会涌现出两种行为：

- **志愿者(Volunteer)。** 智能体主动提供帮助（“我可以做下一步”）。有用：它将任务分配给最擅长子任务的智能体。
- **从众(Conformity)。** 智能体调整自己的立场以迎合批评者，即使批评者是错的。这是辩论中的谄媚等价物 (第14课)。

从众正是为什么“辩论直到达成一致”会奖励霸凌者。带有限轮次和独立法官的方法可以缓解。

### 异构性：真正提升准确性的旋钮

在2024-2026年的实践文献中有一个模式：将N个智能体中的一个换成不同基础模型，所带来的准确性提升比增加一个智能体更大。直觉是单一文化——每个新的独立错误源比额外的相关样本更有价值。

在极限情况下，异构性胜过数量。在大多数具有清晰真实值的任务上，三个不同模型胜过同一模型的五个副本。

### 陪审团方法(Jury Methods)

Sibyl框架（在Minsky-LLM文献中被引用）形式化了“陪审团”——一小批专门化的智能体，在每个阶段通过投票来细化答案。与简单多数投票不同，陪审团有角色：一个智能体交叉检查，一个提供上下文，一个评估合理性。陪审团方法介于简单投票（廉价、易受单一文化影响）和完整MAD（昂贵、易受从众影响）之间。

### 何时“辩论式投票”占主导

- 问题有真实答案（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同来源或工具（存在异质性）。
- 轮次有上限（通常2-3轮），并且有独立的裁判或验证器。
- 预算允许3-5个智能体。在超过5-7个的图拓扑中，协调成本占主导地位。

### 辩论投票何时有损

- 问题是观点性的。智能体会收敛到看起来最自信的答案，而非最正确的。
- 所有智能体共享一个基础模型。单一文化使得共识毫无意义。
- 轮次无上限。从众每次都赢。
- 任务简单。单个智能体在N=5时自洽性更便宜且同样准确。

## 动手构建

`code/main.py` 实现：

- `run_star(agents, hub, question)` — 枢纽(hub)轮询每个工作者，聚合结果。
- `run_star(agents, hub, question)` — 顺序细化。
- `run_star(agents, hub, question)` — 层级深度2聚合。
- `run_star(agents, hub, question)` — 全连接辩论，有界轮次。
- 一个脚本化的异质性旋钮：每个智能体有一个`run_star(agents, hub, question)`指示其系统性错误倾向。
- 一个测量框架，以N=3、5、7运行每种拓扑并报告(准确率, 总令牌数, 模拟挂钟时间)。

运行：

```
python3 code/main.py
```

预期输出：一个拓扑×N → (准确率, 令牌数, 延迟)的表格。在研究型任务上，图拓扑在N=3-5时获胜；在快速事实型任务上，星型获胜；N=7时的图拓扑显示出协调成本（延迟增长快于准确率）。

## 使用它

`outputs/skill-topology-picker.md` 是一种技能，它读取任务描述并推荐拓扑（星型/链型/树型/图型）、智能体数量N、异质性配置（使用的基础模型）以及轮次上限。

## 发布

对于任何集成方法：

- 从**使用一个强基础模型在N=5时自洽性**开始。这是一个廉价的基准。
- 如果准确性重要，升级到**N=3异质性投票**。测量差异。
- **只有当任务具有结构（研究型、多步骤）且有界轮次可行时**，才升级到**辩论拓扑**。
- 始终记录少数派集群。当少数派持续正确时，你就获得了多样性信号。
- 将挂钟时间和令牌数与准确性一起基准测试。“以10倍成本换取更好准确性”是一个商业决策。

## 练习

1. 运行`code/main.py`。绘制图拓扑的协调成本曲线：准确率 vs N，令牌数 vs N。曲线在哪个N处拐点？
2. 实现A-HMAD：三个具有故意不同偏差的智能体。全相同偏差的基线如何与A-HMAD在第14课的单一文化攻击上比较？
3. 在图拓扑中添加一个“裁判”角色，不投票，仅对最终共识评分。这会改变出现的从众行为吗？
4. 阅读AgentVerse论文（ICLR 2024）。识别你的实现最强烈地表现出哪种涌现行为。你能通过提示更改来引发相反的行为吗？
5. 阅读MultiAgentBench（arXiv:2503.01935）第4节（拓扑实验）。使用你的框架在论文的一个任务上重现“图拓扑在研究任务上获胜”的结果。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
| 自洽性 | “采样N次，投票” | Wang 2022。单一模型，N个温度>0的样本，推理路径的多数投票。 |
| 异质性 | “不同模型” | 不同基础模型或提示族的集成。打破单一文化。 |
| MAD | “多智能体辩论” | 智能体在轮次中交换批评的通用术语。参见Du 2023。 |
| A-HMAD | “对抗性异质性MAD” | MAD变体，强调不同模型+对抗性结构。 |
| 拓扑 | “谁与谁交谈” | 星型、链型、树型、图型。确定信息流。 |
| 协调成本 | “递减回报” | 在图上超过约4个智能体时，成本增长快于质量。 |
| 志愿者行为 | “未提示的帮助” | AgentVerse涌现模式：智能体主动提出执行一步。 |
| 从众行为 | “压力下的一致” | AgentVerse涌现模式：智能体与批评者保持一致。 |
| 评审团(Jury) | “小型专业小组” | 西比尔式(Sibyl-style)集成，有角色（审查员、上下文提供者、评分者）。 |
