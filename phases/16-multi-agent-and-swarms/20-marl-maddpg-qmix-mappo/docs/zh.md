# MARL — MADDPG, QMIX, MAPPO

> 多智能体强化学习的遗产至今仍影响着2026年的LLM智能体系统。**MADDPG**（Lowe等人，NeurIPS 2017，arXiv:1706.02275）引入了集中式训练与分散式执行（CTDE）：每个评论家在训练期间观察所有智能体的状态和动作；测试时仅运行本地行动者。适用于合作、竞争和混合场景。**QMIX**（Rashid等人，ICML 2018，arXiv:1803.11485）是一种带有单调混合网络的价值分解方法；每个智能体的Q值组合成联合Q值，使得`argmax`可以干净地分配——在星际争霸多智能体挑战（SMAC）中占据主导地位。**MAPPO**（Yu等人，NeurIPS 2022，arXiv:2103.01955）是带有集中式价值函数的PPO；在粒子世界、SMAC、Google Research Football、Hanabi等任务中“出奇地有效”，且无需大量调参。这些算法为需要分散执行智能体团队的策略训练奠定了基础。MAPPO是**2026年合作MARL的默认基线**。本课将从一个小型网格世界玩具构建每种算法，并在接触LLM智能体训练之前将这三个思想牢固地植入肌肉记忆。

## 问题

LLM智能体系统越来越多地训练用于智能体间协调的策略：何时等待，何时行动，调用哪个同伴。描述如何训练此类策略的文献是多智能体强化学习（MARL），它早于LLM浪潮，并拥有一小套主流算法。

阅读MARL论文而不掌握模式词汇是痛苦的。集中式训练与分散式执行（CTDE）、价值分解和集中式评论家并非流行词——它们是对特定问题的特定答案：

- 独立强化学习（每个智能体独立学习）从每个智能体的角度来看是非平稳的。糟糕。
- 集中式强化学习（一个智能体控制所有）无法扩展且违反执行约束。
- CTDE兼顾两者：训练时使用全局信息，部署时使用局部策略。

## 概念

### 论文中使用的三个环境

- **粒子世界（多智能体粒子环境）。** 简单的2D物理，包含合作/竞争任务。MADDPG的原始测试平台。
- **星际争霸多智能体挑战（SMAC）。** 合作微操作，部分可观测。QMIX的测试平台。离散动作，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO基准环境。

不同环境具有不同的动作/观测类型。算法据此选择。

### MADDPG（2017）—— CTDE模式

每个智能体`i`拥有一个行动者`mu_i(o_i)`，将其自身观测映射到动作。每个智能体还拥有一个评论家`Q_i(x, a_1, ..., a_n)`，在训练期间观测所有智能体的观测和所有动作。行动者根据评论家的评估通过策略梯度进行更新。

```
actor update:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
critic update:   TD on Q_i(x, a_1..n) given next-state joint estimate
```

为什么CTDE：训练时，我们知道每个智能体的动作；我们利用这些信息来降低每个评论家的方差。部署时，每个智能体仅看到`o_i`并调用`mu_i(o_i)`。

失败模式：评论家随N个智能体增长（输入包含所有动作）。在没有近似的情况下，无法扩展到约10个智能体以上。

### QMIX（2018）—— 价值分解

仅限合作。全局奖励是每个智能体Q值的单调函数之和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证了`argmax_a Q_tot`可以通过每个智能体独立选择`argmax_{a_i} Q_i`来计算。这正是你需要的**分散式执行属性**。训练时，混合网络根据每个智能体的Q值生成`Q_tot`。

为什么QMIX在SMAC上获胜：合作星际争霸微操作具有同质智能体、局部观测、全局奖励——完美契合价值分解。

失败模式：单调性约束具有限制性；某些任务具有非单调可分解的奖励结构（如一个智能体为团队牺牲）。扩展方法（QTRAN、QPLEX）放松了这一约束。

### MAPPO（2022）—— 被忽视的默认方法

多智能体PPO：带有集中式价值函数的PPO。每个智能体拥有自己的策略；所有智能体共享（或拥有各自）价值函数，这些函数能观测到完整状态。Yu等人2022年将MAPPO与MADDPG、QMIX及其扩展方法在五个基准上进行比较，发现：

- MAPPO在粒子世界、SMAC、Google Research Football、Hanabi、MPE上达到或超过离策略MARL方法。
- 几乎不需要超参数调优。
- 训练稳定；不同随机种子下可重复。

社区在此论文之前一直低估了在策略MARL。到2026年，MAPPO是合作MARL的默认基线；任何新方法都必须超越它。

### 为什么LLM智能体工程师应该关注

三个直接用途：

1. **路由器训练。** 一个元智能体选择哪个子智能体处理任务。这是一个包含N个分散子智能体和一个集中式路由器的MARL问题。MAPPO适合。
2. **角色涌现。** 在生成式智能体模拟中，训练智能体随时间采用互补角色，这本质上是一个MARL问题。QMIX风格的价值分解通过构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过CTDE进行训练可以生成可部署的本地策略，这些策略尊重资源约束。

实际注意事项：在2026年，大多数生产级LLM智能体系统是通过提示来设定策略，而非训练策略。MARL适用于以下情况：(a) 有大量交互数据，(b) 有明确的奖励信号，(c) 愿意投资训练基础设施。

### CTDE作为超越强化学习的设计模式

即使不进行训练，CTDE也是一种有用的架构模式：

- 在**设计**阶段，假设完全团队可见性。
- 在**运行时**，强制分散执行：每个智能体仅看到`o_i`。

这种模式迫使你明确每个智能体的状态，并提前考虑部分可观测性。许多生产级多智能体系统默认假设所有地方共享状态——CTDE纪律可以防止这种情况。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（包含其他智能体的策略）是非平稳的。经典的单智能体强化学习证明失效。本课中的MARL算法都解决了这一问题：

- MADDPG：全局评论家看到所有动作，因此其价值估计是平稳的。
- QMIX：价值分解将学习移到一个联合Q空间，其中最优性定义清晰。
- MAPPO：集中式价值函数抑制了其他策略变化带来的方差。

在LLM智能体系统中，非平稳性表现为“我的智能体上个月还能工作，现在上游其他智能体变了，我的智能体就出错了”。使用CTDE训练MARL是原则性的修复方法；提示级别的修复更快但持久性较差。

### 本课未涵盖的内容

实际训练网络是第09阶段的话题。本课构建了脚本化策略版本，演示CTDE、价值分解和集中式价值模式，无需梯度更新。目标是在使用完整MARL库（PyMARL、MARLlib、RLlib多智能体）之前内化这些模式。

## 动手构建

`code/main.py`实现了三种模式演示，全部在一个小型2智能体协作网格世界中：

- 环境：2个智能体在4x4网格上，一个奖励小球。如果任一智能体到达小球则奖励为1；任务结束。
- `IndependentAgents`——每个智能体将其他智能体视为环境。基线。
- `IndependentAgents`——集中式评论家计算联合价值；行动者策略据此更新。脚本化策略改进。
- `IndependentAgents`——使用单调混合器的价值分解。
- `IndependentAgents`——集中式价值函数；策略根据共享基线更新。

所有四种方法运行相同的回合，并报告平均到达目标步数。CTDE变体收敛到比独立基线更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约6步；CTDE变体收敛到约3.5步（4x4网格最优为3步）。尽管使用脚本化策略，模式差异仍然显现。

## 发布

MARL在生产中很少见。当您确实使用时：

- **从MAPPO开始。** 2022年的论文将其确立为基线；先复现它可以节省数周追逐更复杂方法的时间。
- **记录每个智能体的观察和动作流。** 没有每个智能体的轨迹，调试MARL是毫无希望的。
- **将训练代码与执行代码分离。** CTDE是一种纪律；让执行路径真的只看到`o_i`。
- **奖励塑形警告。** MARL对奖励设计极其敏感。塑形中一个协调错误会让智能体学会利用它。运行对抗性测试。
- **对于LLM智能体**，首先考虑提示级别的策略。只有当交互数据、奖励信号和基础设施都具备时，才投资于MARL训练。

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  MARL  |  "多智能体强化学习"  |  多智能体系统的强化学习。  |
|  CTDE  |  "集中式训练，分散式执行"  |  使用全局信息训练；使用本地策略部署。  |
|  MADDPG  |  "多智能体DDPG"  |  每个智能体评论家看到所有观察和动作的CTDE。  |
|  QMIX  |  "价值分解"  |  每个智能体Q值的单调混合。协作式。  |
|  MAPPO  |  "多智能体PPO"  |  带有集中式价值函数的PPO。2026年默认基线。  |
|  价值分解  |  "单个Q值之和"  |  联合Q表示为每个智能体Q值的单调函数。  |
|  非平稳性  |  "移动目标"  |  每个智能体的环境随着其他智能体学习而变化。核心MARL问题。  |
|  在策略/离策略  |  "从当前/回放中学习"  |  PPO是在策略（MAPPO）；DDPG和Q学习是离策略。  |
|  SMAC  |  "星际争霸多智能体挑战"  |  协作微操基准；QMIX的诞生地。  |
