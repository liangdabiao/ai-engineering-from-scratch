# 监督者/编排者-工作者模式

> 一个主导代理(agent)负责规划和委派；专门工作者在并行上下文中执行并汇报结果。这是Anthropic研究系统背后的模式（Claude Opus 4为主导，Sonnet 4为子代理），在内部研究评估中比单代理Opus 4高出+90.2%。Anthropic的工程博文报告称，BrowseComp上80%的差异仅由token用量解释——多代理获胜主要是因为每个子代理都有全新的上下文窗口。本课从基础构件构建监督者模式，并涵盖2026年生产部署的工程经验。

**类型：** 学习+构建
**语言：** Python (标准库, @@SKOP0000@@)
**前置条件：** 阶段16 · 04 (基础模型)
**时间：** ~75分钟

## 问题

研究是单代理系统无法完成的典型任务。你问“2023年到2026年间多代理系统发生了什么变化？”一个单代理顺序阅读五篇论文，用它们的文本填满一半上下文，然后必须一起推理所有内容。等它读到第五篇时，已经忘记了第一篇。它无法并行化。

监督者模式解决了这个问题：一个主导代理规划搜索，将每个子问题委派给工作者，并综合结果。每个工作者拥有自己的20万token窗口来处理一个狭窄的问题。主导代理从未看到原始论文——只看到工作者的摘要。

Anthropic的生产研究系统报告称，与单个Opus 4相比，内部研究评估提高了+90.2%。同一篇博文指出，BrowseComp上80%的差异由*仅token用量*解释。每个子代理拥有全新上下文是主要机制。

## 概念

### 该模式

```
                 ┌──────────────┐
                 │   Lead       │  plans, decomposes,
                 │  (Opus 4)    │  synthesizes
                 └──┬────┬───┬──┘
                    │    │   │
            ┌───────┘    │   └───────┐
            ▼            ▼           ▼
      ┌─────────┐  ┌─────────┐  ┌─────────┐
      │ Worker1 │  │ Worker2 │  │ Worker3 │
      │(Sonnet) │  │(Sonnet) │  │(Sonnet) │
      └─────────┘  └─────────┘  └─────────┘
         fresh       fresh        fresh
         context     context      context
```

主导代理从不阅读原始材料。工作者在主导综合之前不会看到彼此的工作。每个箭头是一次带有狭窄产物的交接。

### 为何胜出

三个机制：

1. **每个子代理都有全新上下文。** 一个探索“FIPA-ACL遗产”的工作者不会携带主导规划时花费的4万token。它获得20万token窗口来处理一个问题。
2. **通过提示专门化。** 主导的提示是“分解并综合”，而不是“研究”。每个工作者的提示是狭窄的：“找出X中发生了什么变化”。专注的提示产生专注的输出。
3. **并行性。** 工作者并发运行。挂钟时间大约是`max(worker_times) + plan + synthesis`，而不是`sum(worker_times)`。

### 工程经验（Anthropic 2025）

Anthropic的博文列出了几条在2026年仍然相关的生产经验：

- **根据查询复杂度调整工作量。** 简单查询：一个代理，3-10次工具调用。复杂查询：10+个代理。主导必须自行估算，而不是由调用者估算。
- **先宽后窄。** 首先分解为宽泛的子问题，如果答案需要深度，再为每个子问题衍生更多工作者。
- **彩虹部署。** 代理是长期运行且有状态的。传统的蓝绿部署不起作用。Anthropic使用彩虹部署：逐步推出新版本，同时旧版本逐渐退出。
- **token用量占主导。** 多代理的token用量大约是单代理的15倍。仅在任务价值证明成本合理时才运行。

### LangGraph的转变

LangGraph最初发布了一个`langgraph-supervisor`库，带有一个高级`create_supervisor`辅助函数。2025年，LangChain将建议改为通过直接工具调用实现监督者模式，因为工具调用在*监督者看到的内容*（上下文工程）方面提供了更多控制。该库仍然有效；文档现在推荐工具调用的形式。

### 失败模式

- **主导产生幻觉计划。** 如果主导生成的子问题并未分解真实问题，工作者就会在错误目标上进行精确研究。
- **工作者过度探索。** 如果没有明确的范围边界，工作者会偏离分配的子问题，污染综合步骤。
- **综合冲突。** 两个工作者返回矛盾的事实。主导必须要么重新询问（增加一轮），要么明确记录分歧。默默选择一方是最糟糕的失败：用户永远不知道发生了分歧。

### 何时监督者模式不适用

- **顺序任务。** 如果步骤2确实需要步骤1的输出，并行性毫无意义。使用流水线（CrewAI Sequential，LangGraph线性图）。
- **简单查询。** 单代理处理更快更便宜。在产生工作者之前，使用主导的“工作量调整”检查。
- **严格确定性。** 监督者使用LLM选择的委派。当审计/重放比适应性更重要时，静态图更好。

## 动手构建

`code/main.py`使用`threading`实现了一个有三个并行工作者的监督者。主导将查询分解为子问题，工作者在各自子问题上并发运行，主导进行综合。没有真正的LLM——工作者被脚本化为模拟“获取并总结”。

关键结构：

- `Lead.plan(query)`将查询拆分为3个子问题。
- `Lead.plan(query)`返回一个假摘要（生产环境中可以是任何使用工具的代理）。
- `Lead.plan(query)`在线程中启动工作者，等待完成，然后综合。

运行：

```
python3 code/main.py
```

输出展示了计划、带有开始/结束时间戳的并行工作者轨迹以及最终的综合结果。你可以看到挂钟时间的优势：三个0.3秒的工作者在约0.35秒内运行完毕，而不是0.9秒。

## 使用它

`outputs/skill-supervisor-designer.md`接受用户查询并产生一个监督者模式的设计：主导系统提示、工作者角色、子问题分解规则以及综合模板。在构建新的研究型代理系统之前使用此功能。

## 发布

部署监督者模式前的检查清单：

- **模型配对。** 主导使用推理型模型（Opus类，`o3`类）。工作者使用更快更便宜的模型（Sonnet，`o4-mini`）。
- **工作者超时。** 任何超过2倍中位数运行时间的工作者被终止；主导要么重新产生范围更窄的工作者，要么不等待它继续。
- **每个工作者的token上限。** 硬性限制（例如预期综合输入的10倍）防止失控的工作者超出预算。
- **可观测性。** 追踪主导的计划、每个工作者的工具调用以及综合结果。这是事后调试的基础。
- **彩虹部署。** 有状态的长期运行代理需要逐步版本过渡，而不是热切换。

## 练习

1. 运行`code/main.py`，然后将主导修改为产生5个工作者而不是3个。观察挂钟时间的影响。在这个演示中，当工作者数量达到多少时，产生开销超过并行节省？
2. 实现工作者超时：杀死任何运行超过0.5秒的工作者，并让主导综合剩余结果。你需要哪些可观测性来知道某个工作者被截断？
3. 在主导的综合步骤中添加冲突检测：如果两个工作者返回矛盾的答案，主导记录分歧而不是选择其中一个。如何在不调用LLM的情况下检测矛盾？
4. 阅读Anthropic的研究系统工程博文。列出这个玩具演示需要采用才能在生产的三个实践。
5. 比较LangGraph的`code/main.py`（旧版）与新的工具调用推荐。哪个能更好地控制监督者看到的内容？为什么Anthropic明确只将子答案（而不是工作者原始上下文）传递给综合步骤？

## 关键术语

|  术语  |  人们的说法  |  实际含义  |
|------|----------------|------------------------|
|  Supervisor  |  "主导代理"  |  一个负责规划、委派和综合的编排代理。本身不执行具体工作。 |
|  Worker  |  "子代理"  |  由监督者调用的专注代理，具有狭窄范围和自己的上下文窗口。 |
| 编排者-工作者  |  "监督者模式"  |  同一事物，不同名称。2026年的文献同时使用这两个术语。 |
| 全新上下文  |  "干净窗口"  |  工作者的上下文从系统提示和分配的问题开始，而不是领导者的历史记录。 |
| 彩虹部署  |  "渐进式推出"  |  长期运行的有状态智能体需要版本化的排空替换，而非蓝绿部署。 |
| Token主导  |  "上下文即变量"  |  根据Anthropic的研究，80%的研究评估方差来自使用的Token总数，而非模型选择。 |
| 规模调整  |  "根据复杂度匹配智能体数量"  |  领导者评估查询难度，相应产生1个或10+个工作者。 |
| 综合冲突  |  "工作者意见分歧"  |  两个工作者返回矛盾的事实；领导者必须暴露分歧，而不是默默选择其中一个。 |
